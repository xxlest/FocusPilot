# Pilot 产品家族

> **状态**：规划中（当前仅实现 FocusPilot 本地功能，其余为远期规划）
> **基于**：FocusPilot V4.3 → Pilot 产品家族升级

---

## 1. 版本总览

| | **FocusPilot**（个人版） | **PilotOne**（企业版） |
|---|---|---|
| 口号 | You think. FocusPilot builds, learns, and remembers. | AI 洞察、决策、执行。你只需掌舵。 |
| 一句话 | 个人 AI 副驾驶 | 企业 AI 经营系统 |
| 面向 | 独立开发者、重度知识工作者、终身学习者 | 5-50 人小团队（老板+员工+AI） |
| 谁驱动 | 人驱动，AI 辅助 | AI 驱动，人在关键节点确认/审批/补充 |
| 部署 | 本地 macOS App（+ 可选云端 Engine） | 云端 Engine + Web + 飞书/钉钉 |
| 注册 | 不需要（纯本地）/ 需要（云端功能） | 需要 |
| AI 能力 | 自带 API Key / 含额度 | 公司 Key / 含额度 |
| 关系 | 个人产品线 | 企业产品线，复用同一 Engine |

**FocusPilot** 提供完整的 macOS 桌面体验——悬浮球、Quick Panel 窗口快切、番茄钟、项目管理（四模式/两阶段/Task 双轴）、本地数字专家（代码工程师）、知识管道（素材收集→整合报告→知识卡片→Anki 同步）、可选的云端 Engine（7x24 常驻）、Web App 和飞书 Bot 远程访问。核心循环：想法 → Task → Agent 执行 → 个人知识成长。

**PilotOne** 是面向团队的 AI 经营系统。老板设定战略目标，AI 自动拆解为可执行任务，由数字专家团（战略参谋、运营专家、获客专家、客户专家、知识官）自动执行，人类员工在关键节点做决策确认和信息补充。核心循环：战略 → AI 洞察 → 决策 → 执行 → 复盘 → 组织进化。

FocusPilot 和 PilotOne 不存在包含关系，是两条独立产品线。

### 2.2 战略驾驶舱

老板的统一入口，AI 主动提问而非老板去填表。

| 功能 | 说明 |
|------|------|
| 每日聚焦 | AI 提问"今天最重要的 3 件事"，结合内外部数据给出建议 |
| 目标拆解 | 从战略目标自动拆解为本周/今日可执行任务，分配给专家团 |
| 经营仪表盘 | 关键指标一览（营收/客户/产能/专家团状态/成本） |
| 经营洞察 | 部门级/全局级/行业级三维洞察，AI 主动推送异常和趋势 |
| 决策建议 | 基于数据+历史经验，AI 给出可选方案，老板确认即执行 |

### 2.3 数字专家团

PilotOne 的核心交互概念——**用户面对的不是技术系统，而是一个专家团队**。每个数字专家 = 一个配置好的高级 AI Agent 角色，能洞察、能决策、能执行。

#### 五大通用角色

**🧠 战略参谋**

| 职责 | 说明 |
|------|------|
| 经营洞察 | 营收趋势、成本结构、利润分析、同比环比 |
| 客户价值分析 | ABC 分析法分层客户，识别优质/淘汰客户 |
| 供应链分析 | 供应商横向纵向比价，推荐最优进价 |
| 人效分析 | 员工投入产出比、工时分析、产能利用率 |
| 行业洞察 | 行业动态、竞品监控、市场趋势 |
| 决策支持 | 基于数据+历史经验生成决策建议，供老板确认 |

**👔 运营专家**

| 职责 | 说明 |
|------|------|
| 采购流水线 | 需求汇总 → 供应商比价 → 推荐 → 审批(人) → 下单 |
| 生产交付线 | 接单 → 排产 → 执行 → 质检(人) → 交付 → 回款 |
| 财务对账 | 订单毛利核算、跨系统数据对齐、账单生成+发送 |
| 内部流程 | 审批流转、进度跟踪、超时提醒 |

**📢 获客专家**

| 职责 | 说明 |
|------|------|
| 内容生产 | 公众号文章、短视频脚本、种草图文，AI 批量生产 |
| 多平台分发 | 抖音/小红书/视频号/公众号一键同步 |
| 商机监控 | 自动搜索招投标信息，匹配公司资质，评分推荐 |
| SEO/全域获客 | 各 AI 平台智能体布局（DeepSeek/豆包/文心等） |
| 个人 IP 打造 | 内容规划、发布节奏、数据复盘 |

**🤝 客户专家**

| 职责 | 说明 |
|------|------|
| 客户分层 | ABC 分析，自动标记优质/普通/待淘汰 |
| 定期维护 | 节假日问候、促销推送、日历驱动的客户触达 |
| 流失预警 | 监测客户活跃度，预警流失风险，建议挽留动作 |
| 复购促进 | 基于历史订单推荐复购时机和产品 |
| 账单管理 | 月度账单自动生成+发送+回款提醒 |

**📚 知识官**

| 职责 | 说明 |
|------|------|
| 经验归档 | 每次项目/决策/执行的结果自动归档 |
| 规则提炼 | 从历史经验中提炼 SOP、定价策略、决策规则 |
| 自迭代 | 新经验 → 更新规则 → 老板确认 → 全团队生效 |
| 决策参考 | 其他专家执行时自动调取"上次类似情况怎么做的" |

#### 人机协作模式

数字专家团中，**AI 为主、人为辅**。人类员工在流水线中承担 AI 无法完成的节点：

```
[AI] 战略参谋分析本周重点
  → [AI] 运营专家拆解为执行任务
  → [AI] 获客专家生成内容初稿
  → [人] 老板审核内容方向          ← 决策确认
  → [AI] 获客专家排版+多平台发布
  → [AI] 客户专家跟进互动数据
  → [AI] 知识官归档本次经验
```

人只在三种场景介入：**决策确认**（方向对不对）、**信息补充**（AI 不知道的）、**审批授权**（涉及资金/对外）。其余全部由专家团自动完成。

### 2.4 系统集成层（Bridge）

作为基础设施，同时为战略驾驶舱和数字专家团提供数据能力。

| 连接器类型 | 说明 |
|-----------|------|
| ERP 连接器 | 读取/写入订单、库存、财务数据；跨系统数据对齐 |
| CRM 连接器 | 客户数据同步、跟进记录、客户画像 |
| 平台连接器 | 抖音/小红书/视频号/公众号 API，内容发布+数据回收 |
| AI 模型连接器 | 多模型调度（Claude/DeepSeek/豆包/文心） |
| 通信连接器 | 飞书/钉钉/微信/短信，消息推送+Bot 交互 |
| 数据桥接 | 跨系统字段映射、数据冲突裁决、增量同步 |

### 2.5 企业知识库（OrgBrain）

作为基础设施，所有专家的执行记录和决策经验自动沉淀，使专家团越用越懂这家公司。

| 层 | 内容 | 来源 | 用途 |
|---|---|---|---|
| **业务数据层** | 客户数据、订单数据、供应商数据、财务数据 | ERP/CRM/平台自动同步 | 专家团实时查询和分析的基础 |
| **经验沉淀层** | 项目复盘、报价经验、客户偏好、成功案例、失败教训 | 专家团执行后自动归档 + 人工补充 | "上次类似项目怎么做的" |
| **规则引擎层** | SOP、审批规则、定价策略、客户分级标准 | 老板/管理者设定 + AI 从经验中提炼 | 专家团执行时的决策依据 |

**自迭代机制**：

```
专家团执行任务
  → 结果自动归档到经验沉淀层
  → AI 发现规律后建议更新规则引擎层
  → 老板确认 → 规则更新
  → 专家团下次执行时自动使用新规则
  → 越用越准
```

**与 FocusPilot 个人知识库的对比**：

| | FocusPilot（个人） | PilotOne（企业） |
|---|---|---|
| 本质 | 个人学习记忆系统 | 企业业务记忆系统 |
| 输入 | 文章/笔记/论文 | 业务数据/项目记录/执行日志 |
| 加工 | 整合报告→知识卡片→Anki | 经验归档→规则提炼→决策参考 |
| 目标 | 个人变聪明 | 组织变聪明（专家团越来越懂这家公司） |

## 4. 两条产品线的本质区别

| | **FocusPilot** | **PilotOne** |

## 5. 付费订阅方案

### 5.1 定价模型

| 版本 | 定价模式 | 参考价格 | 包含 |
|------|---------|---------|------|
| **FocusPilot（本地）** | 免费 / 低价买断 | ¥0 或 ¥98 买断 | 全部本地功能，用户自带 API Key |

### 5.2 AI 额度

| 版本 | AI 能力来源 | 说明 |
|------|-----------|------|
| FocusPilot（本地） | 用户自带 API Key | Claude/OpenAI Key，不承担 AI 成本 |

| 计费项 | 说明 |

### 5.4 版本升级路径

```

```
