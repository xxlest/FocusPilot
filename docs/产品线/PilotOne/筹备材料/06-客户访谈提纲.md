# 客户访谈提纲

## 一、文档目的

本文档用于在项目早期进行客户访谈时，帮助统一提问框架、记录方式和判断标准。

客户访谈的目标不是推销方案，而是验证以下问题：

1. 客户是否真的存在明确痛点
2. 当前问题是怎么被解决的
3. 现有替代方案到底有多强
4. 谁是使用者，谁是买单人
5. 客户是否愿意给试点机会或真实数据

## 二、适用访谈对象

建议优先访谈以下角色：

- 代建公司管理层
- 代建公司项目负责人
- 房地产代理销售公司管理者
- 开发商区域营销负责人
- 项目 PM / 单项目营销负责人
- AMC / 问题项目接管相关角色

## 三、关于“内部行业专家样本”的特殊说明

如果联合发起人本身就在房地产行业深耕多年，例如长期做地产代理、策划、销售管理或项目操盘，那么他可以作为一个非常高质量的行业专家样本。

但必须明确：

### 1. 他可以替代什么

他可以帮助你们快速完成以下几类认知验证：

- 行业基本流程理解
- 行业术语和真实工作习惯理解
- 一线高频痛点识别
- 哪些问题是伪需求、哪些问题是真需求的初步筛选
- 第一批潜在客户名单的获取

也就是说，他可以显著降低你们“从零理解行业”的成本。

### 2. 他不能替代什么

他不能替代以下关键市场验证：

- 不同公司类型之间的差异验证
- 不同买单人的采购逻辑验证
- 客户是否愿意试点的验证
- 客户是否愿意提供真实数据的验证
- 新流程是否被外部客户接受的验证

因此，不能把“一个很懂行业的人认可这个方向”误判成“市场已经认可这个方向”。

### 3. 正确使用方式

更合理的方式是把这类角色视为：

- 1 个高质量行业专家样本
- 持续的领域顾问来源

而不是：

- 整个市场的代表

### 4. 对访谈数量的理解

“10 个访谈”不是绝对硬指标，真正重要的是：

- 访谈对象是否足够多样
- 是否覆盖不同角色
- 是否覆盖使用者和买单人

如果资源有限，第一轮更实际的最低可行组合是：

- 1 个内部行业专家样本
- 2-3 个同类型外部从业者
- 2-3 个潜在买单人
- 1-2 个潜在使用者

这样即使总数不到 10 个，也比只访谈单一类型对象更有价值。

## 四、访谈原则

### 1. 不要一开始讲产品

先理解客户，而不是先教育客户。

### 2. 问现状，不问假设

少问：

- “如果有一个 AI 系统你会不会用？”

多问：

- “你们现在怎么做？”
- “上周最近一次怎么处理？”
- “是谁在做？”

### 3. 问真实行为，不问礼貌性意见

少问：

- “你觉得这个方向怎么样？”

多问：

- “你们最近有没有为这个问题花钱？”
- “如果试点，谁能拍板？”

### 4. 目标是找到证据，不是找到认同

客户说“挺好”“很有意思”不算验证。

以下才算强信号：

- 愿意提供真实数据
- 愿意安排内部人继续讨论
- 愿意跑试点
- 愿意讨论预算和价格

## 五、标准访谈结构

建议一场访谈控制在 30-60 分钟。

### 第一部分：角色和背景

目的：

- 弄清楚对方是谁
- 在组织里负责什么
- 是否真有该问题的发言权

建议问题：

1. 你现在负责的具体业务是什么？
2. 你目前管几个项目/团队？
3. 你平时最关注哪些经营指标？
4. 你在这类系统或工具采购上有没有话语权？

## 六、现状问题访谈

目的：

- 了解客户现在怎么做
- 找出最痛的环节

建议问题：

1. 你们现在平时怎么做项目经营分析/周报/经营复盘？
2. 谁负责整理这些内容？
3. 从数据收集到最后形成判断，一般要多久？
4. 最容易卡在哪些环节？
5. 最近一次让你觉得最麻烦的经营分析/汇报是什么？
6. 现在最不透明、最难判断的问题是什么？

## 七、替代方案访谈

目的：

- 找到现有替代品
- 判断你未来真正的对手是谁

建议问题：

1. 你们现在主要靠什么方式解决这些问题？
2. 是靠 Excel、BI、人工会议，还是现有系统？
3. 目前有哪些供应商或内部系统在支持这件事？
4. 这些方式最大的不足是什么？
5. 为什么这些方法到现在还没彻底解决问题？

## 八、价值判断访谈

目的：

- 判断这个问题在客户心里到底有多重要

建议问题：

1. 如果这个问题解决得更好，对你们最大的价值是什么？
2. 更快发现问题、更快出判断，对你们是“方便”还是“关键”？
3. 你更关心的是节省人力，还是提升经营判断质量？
4. 如果这个问题长期解决不好，后果通常是什么？

## 九、买单与决策访谈

目的：

- 判断商业路径是否成立

建议问题：

1. 如果真要引入一套新工具，一般是谁先提需求？
2. 谁能拍板？
3. 预算一般从哪个口径走？
4. 决策流程一般多久？
5. 你们过去一年有没有买过类似产品或服务？
6. 如果是试点，一般多大金额以内你们可以快速决策？

## 十、试点可能性访谈

目的：

- 判断对方是不是种子客户候选

建议问题：

1. 如果我们先不谈完整产品，只做一个小范围试点，你们会不会有兴趣？
2. 你们内部有没有一个项目适合先试？
3. 如果试点，需要哪些数据或哪些人参与？
4. 你们最希望先验证哪个场景？
5. 如果要推进下一步，应该找谁继续对接？

## 十一、访谈结尾建议

结束时不要只说“谢谢”，建议明确推进下一步。

可以这样收尾：

> 今天主要是想理解你们的真实情况，不急着讲完整方案。听下来我觉得有几个点比较值得继续验证。后面如果我们整理一版更贴近你们场景的试点思路，方便的话我再和你们做一次更聚焦的交流。

## 十二、每次访谈后必须记录的内容

建议每次访谈后立刻记录以下内容：

### 基础信息

- 公司名称
- 访谈对象
- 岗位
- 日期

### 关键结论

- 当前痛点是什么
- 当前替代方案是什么
- 谁是使用者
- 谁是买单人
- 是否存在预算
- 是否愿意试点

### 主观判断

- 痛点强度：高 / 中 / 低
- 付费意愿：高 / 中 / 低
- 决策难度：高 / 中 / 低
- 是否值得继续跟进：是 / 否

## 十三、访谈后的判断标准

如果一个客户满足以下条件中的 3 条以上，可以列入重点跟进对象：

- 痛点明确且频繁
- 现有替代方案明显不满意
- 有明确买单人
- 愿意继续深入沟通
- 愿意给试点机会
- 愿意讨论预算

如果只是觉得“挺有意思”，但没有任何后续动作，不应高估。

## 十四、最后提醒

客户访谈最大的误区是把“对方觉得不错”误当成“市场已经验证”。

真正有效的访谈结果，不是得到赞美，而是得到：

- 真实痛点
- 真实流程
- 真实阻力
- 真实决策路径
- 真实试点机会

对于内部行业专家样本，同样适用这个原则：

- 他的判断非常有价值
- 但他的判断不能替代外部客户的真实动作
