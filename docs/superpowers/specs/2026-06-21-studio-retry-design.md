# Studio 执行失败/超时 一键重试 — 设计 spec

> 日期：2026-06-21 ｜ 载体：**V1 Studio `run_state`**（不动 V4.3 Swift 代码）
> 来源：本会话头脑风暴（载体确认 → 语义方案 A → 形态方案 A → 覆盖默认 → review 修正版）

## 1. 背景与定位

V1 Studio 的执行状态轴 `run_state` 已有两个终止态 `timeout`（执行超时）/`failed`（执行失败），原型说明也写了"等人工**重试**/干预"，但**缺三样东西**：①「重试」这个动作在状态机上的定义；②触发它的统一交互入口；③在各视图的露出规则。本 spec 补齐这三样。

非本 spec 范围：V4.3 悬浮球应用（coder-bridge 单向上报监控架构，无法真正重试，已在调研中排除）。

## 2. 状态语义（`run_state` 轴）

- **触发态**：`run_state ∈ { failed(#DC2626), timeout(#EF4444) }` 两个引擎终止态。**不新增枚举**；"卡死 / 没反馈"归入 `timeout`（Run 时长超上限自动中止）。
- **重试动作**：
  1. 清当前（已 aborted）Run 标记 → `run_state = queued`（status **不变**，仍 `in_progress`）；
  2. 下轮中央扫描器在**保留的 worktree / 路径锁 + 同一 session** 上 `resume` 上下文，起一个**新 executor Run**（类 `claude -p --resume`）→ 抢槽 → `running`。
- **关键澄清（review 修正）**：这是「起新 Run + resume 上下文」**并存**，对齐 [§3.3 派发范式](../../fp-ui/04-studio.md:319)，**不是**走 blocked 流程，也**不是** fresh 从头重跑。"不复用旧 Run" 指不复用那个已 aborted 的 **Run 实例**，上下文靠 worktree/session 跨 Run 保持（[§3.3:325](../../fp-ui/04-studio.md:325)）。
- **worktree 保留依据**：失败/超时时任务 status 仍 `in_progress`，未离开执行流，按 [§3.3 路径锁/worktree 任务级持有规则](../../fp-ui/04-studio.md:306) 不释放 → resume 成立。
- **与 `blocked` 划清**：`blocked` = 外部依赖（V1 仅人工标/解除，[§3.2:283](../../fp-ui/04-studio.md:283)）；`failed/timeout` = 引擎内失败（一键重试）。两种语义不混。
- **即时反馈**：整卡 深红/红 →（点击）→ 琥珀（queued）→（抢槽）→ 青（running）。

## 3. 交互形态（方案 A：卡片常驻按钮）

- 失败/超时卡片**底部常驻轻量「↻ 重试」按钮**。
- **按钮配色（review 修正）**：用**中性描边**，`↻` 图标已表达"重跑"。**严禁**使用任何状态色 —— 避开 `#14B8A6`(running)、`#147af3`(进行中)、`#0d9488`(分组标题强调)，以守住"颜色=状态"不变量（[DesignGuide §4.8 固定语义色原则](../../DesignGuide.md:190)）。具体中性值取 DesignGuide 既有边框/次要文字色，实现时定。
- **点击行为**：立即 `run_state→queued`，按钮瞬间转禁用/"排队中…"。
- **任务详情面板**：完整版「↻ 重试」+「📄 查看失败日志」。日志**复用 [§7.4 详情页"对话/变更"视图](../../fp-ui/04-studio.md:673)** 滚到失败 Run 的最后输出，**不新造组件**。

## 4. 覆盖范围

全覆盖凡展示 `failed/timeout` 之处，**统一形态**（不在每种卡片各做一套）：

✅ 看板卡片 · 列表行 · 泳道卡片 · Task 详情面板 · Session级/项目级 Inspector 任务投影 · Home「今日聚焦」卡片

🔸 **时间轴甘特条：不加** —— 甘特以色条表排期，塞按钮太挤；点条开详情再重试。
🔸 **列表批量重试：V1 不做** —— YAGNI，列为 V2 可选增量。

## 5. 边界与约束（克制）

- 只对**配了执行 Agent 的可执行任务**有重试；手动卡片无 `run_state`、不出现重试按钮。
- **不改 status、不新增看板列、不新增状态枚举** —— 完全在已有两轴模型内。
- 整卡 tint 变色与重试按钮二者并存（变色表"出了什么事"，按钮给"怎么办"）。

## 7. 验收用例

1. 一个 `failed` 看板卡片显示「↻ 重试」（中性色）；点击 → 整卡转琥珀（queued）、按钮禁用；抢槽后转青（running）；全程 status="进行中"。
2. `timeout` 卡片同上。
3. 同一任务在列表行/泳道卡片/Inspector 投影/Home 卡片上重试入口形态一致。
4. 手动卡片（无执行 Agent）不出现重试按钮。
5. 时间轴甘特条**不**出现重试按钮；点甘特条 → 开详情 → 详情内可重试。
6. 详情面板可查看失败 Run 的最后输出（复用对话/变更视图）。
7. 重试按钮颜色 ≠ 任何 run_state / status / 分组状态色。
