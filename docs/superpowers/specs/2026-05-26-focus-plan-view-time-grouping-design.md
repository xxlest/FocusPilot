# Focus 规划视图分级时间分组

> **状态**：已批准
> **日期**：2026-05-26

---

## 目标

将 Focus 规划视图的时间分组粒度与筛选范围对齐：每个 scope 使用下一级更细的时间粒度进行分组，形成"全局→月→周→日→时"的递进结构。

## 现状问题

| Scope | 当前左侧分组 | 问题 |
|-------|-------------|------|
| 全局规划 | 季度→月 | 无问题 |
| 本月计划 | 季度→月（同全局） | 粒度太粗，看不到周维度分布 |
| 本周计划 | 扁平列表 | 无天维度分组，看不到每天安排 |
| 今日聚焦 | 时段分组 | 无问题 |

## 设计方案

### 分级时间粒度映射

| Scope | 左侧分组 | 右侧甘特列 | 子视图 ID |
|-------|---------|-----------|----------|
| 全局规划 | 季度→月（不变） | 月列（不变） | fp-year |
| **本月计划** | **按周分组**（W22, W23…） | **周列**（4~5列） | **fp-month（新建）** |
| **本周计划** | **按天分组**（周一~周日） | 天列（不变） | fp-week（改造左侧） |
| 今日聚焦 | 时段分组（不变） | 小时列（不变） | fp-day |

### 变更范围

#### 1. 新建 fp-month 子视图

- 左侧树：按 ISO 周号分组（W22、W23、W24、W25 等），每组下列任务
- 右侧甘特：列头为周号，每列代表一周，任务条横跨对应周
- 周号计算：基于任务的 `data-fp-schedule` 属性（today/week/month）和当前日历推算
  - today → 当天所在周
  - week → 本周
  - month → 按 `data-fp-plan-month` 或默认当月第一周
- 目标容器保持不变，仅内部任务按周重新分组

#### 2. 改造 fp-week 左侧分组

- 现状：扁平任务列表，无天维度
- 改造：按星期几分组（周一、周二、…、周日），每组下列任务
- 右侧甘特天列保持不变
- 分组依据：
  - `data-fp-schedule="today"` → 当天对应的星期几
  - `data-fp-schedule="week"` → 分散到各天（或归入"本周"通用组）
  - 无明确日期的任务 → 归入"待安排"组

#### 3. modeMap 更新

```javascript
// 现状
var modeMap = { today: 'fp-day', week: 'fp-week' };
// 改为
var modeMap = { today: 'fp-day', week: 'fp-week', month: 'fp-month' };
```

当前 month scope 未命中 modeMap 时 fallthrough 到 fp-year。加入映射后切换到 fp-month。

#### 4. 未关联目标

保持现状不变。未关联目标在各视图中以独立分组显示，不参与时间维度的重新分组。

### 不做的事

- 不修改全局规划（fp-year）的分组方式
- 不修改今日聚焦（fp-day）的分组方式
- 不为未关联目标添加时间分组
- 不修改筛选逻辑（日历月过滤已在前序提交中完成）

## 技术要点

1. **周号计算**：使用 ISO 8601 周号，`getWeekNumber(date)` 工具函数
2. **跨月周处理**：月末跨入下月的周仍然显示在当月视图中
3. **Demo 数据**：为现有 11 个任务补充 `data-fp-week` 属性标记所属周号
4. **甘特列生成**：fp-month 的甘特需动态生成当月包含的周列（通常 4~5 列）
5. **响应式**：周/天分组的折叠/展开行为与现有季度→月分组保持一致
