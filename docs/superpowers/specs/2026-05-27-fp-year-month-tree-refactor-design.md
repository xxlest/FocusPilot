# fp-year/fp-month 目标树层级重构设计

## 问题

全局规划（fp-year）目标树当前把季度和月份与单个目标名绑定（如 `Q2 — FocusPilot 0.0.1 发布`、`5月 — AI + Crew 集成`），但实际使用中一个季度/月份可能包含多个独立目标，1:1 绑定结构不可扩展。

## 方案

采用**时间优先 + 目标子分组**策略：

1. 删除季度层（Q2/Q3），月份直接挂在年度下
2. 月份变为纯时间容器，去掉目标名后缀
3. 每个月份内按目标子分组，目标作为二级分组头
4. 无任务的未来占位符（如 Q3）不显示

## 设计规则

1. **月份头样式**：与当前一致（📆 图标 + 月名 + status pill + progress bar + 百分比），仅去掉 `— 目标描述` 部分
2. **目标子分组头样式**：复用当前季度行的 📌 样式，缩进到月份下一级，显示目标名 + status + progress
3. **单目标月份**：即使只有一个目标，也显示目标子分组头，保持结构一致
4. **月份 progress**：聚合该月所有目标下任务的完成率
5. **未关联目标**：保持不变，仍在底部独立显示
6. **甘特对齐**：右侧甘特行数和左侧目标树行数一一对应

## 不改动

- fp-week（本周计划 · 天视图）：不受影响
- fp-day（今日聚焦 · 小时视图）：不受影响
- 看板和列表视图：不受影响
- JS 过滤逻辑（applyFocusFilter）：不受影响（过滤基于 data-fp-schedule/data-fp-month 属性，与树层级无关）
- 未关联目标区块：保持原样
