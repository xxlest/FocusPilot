## 7. 性能测试

| # | 指标 | 目标值 | 实测值 | 结果 |
|---|------|--------|--------|------|
| P01 | 空闲内存占用 | < 80MB | 94.8MB | ❌ 超标 |
| P02 | 空闲 CPU 占用 | < 1% | 0.0% | ✅ 通过 |
| P03 | 启动时间 | < 3s | ~2s | ✅ 通过 |
| P04 | 面板打开时 CPU | - | 3.7-6.9% | ⚠️ 偏高 |
| P05 | 面板打开时内存 | < 80MB | 94.6MB | ❌ 超标 |
| P06 | 面板弹出响应 | < 100ms | ~300ms (含 hover 延迟) | ✅ 通过 |
| P07 | App 切换响应 | < 200ms | < 500ms | ✅ 通过 |

### BUG-02: 内存占用超标

**严重程度**：P2
**描述**：空闲状态 RSS 内存约 94.8MB，超过 PRD 规定的 80MB 上限。
**可能原因**：
- SwiftUI 主看板窗口占用较多内存
- App 图标缓存
- AX API 对象未及时释放

### BUG-03: 面板打开时 CPU 偏高

**严重程度**：P2
**描述**：快捷面板打开时 CPU 占用 3.7-6.9%，面板关闭后降为 0%。
**原因**：窗口刷新定时器 (`windowRefreshTimer`) 持续调用 `refreshAllWindows()`，每次都重建 AX 标题映射。
**建议**：增加刷新间隔或使用差量更新。

## 10. 已修复问题

| # | 描述 | 修复方案 |
|---|------|---------|
| FIX-01 | 窗口标题全部显示"(无标题)" | `buildAXTitleMap` 改用 `AXIsProcessTrusted()` 直接检查，避免缓存值在 codesign 后过期 |
| FIX-02 | `activateWindow` 在 AX 权限变化后失效 | 同上，改用实时权限检查 |
| FIX-03 | 面板打开时未同步权限状态 | `startWindowRefresh()` 中添加 `PermissionManager.shared.checkAccessibility()` |
| FIX-04 | codesign 后 AX 权限丢失导致所有标题"(无标题)" | AppDelegate 启动时检测无权限 → `requestAccessibility()` 提示用户 + 启动轮询；PermissionManager 权限恢复时自动调用 `AppMonitor.shared.refreshRunningApps()` 刷新窗口列表 |

## 12. 结论

PinTop V1.1 核心功能稳定，窗口标题问题已修复：

- **悬浮球**：常驻显示、拖拽、贴边吸附、层级正确、视觉效果优化（毛玻璃+渐变+光晕）
- **快捷面板**：hover 弹出、App/窗口列表正确、切换功能正常、自动收起正常、钉住模式、拖拽 resize
- **窗口标题**：AX API + `_AXUIElementGetWindow` 精确映射，修复权限缓存问题后标题显示正确
- **窗口切换**：App 切换（Cursor↔豆包）和窗口级切换（点击具体窗口条目）均正常
- **主看板**：三个页签正常、配置持久化正确
- **系统集成**：菜单栏、权限检测正常

**待改进项**：
1. 内存占用超标（94.8MB vs 80MB 目标）
2. 面板打开时 CPU 偏高（窗口刷新定时器）
3. 全局快捷键的自动化测试受限于合成事件限制，建议补充手动测试
