## 1. 产品概述

### 1.1 产品定位

FocusByTime 是 FocusPilot 的**专注计时子模块**，通过循环计时机制（类似番茄钟）辅助用户训练注意力觉察能力。

### 1.2 核心理念

正如身体锻炼中的“核心”是力量传递与稳定的基础，精神层面同样有一个“注意力核心” —— 即对自身注意力流向的觉察与主动回收。当注意力游离或分散时，能及时觉察并有意识地拉回，是保持高效专注和完成任务的关键能力。

FocusByTime 通过科学设计的计时机制，帮助用户建立“专注-休息”交替循环，在每一个专注阶段持续锻炼和强化注意力的觉察及自我调节能力，使用户能够专注高效地完成一个个具体任务，实现目标的逐步达成。

### 1.3 与 FocusByTask 的关系

| 维度       | FocusByTime                                            | FocusByTask                       |
| ---------- | ------------------------------------------------------ | --------------------------------- |
| 回答的问题 | **怎么做**——工作节奏管理                               | **做什么**——任务调度与执行        |
| 核心能力   | 专注/休息 cycle 计时                                   | Obsidian 同步、IDE 派发、执行监控 |
| 任务数据   | 不维护独立列表，从 Obsidian Task 中选择关联            | Obsidian Vault 为唯一数据源       |
| 互补场景   | 用户从 Today 视图选 Task → 派发 IDE → 启动 60 分钟专注 | 并行推进，互不干扰                |

## 2. 功能需求

### 2.1 专注计时器（核心）

#### FR-FT01：专注阶段

**描述**：可自定义时长的倒计时，默认 60 分钟。

**详细需求**：

- 倒计时显示（分:秒格式）
- 时长可配置，默认 3600 秒（60 分钟）
- 运行中支持暂停 / 恢复
- 可随时手动停止（终止当前 cycle）

**验收标准**：

- 倒计时精度误差 < 1 秒
- 暂停后恢复不丢失已计时间
- 窗口最小化/不可见时计时仍准确运行

#### FR-FT02：休息阶段

**描述**：专注结束后进入休息倒计时，默认 5 分钟。

**详细需求**：

- 时长可配置，默认 300 秒（5 分钟）
- 休息阶段同样支持暂停 / 恢复 / 手动停止
- 休息阶段的 UI 指示与专注阶段明确区分（颜色/标签）

**验收标准**：

- 休息倒计时与专注倒计时行为一致
- 阶段切换时 UI 状态即时更新

#### FR-FT03：自动阶段切换

**描述**：专注阶段结束后自动进入休息阶段。

**详细需求**：

- 专注倒计时归零后，自动切换为休息阶段并开始倒计时
- 切换时播放提示音（如果声音已启用）
- 切换时更新所有窗口的状态显示（Bubble + Cockpit）

**验收标准**：

- 切换延迟 < 1 秒
- 提示音在阶段结束时准时播放

#### FR-FT04：声音提醒

**描述**：每个阶段结束时播放提示音。

**详细需求**：

- 内置默认提示音
- 声音开关可配置
- 自定义提示音文件（V2，暂不实现）

**验收标准**：

- 声音提醒即使所有窗口最小化也能触发
- 关闭声音后不播放任何提示

#### FR-FT05：循环模式

**描述**：休息结束后可自动开始下一轮专注或等待手动开始。

**详细需求**：

- 可配置两种模式：
  - **自动循环**：休息结束后自动开始下一轮专注，cycle 计数 +1
  - **手动开始**：休息结束后回到 Idle 状态，等待用户手动启动
- 当前轮次（cycle number）在 UI 中可见

**验收标准**：

- 自动循环模式下 cycle 正确递增
- 手动模式下休息结束后状态正确回到 Idle

### 2.2 任务关联

#### FR-FT06：关联 Obsidian Task

**描述**：启动专注时可选择关联一个 Obsidian Task，显示在计时器区域。

**详细需求**：

- 启动专注时从 Today 列表中选择一个 Task（可选，允许不关联）
- 关联 Task 的名称显示在 Cockpit 的 Timer 区域
- 不维护独立任务列表，任务来源统一为 Obsidian（通过 FocusByTask 的 Sync 模块）

**验收标准**：

- 可从 Today Task 列表中选择关联
- 不选择关联时 Timer 仍可正常运行

## 3. Timer 状态机

### 3.1 状态定义

| 状态    | 含义                   |
| ------- | ---------------------- |
| `idle`  | 空闲，未启动           |
| `focus` | 专注阶段，倒计时进行中 |
| `rest`  | 休息阶段，倒计时进行中 |

每个状态附加 `paused` 布尔值，表示是否暂停。

### 3.2 状态转移规则

```
Idle + start_focus()              → Focus
Focus + pause()                   → Focus (paused)
Focus (paused) + resume()         → Focus
Focus + timer_expired()           → Rest（自动切换）
Focus + stop()                    → Idle
Rest + pause()                    → Rest (paused)
Rest (paused) + resume()          → Rest
Rest + timer_expired() + auto     → Focus（下一轮，cycle+1）
Rest + timer_expired() + manual   → Idle
Rest + stop()                     → Idle
```

## 4. UI 集成

FocusByTime 不拥有独立窗口，其 UI 集成在 FocusPilot 的三层窗口中：

| 窗口                    | Timer 展示内容                                                              |
| ----------------------- | --------------------------------------------------------------------------- |
| **悬浮球（Bubble）**    | 倒计时数字 + 环形进度条 + 阶段颜色（Focus=绿 / Rest=蓝 / Idle=灰）          |
| **Cockpit**             | Timer 控制区：倒计时大字、阶段/轮次指示、开始/暂停/停止按钮、关联 Task 选择 |
| **主面板（MainPanel）** | Timer 配置页面：专注时长、休息时长、循环模式、声音开关                      |

## 5. 配置项

| 配置                  | 类型 | 默认值 | 说明                         |
| --------------------- | ---- | ------ | ---------------------------- |
| `focus_duration_secs` | u32  | 3600   | 专注阶段时长（秒）           |
| `rest_duration_secs`  | u32  | 300    | 休息阶段时长（秒）           |
| `auto_start_next`     | bool | true   | 休息结束后是否自动开始下一轮 |
| `sound_enabled`       | bool | true   | 阶段结束时是否播放提示音     |

配置持久化到 Tauri 应用数据目录的 JSON 文件。

## 6. 技术要点

### 6.1 计时精度

计时引擎运行在 Rust 后端（非前端 JavaScript），原因：

- 前端 `setInterval` 在窗口不可见时会被 Chromium 节流
- 三个窗口需要同步 Timer 状态，后端统一管理避免不一致
- 声音提醒即使窗口最小化也需触发

计时方式：记录阶段开始时间（`phase_started_at`），每次 tick 计算 `remaining = target_end - now`，不依赖 sleep 循环累积。

### 6.2 事件推送

| 事件                  | 频率                   | Payload                             |
| --------------------- | ---------------------- | ----------------------------------- |
| `timer:tick`          | 1次/秒（Timer 运行时） | `{ remaining_seconds, phase }`      |
| `timer:phase_changed` | 阶段切换时             | `{ phase, paused, cycle, task_id }` |
