## 1. 产品概述

### 1.1 产品定位

Focus Copilot（原 PinTop）是一款 macOS 应用，以**悬浮球**为常驻入口，提供**应用快速切换**与**窗口标记**能力。产品采用三层 UI 架构——悬浮球、快捷面板、主看板——信息密度递增，让用户在不离开当前工作区的前提下高效管理多个应用窗口。App 显示在 macOS Dock 栏中，点击 Dock 图标时打开/聚焦主看板。

### 1.2 目标用户

- 日常打开 10+ 应用的重度多任务用户
- 开发者（IDE + 终端 + 文档 + 浏览器频繁切换）
- 设计师（设计工具 + 参考图 + 标注工具并行）
- 交易员（多行情窗口同时监控）
- 内容创作者（素材 + 编辑器 + 预览多窗口协作）

### 1.3 核心痛点

| 痛点 | 现有方案的不足 |
|---|---|
| 打开 App 太多，切换效率低 | `⌘Tab` 在 15+ App 时几乎不可用 |
| 需要多个窗口同时可见 | macOS 原生不支持多窗口置顶 |
| 缺少常驻可见的应用切换入口 | Dock 在全屏时隐藏，Mission Control 打断工作流 |

### 1.4 竞品分析

| 产品 | 悬浮入口 | App 快速切换 | 多窗口标记 |
|---|---|---|---|
| macOS Dock | ❌ | ✅（但全屏时隐藏） | ❌ |
| `⌘Tab` | ❌ | ✅（App 多时低效） | ❌ |
| Stage Manager | ❌ | 分组切换 | ❌ |
| TopNotch / AfloatX | ❌ | ❌ | ✅（单窗口） |
| **Focus Copilot** | **✅** | **✅** | **✅** |

## 3. 功能需求

### 3.1 悬浮球（Floating Ball）

**描述**：常驻屏幕的球形入口，始终置顶于所有窗口之上，是整个产品的唯一常驻入口。

**外观与行为**：

| 属性 | 规格 |
|---|---|
| 默认尺寸 | 40×40px（可配置 30-60px） |
| 外观 | 品牌 Logo（橙色渐变底 + 图钉 + FC 文字），毛玻璃背景（`.popover`） |
| 层级 | `statusWindow + 100`（高于所有应用窗口和快捷面板） |
| 可见范围 | 所有桌面空间、全屏应用中均可见 |
| 拖拽 | 支持拖拽到屏幕任意位置 |
| 贴边半隐藏 | 拖到屏幕边缘时自动收入一半，鼠标靠近时滑出 |
| 隐藏/唤出 | 快捷面板底部或主看板侧边栏底部按钮隐藏，`⌘⇧B` 全局快捷键唤出/隐藏 |

**交互方式**：

| 操作 | 行为 |
|---|---|
| Hover 300ms | 弹出快捷面板（面板已钉住时不响应） |
| 单击 | 切换快捷面板钉住状态：未显示→弹出+自动钉住；已钉住→取消钉住+关闭 |
| 双击 | 打开主看板 |
| 拖拽 | 自由拖动，拖拽结束后吸附到最近屏幕边缘 |

**拖拽约束**：
- 限制在当前屏幕可见区域内
- 多显示器下仅存在于一个屏幕
- 拖拽结束后自动吸附到最近的屏幕边缘（上/下/左/右）
- 位置持久化，重启后恢复到上次位置
- 拖拽开始时自动关闭快捷面板

### 3.2 快捷面板（Quick Panel）

**描述**：hover 或单击悬浮球时弹出的紧凑布局面板，提供 App 快速切换和窗口标记快捷操作。支持钉住模式、拖拽调整大小、顶部拖拽移动。

**触发与收起**：

| 行为 | 规则 |
|---|---|
| Hover 弹出 | 鼠标悬停悬浮球 300ms 后弹出（面板已钉住时不触发） |
| 单击弹出 | 单击悬浮球弹出并自动钉住（面板已钉住时取消钉住并关闭） |
| 弹出动画 | 从悬浮球方向滑出，150ms ease-out |
| 收起触发 | 鼠标离开面板+悬浮球区域 500ms（钉住模式下不自动收起） |
| 收起动画 | 淡出，150ms ease-in |
| 弹出方向 | 自动判断屏幕空间，向充裕方向展开 |
| 拖动悬浮球 | 拖动悬浮球时自动关闭面板 |
| 钉住模式 | 顶部栏提供钉住按钮，钉住后面板不随鼠标离开而收起；钉住时面板和悬浮球联动拖拽 |
| 拖拽调整大小 | 面板右边缘/底边缘/右下角热区（5-10px）支持拖拽调整尺寸，尺寸持久化 |
| 顶部拖拽移动 | 顶部栏区域（24px）支持拖拽移动面板位置 |

**面板布局**：

```
┌────────────────────────────────────┐
│  📱 打开主界面           📌 钉住面板  │  ← 顶部栏（24px）
│────────────────────────────────────│
│                                    │
│  🟢 Safari                         │  ← 单窗口 App，点击直接切换
│                                    │
│  🟢 Cursor                ▼ 3 个窗口│  ← 多窗口 App，▼ 展开状态
│  ┌────────────────────────────────┐│
│  │ 📌 Cursor - main.swift      [×]││  ← 已标记（红色图钉）
│  │ ⚪ Cursor - PRD.md              ││  ← 未标记（灰色图钉）
│  │ ⚪ Cursor - Untitled-1          ││
│  └────────────────────────────────┘│
│                                    │
│  ⚪ Terminal                        │  ← 未运行，灰度
│                                    │
│  🟢 Chrome                ▶ 2 个窗口│  ← 多窗口 App，▶ 折叠状态
│                                    │
│  🟢 Slack                          │  ← 单窗口 App
│                                    │
│────────────────────────────────────│
│  👁 显示/隐藏     │     ⏻ 退出      │  ← 底部按钮栏（36px）
└────────────────────────────────────┘
```

**面板内容——App 列表**：

| 元素 | 说明 |
|---|---|
| App 图标 | 从系统获取，20×20px |
| App 名称 | 显示完整名称 |
| 运行状态 | 🟢 = 已运行，⚪ = 未运行（灰度） |
| 窗口数量 | 多窗口时右侧显示"N 个窗口" |
| 折叠指示器 | 多窗口 App 显示 ▼（展开）或 ▶（折叠） |
| hover 高亮 | 鼠标悬浮时行背景高亮 |

**App 行点击行为**：

| 场景 | 行为 |
|---|---|
| 多窗口 App | 点击整行切换折叠/展开状态 |
| 单窗口 App（运行中） | 点击整行激活该 App 的窗口 |
| 未运行 App | 不响应，tooltip 提示"未运行" |

**窗口行交互**：

| 元素 | 说明 |
|---|---|
| 标记按钮 | 窗口行前的图钉图标，已标记=红色旋转45°、未标记=灰色；**点击整行**即可切换标记状态 |
| 窗口标题 | 格式为"App名 - 窗口标题"，通过 AX API 获取，支持用户重命名 |
| 取消标记按钮 | 已标记窗口右侧显示 ✕ 按钮，点击取消标记 |
| hover 高亮 | 鼠标悬浮时背景高亮 |
| 点击行为 | 点击整行激活该窗口并将其提到最前，同时收起面板（钉住模式下不收起） |
| 右键菜单 | 支持右键窗口条目弹出菜单，提供"重命名窗口"选项 |

**窗口列表分区**：

| 区域 | 说明 |
|---|---|
| 置顶区 | 窗口标题命中用户配置的关键词，带 ★ 标识，排在上方 |
| 分割线 | 视觉分隔置顶区和普通区（仅在有关键词命中时显示） |
| 普通区 | 未命中关键词的窗口，按系统窗口层级排列 |

**无关键词配置时**：不显示分割线，所有窗口按系统顺序排列。

**多窗口折叠/展开**：
- 多窗口 App **默认展开**
- 点击 App 行切换折叠/展开
- 折叠时隐藏窗口列表，仅显示 App 行
- 折叠状态跟随面板生命周期（面板关闭后重置为展开）

**面板尺寸**：

| 属性 | 规格 |
|---|---|
| 默认宽度 | 280px |
| 宽度范围 | 220-500px |
| 默认高度 | 400px |
| 最大高度 | 屏幕可见区域的 60% |
| 最小高度 | 200px |
| 行高 | App 行 26px，窗口行 22px |
| 窗口行缩进 | 28px |
| 圆角 | 12px |
| 窗口层级 | `statusWindow + 50`（低于悬浮球） |
| 背景 | 毛玻璃效果（`.menu`） |

**底部按钮栏**：
- **左半**：悬浮球显隐切换按钮（eye / eye.slash 图标）
- **右半**：退出按钮（power 图标，红色），带确认对话框

**关键规则**：
- 最多展示 **8 个 App**
- 展示的是用户在主看板中配置选中的 App
- **快捷面板只做切换，不做启动**
- 未在主看板配置的 App 不会出现在快捷面板
- 每个 App 最多显示 **10 个窗口**，超出支持滚动
- 面板内容过长时整体可滚动

**实时更新规则**：

| 场景 | 处理方案 |
|---|---|
| App 从单窗口变为多窗口 | 面板内该 App 下方自动展开窗口列表 |
| App 从多窗口变为单窗口 | 窗口列表自动收起，变为单行 |
| 窗口被关闭 | 实时从列表移除 |
| App 所有窗口关闭 | 该 App 变为灰色（未运行） |
| 新窗口打开 | 实时添加到对应 App 的列表中 |
| 窗口标题变化 | 实时更新显示，重新执行关键词匹配 |

**关键词匹配规则**：

| 规则 | 说明 |
|---|---|
| 匹配方式 | `String.contains`，大小写不敏感 |
| 匹配范围 | 窗口标题（优先使用用户自定义名称） |
| 置顶区内排序 | 按关键词在配置中的顺序排列 |
| 普通区内排序 | 按系统窗口层级排列 |

#### 3.2.1 窗口重命名（Window Rename）

**描述**：用户可为任意窗口设置自定义显示名称，方便在窗口标题不够直观时快速识别。

**交互方式**：
- 在快捷面板的窗口条目上右键，选择"重命名窗口"
- 弹出 NSAlert + NSTextField 对话框，预填当前窗口标题
- 用户输入自定义名称后确认，面板中即显示自定义名称
- 清空名称并确认则恢复为原始窗口标题

**数据模型**：

```swift
// 窗口重命名持久化
// Key 格式: "{bundleID}::{windowTitle}"
// Value: 用户自定义名称（String）
// 存储位置: UserDefaults
```

**规则**：

| 规则 | 说明 |
|---|---|
| 显示优先级 | 自定义名称 > AX API 获取的原始标题 |
| 持久化 | 存储到 UserDefaults |
| 匹配逻辑 | 关键词匹配同时检查自定义名称和原始标题 |
| 清除 | 清空自定义名称后恢复显示原始标题 |

| 窗口位置 | 居中显示 |

**侧边栏底部按钮**：
- **左半**：悬浮球显隐切换（eye/eye.slash 图标 + "显示"/"隐藏"文字）

#### 3.3.1 快捷面板配置

**功能**：管理快捷面板中展示哪些 App，以及它们的显示顺序和窗口排序关键词。

**界面结构**：

```
快捷面板配置
─────────────────────────────────────

已选应用（拖拽排序，最多 8 个）                          [5/8]
┌──────────────────────────────────────────────────────────┐
│  ≡ Safari          [⚙]                                   │
│  ≡ Cursor          [⚙] ▾³  ← 多窗口 App，可配置关键词     │
│  ≡ Terminal        [⚙]                                   │
│  ≡ Slack           [⚙]                                   │
│  ≡ Figma           [⚙]                                   │
└──────────────────────────────────────────────────────────┘

可选应用
┌──────────────────────────────────────────────────────────┐
│  [已激活 App ▼]  [已安装 App]                             │
│                                                          │
│  ☐ Google Chrome          运行中 🟢                      │
│  ☐ Apple Notes            运行中 🟢                      │
│  ☐ WeChat                 运行中 🟢                      │
│                                                          │
│  🔍 搜索应用...                                           │
└──────────────────────────────────────────────────────────┘
```

**规则**：

| 规则 | 说明 |
|---|---|
| 数据来源——已激活 App | `NSWorkspace.shared.runningApplications`，过滤后台进程 |
| 数据来源——已安装 App | 扫描 `/Applications` + `~/Applications` 目录 |
| 添加操作 | 在可选区勾选 → 自动添加到已选区 |
| 移除操作 | 在已选区点击"✕"移除 |
| 排序 | 已选区支持拖拽排序，顺序即快捷面板展示顺序 |
| 上限 | 最多 8 个，达到上限时可选区复选框禁用 |
| 搜索 | 支持模糊搜索 App 名称 |
| 窗口关键词配置 | 每个 App 可配置多个排序关键词，命中标题的窗口在置顶区 |
| 实时预览 | 配置关键词时，下方实时展示当前窗口的匹配结果 |
| 持久化 | 存储 App 的 Bundle ID + 顺序 + 关键词列表 |

**数据模型**：

```swift
struct AppConfig: Codable, Identifiable, Equatable {
    let bundleID: String       // App 唯一标识
    var displayName: String    // 自定义显示名称
    var order: Int             // 在快捷面板中的排序
    var pinnedKeywords: [String] // 窗口排序关键词
}
```

### 3.5 辅助功能权限

Focus Copilot 依赖 macOS 辅助功能权限来获取窗口标题和操作其他应用的窗口。

**权限检测**：
- 启动后实时检测辅助功能权限
- 未授权时显示引导页面，提供"打开系统设置"按钮
- 后台持续轮询权限状态（应对 codesign 导致的权限失效）
- 授权后自动进入正常工作模式

**权限影响**：
- **不需要权限**：App 切换（`NSRunningApplication.activate()`）、快捷面板 App 列表
- **需要权限**：窗口标题获取（AX API）、窗口激活前置（AXRaise）

### 3.6 菜单栏图标

| 状态 | 图标 |
|---|---|
| 无标记窗口 | `pin.fill`（模板模式，跟随系统颜色） |
| 有标记窗口 | `pin.fill`（红色） |

点击菜单栏图标可打开主看板。

### 3.7 Dock 图标

蓝色渐变底 + 白色图钉 + FC 文字。点击 Dock 图标打开/聚焦主看板。

## 4. 边界场景处理

### 4.1 快捷面板相关

| 场景 | 处理方案 |
|---|---|
| 配置的 App 未运行 | 灰度显示，不可点击，tooltip 提示"未运行" |
| 已激活 App 退出 | 实时更新为灰色 |
| 新应用启动 | 实时更新运行状态 |
| 配置已满 8 个 | 主看板提示上限，需先移除再添加 |
| 未配置任何 App | 快捷面板显示空状态引导 |
| 面板内容过长 | 最大高度为屏幕 60%，超出后整体可滚动 |
| 窗口标题变化 | 实时重新匹配关键词，更新分区 |

### 4.2 悬浮球相关

| 场景 | 处理方案 |
|---|---|
| 拖到屏幕边缘 | 自动贴边半隐藏，鼠标靠近时滑出 |
| 全屏应用中 | 悬浮球可见（`fullScreenAuxiliary`），快捷面板正常弹出 |
| 多显示器 | 悬浮球仅存在于一个屏幕，位置持久化 |
| 悬浮球被隐藏时 | `⌘⇧B` 唤出 |

### 4.3 窗口激活相关

| 场景 | 处理方案 |
|---|---|
| 跨 App 切换 | 先 activate App，然后 AXRaise 窗口，300ms 后兜底重试 |
| 窗口被关闭 | 自动清理标记状态 |
| 辅助功能未授权 | 标记功能和窗口标题获取不可用，App 切换正常 |

| 构建工具 | Xcode 15+ / swift build |
