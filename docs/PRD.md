## 1. 产品概述

### 1.1 产品定位

Focus Copilot（原 PinTop）是一款 macOS 应用，以**悬浮球**为常驻入口，提供**应用快速切换**、**窗口前置**和 **AI 编码工具会话管理**能力。产品采用三层 UI 架构——悬浮球、快捷面板、主看板——信息密度递增，让用户在不离开当前工作区的前提下高效管理多个应用窗口和 AI 编码会话。App 显示在 macOS Dock 栏中，点击 Dock 图标时打开/聚焦主看板。

### 1.2 目标用户

- 日常打开 10+ 应用的重度多任务用户
- 开发者（IDE + 终端 + 文档 + 浏览器频繁切换 + AI 编码工具并行）
- 设计师（设计工具 + 参考图 + 标注工具并行）
- 交易员（多行情窗口同时监控）
- 内容创作者（素材 + 编辑器 + 预览多窗口协作）

### 1.3 核心痛点

| 痛点 | 现有方案的不足 |
|---|---|
| 打开 App 太多，切换效率低 | `⌘Tab` 在 15+ App 时几乎不可用 |
| 需要多个窗口同时可见 | macOS 原生不支持多窗口置顶 |
| 缺少常驻可见的应用切换入口 | Dock 在全屏时隐藏，Mission Control 打断工作流 |
| AI 编码工具多实例难以管理 | 多个 Claude Code/Codex 会话分散在不同终端窗口，切换困难 |

### 1.4 竞品分析

| 产品 | 悬浮入口 | App 快速切换 | 窗口快速前置 | AI 会话管理 |
|---|---|---|---|---|
| macOS Dock | — | 有（全屏时隐藏） | — | — |
| `⌘Tab` | — | 有（App 多时低效） | — | — |
| Stage Manager | — | 分组切换 | — | — |
| TopNotch / AfloatX | — | — | 有（单窗口） | — |
| **Focus Copilot** | **有** | **有** | **有** | **有** |

## 3. 功能需求

### 3.1 悬浮球（Floating Ball）

**描述**：常驻屏幕的球形入口，始终置顶于所有窗口之上，是整个产品的唯一常驻入口。外观与设计哲学详见 [DesignGuide.md](DesignGuide.md)。

**外观与行为**：

| 属性 | 规格 |
|---|---|
| 默认尺寸 | 35x35px（可配置 30-60px） |
| 外观 | 品牌 Logo（禅圆 Enso），毛玻璃背景，钉住时外圈红色发光边框环 |
| 层级 | `statusWindow + 100`（高于所有应用窗口和快捷面板） |
| 可见范围 | 所有桌面空间、全屏应用中均可见 |
| 拖拽 | 支持拖拽到屏幕任意位置 |
| 贴边半隐藏 | 拖到屏幕边缘时自动收入一半，鼠标靠近时滑出 |
| 隐藏/唤出 | 主看板侧边栏底部按钮隐藏，`⌘⇧B` 全局快捷键显示/隐藏（可自定义） |

**交互方式**：

| 操作 | 行为 |
|---|---|
| Hover 150ms | 弹出快捷面板（面板已钉住时不响应） |
| 单击 | 切换快捷面板钉住状态：未显示→弹出+自动钉住；已钉住→取消钉住+关闭 |
| 双击 | 打开主看板 |
| 右键 | 弹出上下文菜单，提供"退出 Focus Copilot"选项（点击后弹出确认对话框） |
| 拖拽 | 自由拖动，拖拽结束后吸附到最近屏幕边缘 |

**AI 角标**：

在**非固定模式**（面板未钉住）下，悬浮球右上角显示 AI 可操作会话数角标（`actionableCount`），数值为处于活跃状态且需要用户处理的 AI 编码会话数。当无可操作会话时角标隐藏。固定模式下角标隐藏，避免与面板内 AI Tab 内容重复。

**拖拽约束**：
- 限制在当前屏幕可见区域内
- 多显示器下仅存在于一个屏幕
- 拖拽结束后自动吸附到最近的屏幕边缘（上/下/左/右）
- 位置持久化，重启后恢复到上次位置
- 拖拽开始时自动关闭快捷面板

### 3.2 快捷面板（Quick Panel）

**描述**：hover 或单击悬浮球时弹出的紧凑布局面板，提供 App 快速切换、窗口前置和 AI 会话管理。支持"活跃"、"关注"和"AI"三个 Tab 页面。支持钉住模式、拖拽调整大小、顶部拖拽移动。

**触发与收起**：

| 行为 | 规则 |
|---|---|
| Hover 弹出 | 鼠标悬停悬浮球 150ms 后弹出（面板已钉住时不触发）；鼠标进入时立即预热窗口数据 |
| 单击弹出 | 单击悬浮球弹出并自动钉住（面板已钉住时取消钉住并关闭） |
| 弹出动画 | 从悬浮球方向滑出，250ms ease-out |
| 收起触发 | 鼠标离开面板+悬浮球区域 500ms（钉住模式下不自动收起） |
| 收起动画 | 淡出，120ms ease-in |
| 弹出方向 | 自动判断屏幕空间，向充裕方向展开 |
| 拖动悬浮球 | 拖动悬浮球时自动关闭面板 |
| 钉住模式 | 单击悬浮球切换钉住状态（悬浮球显示红色发光边框），钉住后面板不随鼠标离开而收起；钉住时面板和悬浮球联动拖拽 |
| 自动回缩设置 | 偏好设置中可配置 hover 离开后是否自动收起面板（默认开启），钉住状态下不受此设置影响 |
| 拖拽调整大小 | 面板右边缘/底边缘/右下角热区支持拖拽调整尺寸，尺寸持久化 |
| 顶部拖拽移动 | 顶部栏区域（32px）支持拖拽移动面板位置 |

#### 3.2.1 Tab 页面切换

快捷面板顶部栏设有**活跃**、**关注**和 **AI** 三个 Tab：

| Tab | 数据来源 | 说明 |
|---|---|---|
| **活跃** | AppMonitor 实时监测 | 自动显示所有有窗口的运行中 App（不依赖 ConfigStore），默认 Tab。**关注的 App 排在列表前面** |
| **关注** | ConfigStore.appConfigs | 展示用户关注的 App 及其窗口，关注 App 未运行时灰度显示。**支持拖拽排序** |
| **AI** | CoderBridgeService.sessions | 展示通过 coder-bridge 注册的 AI 编码工具会话（详见 §3.6） |

**Tab 记忆**：快捷面板关闭再打开时保持上次选中的 Tab，不重置为默认 Tab。

#### 3.2.1a 固定/非固定模式下的交互差异

快捷面板分为**固定模式**（单击悬浮球钉住）和**非固定模式**（hover 弹出、未钉住），两种模式下的交互行为有所不同：

**Tab 切换**：

| 模式 | Tab 切换方式 | 持久化 |
|---|---|---|
| **固定模式** | 点击 Tab 按钮切换，立即生效并写入 UserDefaults | 是（selectedTab 持久化） |
| **非固定模式** | 鼠标 hover Tab 按钮时临时预览（displayTab 临时切换），点击后 selectedTab 才更新 | 仅点击后持久化；离开悬浮后 displayTab 回退到 selectedTab |

**App 展开/折叠**：

| 模式 | 展开/折叠方式 | 默认状态 |
|---|---|---|
| **固定模式** | 点击 App 行切换折叠/展开，跟随面板生命周期 | 多窗口 App 默认展开 |
| **非固定模式** | 鼠标 hover App 行时自动展开，离开时折叠 | 默认折叠（hover 时展开） |

**面板关闭或进入固定模式时**：displayTab 回退到 selectedTab，hover 展开状态清空。

#### 3.2.2 App 行

**App 行元素**：

| 元素 | 说明 |
|---|---|
| App 图标 | 从系统获取，20x20px |
| 关注星号 | 仅"活跃"Tab 显示：已关注=填充星号（金色），未关注=空心星号，点击切换关注状态。**位于行最左侧** |
| App 名称 | 显示完整名称 |
| 运行状态 | 绿点 = 已运行，灰点 = 未运行（灰度显示） |
| 窗口数量 | 多窗口时右侧显示"N 个窗口" |
| 折叠指示器 | 多窗口 App 显示 ▼（展开）或 ▶（折叠） |
| hover 高亮 | 鼠标悬浮时行背景高亮 |

**App 行点击行为**：

| 场景 | 行为 |
|---|---|
| 已运行 + 多窗口 | 点击整行切换折叠/展开状态 |
| 已运行 + 单窗口 | 点击整行激活该 App 的唯一窗口并前置 |
| 未运行（仅关注 Tab） | 点击整行**启动该 App** |

**App 行右键菜单**：

| Tab | 菜单项 | 说明 |
|---|---|---|
| 活跃 | 关闭应用 | 终止该 App 进程，从活跃列表移除 |
| 关注 | 关闭应用 | 仅运行中的 App 显示此选项 |

#### 3.2.3 窗口行

**窗口行元素**：

| 元素 | 说明 |
|---|---|
| 窗口标题 | 通过 AX API 获取，支持用户重命名 |
| 关闭按钮 | 窗口行右侧 ✕ 按钮，hover 时显示，点击弹出确认对话框后关闭 |
| hover 高亮 | 鼠标悬浮时行背景高亮 |
| 选中高亮 | 点击后整行高亮显示 |

**窗口行点击行为**：

| 操作 | 行为 |
|---|---|
| 点击窗口行 | 整行高亮 + 该窗口切换到最前方 |
| 点击另一条窗口行 | 前一行取消高亮 → 新行高亮 + 新窗口前置 |

**核心规则**：同一时间只有一个窗口行高亮，高亮状态不持久化，面板关闭后重置。

#### 3.2.4 窗口重命名

- 右键窗口行 → 选择"重命名窗口" → NSAlert + NSTextField 对话框
- 自定义名称优先于 AX API 原始标题显示
- 持久化到 UserDefaults（key: `{bundleID}::{CGWindowID}`），绑定窗口实例
- 清空名称恢复原始标题

#### 3.2.5 面板尺寸

面板尺寸规格详见 [DesignGuide.md §7.3](DesignGuide.md)。

**实时更新规则**：

| 场景 | 处理方案 |
|---|---|
| App 从单窗口变为多窗口 | 自动展开窗口列表 |
| App 从多窗口变为单窗口 | 窗口列表自动收起 |
| 窗口被关闭 | 实时从列表移除 |
| App 所有窗口关闭 | "活跃"Tab 移除；"关注"Tab 变为灰色 |
| 新窗口打开 | 实时添加到对应列表 |
| 窗口标题变化 | 实时更新 |

**描述**：双击悬浮球或点击 Dock 图标进入的完整管理界面。

**触发方式**：双击悬浮球 / 点击 Dock 图标 / 快捷面板顶部"主看板"按钮（切换模式）

**窗口规格**：

| 属性 | 规格 |
|---|---|
| 窗口尺寸 | 840x620px（可调整，最小 640x480） |
| 窗口位置 | 居中显示 |
| 窗口层级 | 普通窗口层级（不置顶） |
| 关闭行为 | 仅隐藏窗口，App 继续后台运行 |

**布局结构**：侧边栏（180px）+ 内容区，侧边栏底部双按钮（左：悬浮球显隐切换，右：退出+确认对话框）。

#### 3.3.1 关注管理

三 Tab 过滤：

| Tab | 数据来源 | 说明 |

#### 3.3.2 偏好设置

### 3.5 FocusByTime 番茄钟

**描述**：内置在快捷面板底部的番茄钟计时器，帮助用户进行专注工作和科学休息的循环。悬浮球同步显示进度环。

#### 3.5.1 计时器栏（QuickPanel 底部 46px，整栏可点击）

**设计理念**：栏内零按钮，整栏即交互入口，所有操作收进弹窗。

**idle 状态**：左右双入口，左半边 `▶ 开始专注`（accent 色），右半边 `☕ 休息`（绿色），中间竖线分隔。

**运行状态**：居中阶段标签 + 大号倒计时 `MM:SS` + 进度条，底色变化（工作=accent@12%，休息=green@12%）。

**暂停状态**：居中"已暂停" + 冻结时间，底色回到中性色。

**pending 状态**：居中 pill 徽章 + 提示文案，点击重新弹出阶段转换弹窗。

#### 3.5.2 编辑弹窗

点击计时器栏（idle 状态）弹出，包含：
- 工作/休息时长输入框 + ±1 分钟按钮
- 三个推荐方案 radio：深度专注(25/5)、常规节奏(35/7)、轻度脑力(50/10)
- 按钮：直接开始 / 仅保存 / 取消
- 点击推荐方案自动填充，手动修改后自动取消 radio 选中
- 失焦自动关闭

#### 3.5.3 阶段转换弹窗

| 时机 | 内容 | 失焦行为 |
|---|---|---|
| 工作完成 | radio 选择引导休息强度（轻度/标准/深度）+ 自由休息 + 直接结束 | 自动关闭，pending 保留 |
| 休息结束 | 开始工作 / 稍后再说 | 自动关闭，pending 保留 |

#### 3.5.4 悬浮球进度环

工作阶段 accent 色、休息阶段绿色弧形进度环，idle 无。

#### 3.5.5 状态机

```
idle ──(开始专注)──▶ running(work) ──(归零)──▶ pending(startRest)
                     │                         │── 开始休息 ──▶ running(rest)
                     │                         │── 直接结束 ──▶ idle
                     │◀──(继续)── paused(work)
idle ──(独立休息)──▶ running(rest, standalone) ──(归零)──▶ idle
running(rest) ──(归零)──▶ pending(startWork) ──▶ running(work) 或 idle
```

完整状态转换矩阵详见 [Architecture.md](Architecture.md)。

#### 3.5.6 引导休息模式

工作完成后可选择引导休息，三级强度对应三维恢复（眼/脑/肌肉）：

| 强度 | 总时长 | 对应工作预设 | 肌肉环节 |
|---|---|---|---|
| ⭐ 轻度 | 5 min | 深度专注(25+5) | 全程坐着 |
| ⭐⭐ 标准 | 7 min | 常规节奏(35+7) | 站起来活动 |
| ⭐⭐⭐ 深度 | 10 min | 轻度脑力(50+10) | 走→站→墙→走→坐 |

引导休息按步骤自动推进，每步独立倒计时：

- 轻度 4 步：👁 远眺 → 🧠 深呼吸 → 💪 坐姿核心 → 💧 补水
- 标准 5 步：👁 闭眼远眺 → 🧠 深呼吸 → 🚶 起身走动 → 💪 站立核心 → 💧 补水
- 深度 5 步：👁 闭眼冥想 → 🧠 腹式深呼吸 → 🚶 走动放松 → 💪 全链路核心 → 💧 补水

**计时器栏轮播**：引导期间显示当前步骤 emoji + 步骤名 + 步骤倒计时 + 总剩余时间。

**独立休息模式**：通过 idle 状态右半边"☕ 休息"入口触发，休息结束后直接回 idle，不弹"充电完毕"对话框。

### 3.6 AI Tab（Coder-Bridge 会话管理）

**描述**：快捷面板第三个 Tab "AI"，展示通过 coder-bridge 注册的 AI 编码工具会话（Claude Code / Codex / Gemini CLI），支持会话状态感知、窗口快速切换和生命周期管理。

#### 3.6.1 会话列表展示

**目录分组结构**：

```
┌─ FocusPilot                         (2)  ← 按工作目录分组
│   ● Claude · a1b2c3d4  Cursor  执行中     ← Session 行第一行
│     "帮我修复编译错误..."                   ← 第二行：query 摘要
│   ● Claude · e5f6g7h8  iTerm2  等待输入
│     "分析窗口绑定逻辑"
├─ MyBackend                          (1)
│   ● Claude · 91ab44ef  Terminal  已完成
│     "添加用户认证接口"
```

**Session 行双行结构**：
- 第一行：状态圆点 + 工具名 + · + shortID（前8位）+ 宿主 App 名 + 状态文字
- 第二行：最近 query 摘要（三级文字色），无 query 时显示"等待输入..."
- 缩进：相对目录组缩进 28px

**同名目录消歧**：basename 相同时补父级路径片段（如 `app (frontend)` / `app (backend)`）。

**空状态**：无会话时居中显示"还没有 AI 编码会话\n启动一个 AI 编码工具后\n会自动显示在这里"。

#### 3.6.2 会话状态

**双维度状态模型**：

| status | 含义 | 圆点颜色 | 有光晕 |
|---|---|---|---|
| registered | 已连接 | 灰 | — |
| working | 执行中 | 绿 | 有 |
| idle | 等待输入 | 橙 | 有 |
| done | 已完成 | 绿 | 有 |
| error | 出错 | 红 | 有 |

| lifecycle | 含义 | 行透明度 |
|---|---|---|
| active | 活跃 | 1.0 |
| ended | 已结束 | 0.4 |

**Actionable 判定**（计入角标和高亮）：
- `done` + `active` 且未读 → actionable
- `idle`/`error` + `active` 且未读且未忽略 → actionable
- 其他状态 → 非 actionable

**isDismissed 忽略提醒**：
- idle/error 状态下右键"忽略提醒"可标记为已忽略
- 已忽略的会话：降灰显示（0.6 透明度）、圆点无光晕、不计入角标
- 状态变化时（新的 idle/error）自动重置忽略标记

#### 3.6.3 会话生命周期

```
session.start → status=registered, lifecycle=active, 自动采样前台窗口
session.update → status 更新（working/idle/done/error），重置 isRead/isDismissed
session.end → lifecycle=ended，状态保留显示
```

**乱序防护**：每个事件携带 seq 单调递增，`seq ≤ lastSeq` 则丢弃。

**会话不持久化**：纯运行时数据，FocusPilot 重启后清空。

**AI 会话偏好持久化**：按 `tool+cwdNormalized+hostApp` 索引存储 displayName，新 session 自动继承。

#### 3.6.4 窗口绑定与切换

**两层绑定模型**：

| 层级 | 绑定方式 | 强度 | 说明 |
|---|---|---|---|
| autoWindowID | session.start 时自动采样前台窗口 | 弱绑定 | 不参与占用仲裁 |
| manualWindowID | 用户手动确认 | 强绑定 | 排他性（Terminal）或共享（IDE） |

**点击 Session 行的窗口切换优先级**：
1. manualWindowID 有效 → 直接激活
2. autoWindowID 有效且无冲突 → 激活
3. 都无 → 弹绑定引导对话框

**绑定引导**：用户切换到目标宿主窗口 → 点击确认 → 写入 manualWindowID。

#### 3.6.5 HostKind 策略分化

coder-bridge 上报 `hostKind`（ide/terminal），作为绑定策略标签：

| 维度 | IDE（Cursor/VSCode） | Terminal（Terminal/iTerm2/Warp） |
|---|---|---|
| 绑定策略 | **共享**：多 session 共享同一窗口 | **独占**：一窗口一 session |
| 冲突检测 | 跳过（共享正常） | 检测并标记冲突 |
| UI 弱绑定标记 | 不显示 `?`（共享正常） | 显示 `?` |

**两条绑定入口差异**：

| 入口 | 点击行（隐式） | 右键"绑定到当前窗口"（显式） |
|---|---|---|
| Terminal 冲突 | 拦截（不允许替换） | 确认后替换 |
| IDE 冲突 | 跳过冲突检测 | 跳过冲突检测 |

**多绑定白名单**：偏好设置中可配置哪些 App 允许多 session 共享窗口（默认 Cursor、VSCode）。

#### 3.6.6 右键菜单

| 菜单项 | 条件 | 说明 |
|---|---|---|
| 绑定到当前窗口 | 始终可用 | 手动强绑定 |
| 解除绑定 | 有 manualWindowID | 清除手动绑定 |
| 忽略提醒 | idle/error + active + 未忽略 | isDismissed=true |
| 复制 Session ID | 始终可用 | — |
| 移除此会话 | — | 从列表移除 |
| 移除所有已结束会话 | 有 ended session | 批量清理 |

#### 3.6.7 会话清理

| 条件 | 动作 |
|---|---|
| ended + 超 2 分钟 | 自动移除 |
| registered + 超 30 秒无后续事件 | 自动移除（僵尸 session） |
| FocusPilot 重启 | 清空所有（等重新注册） |
| 用户右键 | 手动移除 |

#### 3.6.8 排序规则

**组间**：置顶组优先，其余按注册顺序。
**组内**：置顶 session 优先，其余按注册顺序。

## 4. 边界场景处理

### 4.1 快捷面板

| 场景 | 处理 |
|---|---|
| 关注 App 未运行 | 灰度显示，点击启动 |
| 已激活 App 退出 | 活跃 Tab 移除；关注 Tab 变灰 |
| 面板内容过长 | 最大高度屏幕 60%，超出可滚动 |
| 面板关闭后重新打开 | 高亮重置，Tab 保持记忆 |

### 4.2 悬浮球

| 场景 | 处理 |
|---|---|
| 拖到屏幕边缘 | 自动贴边半隐藏 |
| 全屏应用中 | 悬浮球可见，快捷面板正常弹出 |
| 多显示器 | 悬浮球仅存在于一个屏幕 |

### 4.3 窗口前置

| 场景 | 处理 |
|---|---|
| 辅助功能未授权 | 窗口标题和前置不可用，App 切换正常 |
| 连续快速点击 | 每次都执行前置，保证最后点击的在最前 |

### 4.4 AI Tab

| 场景 | 处理 |
|---|---|
| 窗口绑定失效（窗口被关闭） | 检测失效 → 清空 → 降级到下一层绑定 → 都失效则弹引导 |
| 同名多窗口（如两个 Cursor 开同一项目） | autoWindowID 可能绑错；恢复：手动右键绑定 |
| IDE 多 session 共享 | 正常行为，都绑同一窗口，点击激活同一窗口 |
| Terminal 冲突 | 显示红 ✕，引导用户手动绑定 |
| 旧版 coder-bridge（无 hostKind） | 默认 terminal，行为与升级前一致 |
