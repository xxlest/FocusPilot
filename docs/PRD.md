## 1. 产品概述

### 1.1 产品定位

Focus Copilot（原 PinTop）是一款 macOS 应用，以**悬浮球**为常驻入口，提供**应用快速切换**与**窗口前置**能力。产品采用三层 UI 架构——悬浮球、快捷面板、主看板——信息密度递增，让用户在不离开当前工作区的前提下高效管理多个应用窗口。App 显示在 macOS Dock 栏中，点击 Dock 图标时打开/聚焦主看板。

### 1.2 目标用户

- 日常打开 10+ 应用的重度多任务用户
- 开发者（IDE + 终端 + 文档 + 浏览器频繁切换）
- 设计师（设计工具 + 参考图 + 标注工具并行）
- 交易员（多行情窗口同时监控）
- 内容创作者（素材 + 编辑器 + 预览多窗口协作）

### 1.3 核心痛点

| 痛点 | 现有方案的不足 |
|---|---|
| 打开 App 太多，切换效率低 | `⌘Tab` 在 15+ App 时几乎不可用 |
| 需要多个窗口同时可见 | macOS 原生不支持多窗口置顶 |
| 缺少常驻可见的应用切换入口 | Dock 在全屏时隐藏，Mission Control 打断工作流 |

### 1.4 竞品分析

| 产品 | 悬浮入口 | App 快速切换 | 窗口快速前置 |
|---|---|---|---|
| macOS Dock | ❌ | ✅（但全屏时隐藏） | ❌ |
| `⌘Tab` | ❌ | ✅（App 多时低效） | ❌ |
| Stage Manager | ❌ | 分组切换 | ❌ |
| TopNotch / AfloatX | ❌ | ❌ | ✅（单窗口） |
| **Focus Copilot** | **✅** | **✅** | **✅** |

## 3. 功能需求

### 3.1 悬浮球（Floating Ball）

**描述**：常驻屏幕的球形入口，始终置顶于所有窗口之上，是整个产品的唯一常驻入口。

**外观与行为**：

| 属性 | 规格 |
|---|---|
| 默认尺寸 | 40×40px（可配置 30-60px） |
| 外观 | 品牌 Logo（橙色渐变底 + 图钉 + FC 文字），毛玻璃背景（`.popover`） |
| 层级 | `statusWindow + 100`（高于所有应用窗口和快捷面板） |
| 可见范围 | 所有桌面空间、全屏应用中均可见 |
| 拖拽 | 支持拖拽到屏幕任意位置 |
| 贴边半隐藏 | 拖到屏幕边缘时自动收入一半，鼠标靠近时滑出 |
| 隐藏/唤出 | 主看板侧边栏底部按钮隐藏，`⌘⇧B` 全局快捷键唤出/隐藏 |

**交互方式**：

| 操作 | 行为 |
|---|---|
| Hover 300ms | 弹出快捷面板（面板已钉住时不响应） |
| 单击 | 切换快捷面板钉住状态：未显示→弹出+自动钉住；已钉住→取消钉住+关闭 |
| 双击 | 打开主看板 |
| 右键 | 弹出上下文菜单，提供"退出 Focus Copilot"选项（点击后弹出确认对话框） |
| 拖拽 | 自由拖动，拖拽结束后吸附到最近屏幕边缘 |

**拖拽约束**：
- 限制在当前屏幕可见区域内
- 多显示器下仅存在于一个屏幕
- 拖拽结束后自动吸附到最近的屏幕边缘（上/下/左/右）
- 位置持久化，重启后恢复到上次位置
- 拖拽开始时自动关闭快捷面板

### 3.2 快捷面板（Quick Panel）

**描述**：hover 或单击悬浮球时弹出的紧凑布局面板，提供 App 快速切换和窗口前置操作。支持"已打开"和"收藏"两个 Tab 页面。支持钉住模式、拖拽调整大小、顶部拖拽移动。

**触发与收起**：

| 行为 | 规则 |
|---|---|
| Hover 弹出 | 鼠标悬停悬浮球 300ms 后弹出（面板已钉住时不触发） |
| 单击弹出 | 单击悬浮球弹出并自动钉住（面板已钉住时取消钉住并关闭） |
| 弹出动画 | 从悬浮球方向滑出，150ms ease-out |
| 收起触发 | 鼠标离开面板+悬浮球区域 500ms（钉住模式下不自动收起） |
| 收起动画 | 淡出，150ms ease-in |
| 弹出方向 | 自动判断屏幕空间，向充裕方向展开 |
| 拖动悬浮球 | 拖动悬浮球时自动关闭面板 |
| 钉住模式 | 顶部栏提供钉住按钮，钉住后面板不随鼠标离开而收起；钉住时面板和悬浮球联动拖拽 |
| 拖拽调整大小 | 面板右边缘/底边缘/右下角热区（5-10px）支持拖拽调整尺寸，尺寸持久化 |
| 顶部拖拽移动 | 顶部栏区域（24px）支持拖拽移动面板位置 |

**面板布局**：

```
┌──────────────────────────────────────────┐
│  📱  [已打开] [收藏]   🏠 主看板  📌 钉住  │  ← 顶部栏（24px），含 Tab 切换
│──────────────────────────────────────────│
│                                          │
│  🟢 Safari                               │  ← 单窗口 App，点击激活窗口
│     ┌──────────────────────────────────┐ │
│     │ Safari - Apple.com            ✕  │ │  ← 窗口行（点击高亮+前置，✕关闭）
│     └──────────────────────────────────┘ │
│                                          │
│  🟢 Cursor                  ▼ 3 个窗口    │  ← 多窗口 App（展开状态）
│     ┌──────────────────────────────────┐ │
│     │ Cursor - main.swift           ✕  │ │  ← 窗口行，点击=高亮+前置
│     │ Cursor - PRD.md               ✕  │ │
│     │ Cursor - Untitled-1           ✕  │ │
│     └──────────────────────────────────┘ │
│                                          │
│  🟢 Chrome                  ▶ 2 个窗口    │  ← 多窗口 App（折叠状态）
│                                          │
└──────────────────────────────────────────┘
```

#### 3.2.1 Tab 页面切换

快捷面板顶部栏设有**已打开**和**收藏**两个 Tab：

| Tab | 数据来源 | 说明 |
|---|---|---|
| **已打开** | AppMonitor 实时监测 | 自动显示所有有窗口的运行中 App（不依赖 ConfigStore），默认 Tab |
| **收藏** | ConfigStore.appConfigs | 展示用户收藏的 App 及其窗口，收藏 App 未运行时灰度显示 |

两个 Tab 页面的 App 和窗口行为完全一致（见下方交互规则）。

**Tab 记忆**：快捷面板关闭再打开时保持上次选中的 Tab，不重置为默认 Tab。

#### 3.2.2 App 行

**App 行元素**：

| 元素 | 说明 |
|---|---|
| App 图标 | 从系统获取，20×20px |
| App 名称 | 显示完整名称 |
| 运行状态 | 🟢 = 已运行，⚪ = 未运行（灰度显示） |
| 窗口数量 | 多窗口时右侧显示"N 个窗口" |
| 折叠指示器 | 多窗口 App 显示 ▼（展开）或 ▶（折叠） |
| hover 高亮 | 鼠标悬浮时行背景高亮 |

**App 行点击行为**：

| 场景 | 行为 |
|---|---|
| 已运行 + 多窗口 | 点击整行切换折叠/展开状态 |
| 已运行 + 单窗口 | 点击整行激活该 App 的唯一窗口并前置 |
| 未运行（仅收藏 Tab） | 点击整行**启动该 App**，启动后自动创建新窗口 |

**窗口数量规则**：
- "已打开"Tab 仅显示有窗口的运行中 App（至少 1 个窗口），不显示未运行 App
- "收藏"Tab 中未运行的 App 灰度显示，点击后启动并自动创建窗口

**多窗口折叠/展开**：
- 多窗口 App **默认展开**
- 点击 App 行切换折叠/展开
- 折叠时隐藏窗口列表，仅显示 App 行
- 折叠状态跟随面板生命周期（面板关闭后重置为展开）

#### 3.2.3 窗口行

**窗口行元素**：

| 元素 | 说明 |
|---|---|
| 窗口标题 | 通过 AX API 获取对应窗口标题，支持用户重命名 |
| 关闭按钮 | 窗口行右侧 ✕ 按钮，hover 时显示，点击弹出确认对话框后关闭该窗口 |
| hover 高亮 | 鼠标悬浮时行背景高亮 |
| 选中高亮 | 点击后整行高亮显示（区别于 hover 高亮） |
| 右键菜单 | 支持右键弹出菜单，提供"重命名窗口"选项 |

**窗口行点击行为**（适用于"已打开"和"收藏"两个 Tab）：

| 操作 | 行为 |
|---|---|
| 点击窗口行 | **整行高亮** + 该窗口切换到最前方（不被任何元素遮挡） |
| 点击另一条窗口行 | 前一行取消高亮 → 新行高亮 + 新窗口切换到最前方 |

**核心规则**：
- 无图钉/标记按钮，无取消标记按钮
- 点击整行即为激活，整行高亮表示该窗口已被前置
- 同一时间只有一个窗口行处于高亮状态
- 高亮状态不持久化，面板关闭后重置
- 窗口激活使用 `NSWorkspace.openApplication`（系统级 API）+ `AXRaise` + `AXMain` + `AXFocused` 三重设置，确保窗口不被遮挡

**窗口列表**：
- 无分区（不区分置顶区/普通区）
- 无分割线
- 所有窗口按系统窗口层级排列

#### 3.2.4 窗口重命名

**描述**：用户可为任意窗口设置自定义显示名称，方便在窗口标题不够直观时快速识别。

**交互方式**：
- 在快捷面板的窗口条目上右键，选择"重命名窗口"
- 弹出 NSAlert + NSTextField 对话框，预填当前窗口标题
- 用户输入自定义名称后确认，面板中即显示自定义名称
- 清空名称并确认则恢复为原始窗口标题

**规则**：

| 规则 | 说明 |
|---|---|
| 显示优先级 | 自定义名称 > AX API 获取的原始标题 |
| 持久化 | 存储到 UserDefaults（key: `{bundleID}::{originalTitle}`） |
| 清除 | 清空自定义名称后恢复显示原始标题 |

#### 3.2.5 面板尺寸

| 属性 | 规格 |
|---|---|
| 默认宽度 | 280px |
| 宽度范围 | 220-500px |
| 默认高度 | 400px |
| 最大高度 | 屏幕可见区域的 60% |
| 最小高度 | 200px |
| 行高 | App 行 26px，窗口行 22px |
| 窗口行缩进 | 28px |
| 圆角 | 12px |
| 窗口层级 | `statusWindow + 50`（低于悬浮球） |
| 背景 | 毛玻璃效果（`.menu`） |

**关键规则**：
- "已打开"Tab 自动显示所有有窗口的运行中 App，数量无上限
- "收藏"Tab 显示收藏列表中的 App，数量无上限
- 每个 App 最多显示 **10 个窗口**，超出支持滚动
- 面板内容过长时整体可滚动

**实时更新规则**：

| 场景 | 处理方案 |
|---|---|
| App 从单窗口变为多窗口 | 面板内该 App 下方自动展开窗口列表 |
| App 从多窗口变为单窗口 | 窗口列表自动收起，变为单行 |
| 窗口被关闭 | 实时从列表移除 |
| App 所有窗口关闭 | "已打开"Tab 中移除该 App；"收藏"Tab 中变为灰色（未运行） |
| 新窗口打开 | 实时添加到对应 App 的列表中 |
| 窗口标题变化 | 实时更新显示 |
| 点击未运行 App 启动后 | 实时刷新为运行状态，展示新创建的窗口 |

**窗口规格**：

| 属性 | 规格 |
|---|---|
| 窗口尺寸 | 800×600px（可调整大小，最小 640×480） |
| 窗口位置 | 居中显示 |
| 窗口层级 | 普通窗口层级（不置顶） |
| 关闭行为 | 关闭主看板窗口 = 仅隐藏窗口（`orderOut`），App 继续后台运行 |

**布局结构**：

```

├──────────────┬───────────────────────────────────────────┤

│              │                                           │

   左侧导航栏         右侧内容区
   (180px 固定)
```

**侧边栏底部按钮**：
- **左半**：悬浮球显隐切换（eye/eye.slash 图标 + "悬浮球 显示"/"悬浮球 隐藏"文字），点击区域覆盖整个按钮面积
- **右半**：退出按钮（power 图标 + "退出"文字，红色），点击后弹出确认对话框（"退出 Focus Copilot"，取消/退出），点击区域覆盖整个按钮面积

#### 3.3.1 收藏管理

**功能**：管理收藏 App 列表。收藏管理集中在主看板，快捷面板不提供收藏操作。在 `ConfigStore.appConfigs` 中的 App 即为收藏。

**界面结构**：

```
收藏管理
─────────────────────────────────────
[全部(152)]  [已打开(12)]  [收藏(3)]        ← 三个 Tab，显示数量

🔍 搜索应用...

┌──────────────────────────────────────────────────────────┐
│  Safari              运行中 🟢           ★（已收藏）      │
│  Cursor              运行中 🟢           ☆（未收藏）      │
│  Terminal            运行中 🟢           ★（已收藏）      │
│  Slack               未运行              ☆（未收藏）      │
│  Figma               运行中 🟢           ★（已收藏）      │
│  ...                                                     │
└──────────────────────────────────────────────────────────┘
 ★/☆ = 点击切换收藏状态
```

**三个 Tab**：

| Tab | 数据来源 | 说明 |
|---|---|---|
| **全部** | 系统已安装 App 列表 | 扫描 `/Applications` + `~/Applications`，显示所有 App，Tab 标签显示总数如"全部(152)" |
| **已打开** | `NSWorkspace.shared.runningApplications` | 仅显示运行中的 regular App，Tab 标签显示数量如"已打开(12)" |
| **收藏** | `ConfigStore.appConfigs` | 仅显示已收藏的 App，Tab 标签显示数量如"收藏(3)" |

**规则**：

| 规则 | 说明 |
|---|---|
| 收藏操作 | 点击 App 行的 ★/☆ 按钮切换收藏状态 |
| 添加收藏 | 点击 ☆ 将 App 加入 `appConfigs`（即收藏） |
| 移除收藏 | 点击 ★ 将 App 从 `appConfigs` 移除（即取消收藏） |
| 搜索 | 支持模糊搜索 App 名称 |
| 运行状态 | 实时显示 App 运行状态（🟢 运行中 / 灰色未运行） |
| 持久化 | `appConfigs` 存储收藏 App 的 Bundle ID + 顺序 |

**数据模型**：

```swift
struct AppConfig: Codable, Identifiable, Equatable {
    let bundleID: String       // App 唯一标识
    var displayName: String    // 自定义显示名称
    var order: Int             // 在收藏列表中的排序
}
```

### 3.5 辅助功能权限

Focus Copilot 依赖 macOS 辅助功能权限来获取窗口标题和操作其他应用的窗口。

**权限检测**：
- 启动后实时检测辅助功能权限
- 未授权时显示引导页面，提供"打开系统设置"按钮
- 后台持续轮询权限状态（应对 codesign 导致的权限失效）
- 授权后自动进入正常工作模式

**权限影响**：
- **不需要权限**：App 切换（`NSRunningApplication.activate()`）、快捷面板 App 列表、启动 App
- **需要权限**：窗口标题获取（AX API）、窗口激活前置（AXRaise）

### 3.6 菜单栏图标

`pin.fill`（模板模式，跟随系统颜色）。点击菜单栏图标可打开主看板。

### 3.7 Dock 图标

蓝色渐变底 + 白色图钉 + FC 文字。点击 Dock 图标打开/聚焦主看板。

## 4. 边界场景处理

### 4.1 快捷面板相关

| 场景 | 处理方案 |
|---|---|
| 收藏的 App 未运行 | "收藏"Tab 中灰度显示，点击可启动该 App |
| 已激活 App 退出 | "已打开"Tab 实时移除；"收藏"Tab 变为灰色 |
| 新应用启动 | "已打开"Tab 实时添加；"收藏"Tab 实时更新运行状态 |
| 无收藏 App | "收藏"Tab 显示空状态引导 |
| 无运行中 App | "已打开"Tab 显示空状态引导 |
| 面板内容过长 | 最大高度为屏幕 60%，超出后整体可滚动 |
| 窗口标题变化 | 实时更新显示 |
| 点击已高亮的窗口行 | 保持高亮，不做额外操作 |
| 面板关闭后重新打开 | 所有窗口行高亮状态重置，Tab 保持上次选中状态 |
| 启动 App 失败 | 显示提示"无法启动该应用" |
| App 启动中 | 显示加载状态，启动完成后刷新窗口列表 |
| 点击窗口关闭按钮 | 弹出确认对话框，确认后关闭该窗口 |

### 4.2 悬浮球相关

| 场景 | 处理方案 |
|---|---|
| 拖到屏幕边缘 | 自动贴边半隐藏，鼠标靠近时滑出 |
| 全屏应用中 | 悬浮球可见（`fullScreenAuxiliary`），快捷面板正常弹出 |
| 多显示器 | 悬浮球仅存在于一个屏幕，位置持久化 |
| 悬浮球被隐藏时 | `⌘⇧B` 唤出 |

### 4.3 窗口前置相关

| 场景 | 处理方案 |
|---|---|
| 跨 App 切换 | 使用 NSWorkspace.openApplication 激活 App，然后 AXRaise + AXMain + AXFocused 三重设置窗口前置 |
| 窗口被关闭 | 从列表移除，若该行高亮则取消高亮 |
| 辅助功能未授权 | 窗口标题获取和前置不可用，App 切换正常 |
| 连续快速点击不同窗口行 | 每次点击都执行前置操作，保证最后点击的窗口在最前 |
