> **版本**：V1.0（第四轮——快捷面板改为一次性全展开）
> **状态**：评审通过
> **日期**：2026-02-27
> **作者**：PM

## 1. 产品概述

### 1.1 产品定位

PinTop 是一款 macOS 应用，以**悬浮球**为常驻入口，提供**应用快速切换**与**多窗口置顶平铺**能力。产品采用三层 UI 架构——悬浮球、快捷看板、主看板——信息密度递增，让用户在不离开当前工作区的前提下高效管理多个应用窗口。

### 1.2 目标用户

- 日常打开 10+ 应用的重度多任务用户
- 开发者（IDE + 终端 + 文档 + 浏览器频繁切换）
- 设计师（设计工具 + 参考图 + 标注工具并行）
- 交易员（多行情窗口同时监控）
- 内容创作者（素材 + 编辑器 + 预览多窗口协作）

### 1.3 核心痛点

| 痛点 | 现有方案的不足 |
|---|---|
| 打开 App 太多，切换效率低 | `⌘Tab` 在 15+ App 时几乎不可用 |
| 需要多个窗口同时可见 | macOS 原生不支持多窗口置顶 |
| 窗口布局需要反复手动调整 | Stage Manager 是分组而非置顶+平铺 |
| 缺少常驻可见的应用切换入口 | Dock 在全屏时隐藏，Mission Control 打断工作流 |

### 1.4 竞品分析

| 产品 | 悬浮入口 | App 快速切换 | 多窗口置顶 | 平铺布局 |
|---|---|---|---|---|
| macOS Dock | ❌ | ✅（但全屏时隐藏） | ❌ | ❌ |
| `⌘Tab` | ❌ | ✅（App 多时低效） | ❌ | ❌ |
| Stage Manager | ❌ | 分组切换 | ❌ | 分组 |
| Rectangle / Magnet | ❌ | ❌ | ❌ | ✅ |
| TopNotch / AfloatX | ❌ | ❌ | ✅（单窗口） | ❌ |
| **PinTop** | **✅** | **✅** | **✅** | **✅** |

## 3. 功能需求

### 3.1 悬浮球（Floating Ball）

**描述**：常驻屏幕的球形入口，始终置顶于所有窗口之上（包括已 Pin 的窗口），是整个产品的唯一入口。

**外观与行为**：

| 属性 | 规格 |
|---|---|
| 默认尺寸 | 40×40px |
| 外观 | 半透明毛玻璃质感，图钉 icon，深色/浅色模式自适应 |
| 层级 | `NSWindow.Level.statusBar`（高于所有应用窗口） |
| 可见范围 | 所有桌面空间、全屏应用中均可见 |
| 拖拽 | 支持拖拽到屏幕任意位置 |
| 贴边半隐藏 | 拖到屏幕边缘时自动收入一半，鼠标靠近时滑出 |
| 角标 | 有 Pin 窗口时在右上角显示 Pin 数量（小红点+数字） |
| 隐藏/唤出 | 双击隐藏，`⌘⇧B` 全局快捷键唤出 |

**拖拽约束**：
- 限制在当前屏幕可见区域内
- 多显示器下仅存在于一个屏幕
- 拖拽结束后自动吸附到最近的屏幕边缘（上/下/左/右）
- 位置持久化，重启后恢复到上次位置

### 3.2 快捷看板（Quick Panel）

**描述**：鼠标悬浮在悬浮球上时弹出的横向长条面板，提供 App 快速切换和置顶快捷操作。

**触发与收起**：

| 行为 | 规则 |
|---|---|
| 弹出触发 | 鼠标 hover 悬浮球 300ms |
| 弹出动画 | 从悬浮球方向滑出，150ms ease-out |
| 收起触发 | 鼠标离开面板+悬浮球区域 500ms |
| 收起动画 | 滑入收回，150ms ease-in |
| 弹出方向 | 自动判断屏幕空间，向充裕方向展开 |
| 安全区域 | 悬浮球与面板之间有连接区域，防止鼠标移动过程中误触收起 |

**面板内容——正常模式**：

快捷面板采用**一次性全展开**策略：hover 悬浮球时，一级 App 列表和所有多窗口 App 的二级窗口列表**同时弹出**，无需逐级点击，减少操作步数。

面板整体为纵向布局，每个 App 占一行（单窗口）或一组（多窗口展开）：

```
  ●  悬浮球
  │
  ▼  快捷面板（一次性全展开）
┌───────────────────────────────────────┐
│                                       │
│  🟢 Safari                            │  ← 单窗口，点击直接切换
│                                       │
│  🟢 Cursor                    3 个窗口 │  ← 多窗口 App 标题
│  ┌─────────────────────────────────┐  │
│  │ ★ main.swift — PinTop          │  │  ← 置顶区
│  │ ★ PRD.md — FocusPilot          │  │
│  │ ─────────────────────────────── │  │  ← 分割线
│  │   index.ts — SpecTrader        │  │  ← 普通区
│  │   Untitled-1                   │  │
│  └─────────────────────────────────┘  │
│                                       │
│  ⚪ Terminal                           │  ← 未运行，灰度
│                                       │
│  🟢 Slack                             │  ← 单窗口
│                                       │
│  🟢 Chrome                    2 个窗口 │  ← 多窗口 App
│  ┌─────────────────────────────────┐  │
│  │ ★ GitHub — PR #42              │  │
│  │ ─────────────────────────────── │  │
│  │   Google Search                │  │
│  └─────────────────────────────────┘  │
│                                       │
│  ⚪ Figma                             │  ← 未运行，灰度
│                                       │
├───────────────────────────────────────┤
│  📌 置顶模式  │  ▦ 平铺              │  ← 底部操作栏
└───────────────────────────────────────┘
```

**设计要点**：
- 面板从横向长条改为**纵向列表**，信息承载量更大
- 多窗口 App 的窗口列表**内嵌展开**，和一级列表同时呈现
- 单窗口 App 只显示一行，点击直接切换
- 未运行的 App 灰度显示在对应位置
- 底部固定操作栏：📌 置顶模式入口 + ▦ 平铺快捷按钮

**每个 App 项**：

| 元素 | 说明 |
|---|---|
| App 图标 | 从系统获取，20×20px（列表模式下缩小） |
| App 名称 | 显示完整名称 |
| 运行状态 | 🟢 = 已激活，⚪ = 未运行 |
| 窗口数量 | 多窗口时右侧显示"N 个窗口" |
| 点击行为 | 已激活且单窗口 → 直接切换；多窗口 → 点击窗口列表中的具体条目切换；未运行 → 无响应（tooltip 提示"未运行"） |
| hover 高亮 | 鼠标悬浮时行背景高亮 |

**关键规则**：
- 最多展示 **8 个 App**
- 展示的是用户在主看板中配置选中的 App（不是所有运行中的 App）
- **快捷面板只做切换，不做启动**
- 未在主看板配置的 App 不会出现在快捷面板
- 多窗口 App 的窗口列表随面板**一次性展开**，无需二次点击

#### 3.2.1 多窗口内嵌列表（Window Inline List）

**描述**：多窗口 App 的所有窗口在快捷面板中**内嵌展开**，和一级 App 列表同时呈现，用户 hover 悬浮球后即可一眼看到所有可切换的窗口，直接点击目标窗口完成切换。

**展开规则**：面板弹出时自动展开，无需额外触发。

**窗口列表分区**：

| 区域 | 说明 |
|---|---|
| 置顶区 | 窗口标题命中用户配置的关键词，带 ★ 标识，排在上方 |
| 分割线 | 视觉分隔置顶区和普通区（仅在有关键词命中时显示） |
| 普通区 | 未命中关键词的窗口，按系统窗口层级排列 |

**无关键词配置时**：不显示分割线，所有窗口按系统窗口顺序排列。

**每个窗口条目**：

| 元素 | 说明 |
|---|---|
| ★ 标识 | 仅置顶区窗口显示 |
| 窗口标题 | 取自 `kCGWindowName`，过长则截断并 tooltip 显示完整标题 |
| 缩进 | 相对 App 标题行缩进，视觉上归属于该 App |
| hover 高亮 | 鼠标悬浮时背景高亮 |
| 点击行为 | 激活该窗口并将其提到最前，同时收起整个快捷面板 |

**面板尺寸控制**：

| 规则 | 说明 |
|---|---|
| 面板最大高度 | 屏幕高度的 60%，超出后整体可滚动 |
| 每个 App 窗口列表上限 | 10 个窗口，超出支持滚动 |
| 折叠机制 | 当面板内容过长时，未运行的 App 自动折叠为仅图标+名称一行 |

**实时更新规则**：

| 场景 | 处理方案 |
|---|---|
| App 从单窗口变为多窗口 | 面板内该 App 下方自动展开窗口列表 |
| App 从多窗口变为单窗口 | 窗口列表自动收起，变为单行 |
| 窗口被关闭 | 实时从列表移除 |
| App 所有窗口关闭 | 该 App 变为灰色（未运行） |
| 新窗口打开 | 实时添加到对应 App 的列表中（命中关键词进置顶区，否则进普通区） |
| 窗口标题变化 | 实时更新显示，重新执行关键词匹配 |

**关键词匹配规则**：

| 规则 | 说明 |
|---|---|
| 匹配方式 | `String.contains`，大小写不敏感 |
| 匹配范围 | 窗口标题（`kCGWindowName`） |
| 多关键词命中 | 窗口命中多个关键词时，按第一个命中的关键词顺序排列 |
| 置顶区内排序 | 按关键词在配置中的顺序排列 |
| 普通区内排序 | 按系统窗口层级排列 |

### 3.5 辅助功能权限引导

PinTop 依赖 macOS 辅助功能权限来控制其他应用的窗口。

**引导流程**：

```
首次启动
  → 检测辅助功能权限
    → 已授权 → 进入主界面
    → 未授权 → 显示引导页面
      → 说明需要权限的原因（窗口置顶功能需要）
      → 提供"打开系统设置"按钮
      → 轮询检测授权状态，授权后自动进入主界面
```

**注意**：快捷面板的 App 切换功能不依赖辅助功能权限（使用 `NSRunningApplication.activate()`）。仅置顶功能需要。未授权时，置顶相关功能灰度禁用，其他功能正常可用。

| 3 | 点击悬浮球，进入主看板，配置你的快捷面板 |

支持"跳过"。完成后展示快捷键速查卡。

## 4. 边界场景处理

### 4.1 快捷面板相关

| 场景 | 处理方案 |
|---|---|
| 配置的 App 被卸载 | 主看板中标灰 + 提示"未找到"，不自动移除 |
| 配置的 App 未运行 | 快捷面板中灰度显示，不可点击，tooltip 提示"未运行" |
| 已激活 App 退出 | 快捷面板实时更新为灰色 |
| 新应用启动 | 快捷面板实时更新运行状态为绿色 |
| 配置已满 8 个 | 主看板提示上限，需先移除再添加 |
| 未配置任何 App | 快捷面板显示空状态引导："点击悬浮球配置应用" |
| App 从单窗口变为多窗口 | 面板内该 App 下方自动展开窗口列表 |
| App 从多窗口变为单窗口 | 窗口列表自动收起，变为单行 |
| 面板展开时窗口被关闭 | 实时从列表移除 |
| 面板展开时新窗口打开 | 实时添加到列表（命中关键词进置顶区，否则进普通区） |
| 关键词未命中任何窗口 | 置顶区为空，不显示分割线，所有窗口在普通区 |
| 未配置关键词 | 不显示分割线，所有窗口按系统顺序排列 |
| 窗口标题变化 | 实时重新匹配关键词，更新分区 |
| 面板内容过长 | 面板最大高度为屏幕 60%，超出后整体可滚动 |

### 4.2 悬浮球相关

| 场景 | 处理方案 |
|---|---|
| 拖到屏幕边缘 | 自动贴边半隐藏，鼠标靠近时滑出 |
| 全屏应用中 | 悬浮球可见（`fullScreenAuxiliary`），快捷面板正常弹出 |
| 多显示器 | 悬浮球仅存在于一个屏幕，位置持久化 |
| 主看板已打开时点击悬浮球 | 聚焦到已打开的主看板窗口 |
| 悬浮球被隐藏时 | `⌘⇧B` 唤出 |

### 4.3 置顶相关

| 场景 | 处理方案 |
|---|---|
| Pin 窗口被关闭 | 自动移除 Pin 状态，释放名额 |
| Pin 窗口被最小化 | 自动 Unpin |
| Pin 窗口进入全屏 | 自动 Unpin |
| 系统 Split View | Pin 状态暂停，退出后恢复 |
| Stage Manager | Pin 窗口浮于 Stage Manager 之上 |
| 目标应用崩溃 | 自动清理 Pin 状态 |
| 达到 6 个上限 | 提示先 Unpin 已有窗口 |
| PinTop 退出 | 所有 Pin 状态清除（V1 不做持久化恢复） |

## 6. 技术方案概要

### 6.1 技术栈

| 层面 | 选型 |

| 分发渠道 | DMG 独立分发（主）+ App Store（探索） |

### 6.2 架构概要

```
┌──────────────────────────────────────────────────────────┐
│                       PinTop App                         │
├────────────┬──────────────┬──────────────┬───────────────┤
│ Floating   │ Quick Panel  │ Main Kanban  │ Hotkey        │
│ Ball       │ Module       │ Module       │ Module        │
│ Module     │              │              │               │
├────────────┴──────────────┴──────────────┴───────────────┤
│                    Core Services                         │
├──────────────┬────────────────┬────────────┬─────────────┤
│ App          │ Pin            │ Tile       │ Config      │
│ Monitor      │ Manager        │ Engine     │ Store       │
├──────────────┴────────────────┴────────────┴─────────────┤
│                  Window Service                          │
│       (AXUIElement + CGWindowList + CGSPrivate)          │
├──────────────────────────────────────────────────────────┤
│                 macOS System APIs                        │
└──────────────────────────────────────────────────────────┘
```

**核心模块职责**：

| 模块 | 职责 |
|---|---|
| Floating Ball Module | 悬浮球 UI、拖拽、贴边、hover 检测 |
| Quick Panel Module | 快捷看板 UI、App 列表渲染、切换操作、置顶模式 |
| Main Kanban Module | 主看板 UI、App 配置、置顶管理、偏好设置 |
| Hotkey Module | 全局快捷键注册和分发 |
| App Monitor | 监听系统 App 启动/退出，维护运行中 App 列表 |
| Pin Manager | 窗口置顶状态管理、层级控制、生命周期监听 |
| Tile Engine | 布局计算、窗口位置/大小调整、动画控制 |
| Config Store | 用户配置持久化（已选 App、排序、偏好设置） |
