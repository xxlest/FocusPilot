## 1. 定位

Inbox 是信息的**统一收集入口**，负责管理待阅读内容、灵感、临时任务和便签。所有未经整理的信息先进 Inbox，整理后流向 Focus（成为 Task）或 AreaProjects（沉淀为知识）。

### 与其他页面的职责边界

| 维度 | Inbox | Focus | AreaProjects |
|------|-------|-------|-------------|
| 定位 | 收集、暂存、分类 | 规划、执行、验收 | 沉淀、积累、检索 |
| 数据生命周期 | 临时态，待整理 | 行动态，有排期 | 归档态，持久保存 |
| 流转方向 | -> Focus 或 -> AreaProjects | -- | -- |

**原则**：Inbox 是缓冲区，不是第二个 AreaProjects。信息在此短暂停留，整理后流出。

## 4. 核心交互

| 用户任务 | 操作路径 |
|---------|---------|
| 快速记录一条信息 | [+ 新建] -> 录入弹窗 -> 选择类型 -> 填写内容 -> 保存 |
| 粘贴 URL 自动解析 | 录入弹窗粘贴 URL -> 自动识别来源 -> 提取标题和摘要 |
| 将待阅读内容转为任务 | 选中条目 -> 「移入 Focus」-> 成为 Task（source=inbox） |
| 将灵感沉淀为知识 | 选中条目 -> 「移入 AreaProjects」-> 选择目标项目 |
| 查看自动采集结果 | 侧边栏点击"自动采集" -> 审批/拒绝/移入 |

---

## 5. 状态与规则

### 录入弹窗

- 必填：类型（待阅读 / 灵感 / 临时任务）
- 可选：标题、URL、标签、备注
- URL 粘贴后自动解析：支持 B站、公众号、YouTube、知乎等
- 解析结果填充标题和摘要，用户可修改

### 自动采集

- 需在 Settings 中配置 Agent 和信息源平台
- Agent 自动采集的内容标记为"待审批"
- 用户审批通过后进入对应分类

### 流转规则

- Inbox -> Focus：成为 Task（source=inbox），进入待排期
- Inbox -> AreaProjects：成为文件，归入目标项目文件夹

|------|------|
| Inbox | 页面名称（活动栏 📥） |
| 待阅读 | 收藏的文章/视频/链接，待后续阅读 |
| 灵感 | 临时想法、创意片段 |
| 自动采集 | Agent 根据配置自动抓取的信息源内容 |
