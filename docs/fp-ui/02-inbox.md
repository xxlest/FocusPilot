## 1. 定位

Inbox 是信息的**统一收集入口**，负责管理待阅读内容、灵感和临时事项。所有未经整理的信息先进 Inbox，整理后流向 Focus（成为 WorkItem）或 AreaProjects（沉淀为素材）。

### 与其他页面的职责边界

| 维度 | Inbox | Focus | AreaProjects |
|------|-------|-------|-------------|
| 定位 | 收集、暂存、分类 | 规划、执行、验收 | 沉淀、积累、检索 |
| 数据生命周期 | 临时态，待整理 | 行动态，有排期 | 归档态，持久保存 |
| 流转方向 | → Focus 或 → AreaProjects | — | — |

**原则**：Inbox 是缓冲区，不是第二个 AreaProjects。信息在此短暂停留，整理后流出。

## 2. 侧边栏

按信息类型分类导航，各分类显示未处理数量（`status=new` 的条目数）。

```
┌─ 侧边栏 ──────────────────┐
│  🔍 搜索收集内容...        │
│                            │
│  分类                      │
│  ┌──────────────────────┐ │
│  │▌📖 待阅读          3 │ │  ← 默认选中
│  │ 💡 灵感            5 │ │
│  │ ☑️ 临时事项        2 │ │
│  │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │ │
│  │ 🤖 自动采集     即将 │ │  ← 置灰，V2 实现
│  └──────────────────────┘ │
│                            │
│  [+ 新建]                  │
└────────────────────────────┘
```

- 点击分类 → 工作区筛选展示对应类型
- 默认选中「待阅读」
- 分类右侧显示未处理数量（`status=new` 的条目计数）
- 「自动采集」置灰不可点击，hover 提示「即将上线」
- `[+ 新建]` → 弹出录入弹窗
- 搜索框默认搜索当前分类（匹配字段：标题、标签、备注、URL/来源域名）
- 当前分类搜索无结果时，显示「在全部分类中搜索」可点击链接
- 跨分类搜索时：侧边栏分类取消高亮，搜索结果按分类分组展示，行首标注分类图标
- 清空搜索框 → 回到之前选中的分类
- 搜索仅覆盖 `status=new` 的可见项，不搜已处理/已删除的历史

## 3. 工作区

统一列表视图，行内内容按分类类型适配。

### 3.1 顶部工具栏

```
待阅读 (3)                          [排序 ▾] [批量选择]
───────────────────────────────────────────────────────
```

- 左侧：当前分类名 + 未处理数量
- 右侧：排序控件 + 批量选择按钮
- 排序：默认「最近添加」（时间倒序），可选「最早添加」

### 3.2 各分类行内布局

**待阅读**

```
📖  标题（加粗）
    来源域名 · 相对时间                        [标签]
```

**灵感**

```
💡  标题（加粗）
    正文摘要（最多 2 行，可选）
    相对时间                                   [标签]
```

**临时事项**

```
☑️  标题（加粗）
    相对时间                                   [标签]
```

### 3.3 行交互

**主路径（单击展开详情）**：

```
┌────────────────────────────────────────────────────┐
│ 📖 SwiftUI 状态管理最佳实践                         │
│                                                    │
│ 摘要: 本文介绍了 @Observable 宏在复杂状态...        │
│ 来源: https://mp.weixin.qq.com/s/xxx               │
│ 标签: [SwiftUI] [状态管理]       添加时间: 5/20     │
│                                                    │
│ [编辑] [移入 Focus] [移入 AreaProjects] [删除]      │
└────────────────────────────────────────────────────┘
```

- 单击行 → 行下方内联展开详情面板
- 详情面板包含完整信息和操作按钮
- 再次点击行或点击其他行 → 收起当前详情

**快捷路径（右键菜单）**：

```
┌──────────────────────┐
│ ✏️  编辑              │
│ 🎯 移入 Focus        │
│ 📁 移入 AreaProjects │
│ 🏷  编辑标签          │
│ ─────────────────── │
│ 🗑  删除              │
└──────────────────────┘
```

- 右键菜单是熟练用户的高效路径，跳过展开直接操作
- V1 验收只要求主路径完整，右键菜单作为 P1 增强

### 3.4 批量操作

顶部工具栏点击「批量选择」进入批量模式：

```
  待阅读 (3)                      [排序 ▾] [批量选择·]
  ─────────────────────────────────────────────────
  ☐ 📖 SwiftUI 状态管理最佳实践
  ☑ 📖 WWDC 2026 新 API 汇总
  ☑ 📖 RAG 工程化实践

  ┌─ 底部浮现操作栏 ────────────────────────────────┐
  │ 已选 2 项  [移入 Focus] [移入 AreaProjects] [删除] [取消] │
  └─────────────────────────────────────────────────┘
```

- 行前出现勾选框，底部浮现操作栏
- 仅限当前分类，不可跨分类选择
- 底部操作栏随选中数量浮现/消失
- Esc 或「取消」退出批量模式

### 3.5 删除行为

- 单条删除：无确认弹窗，底部 toast 提示「已删除」+ [撤销]（5 秒内可撤销）
- 批量删除：确认弹窗「确定删除 N 项？」→ 确认后 toast + [撤销]（5 秒）
- 撤销 → InboxItem 从 deleted 恢复为 new

### 3.6 空状态

当前分类无条目时显示专属空状态：

- **待阅读**：📖 没有待阅读的内容 / 粘贴一个 URL 或点击「新建」添加 / [+ 新建]
- **灵感**：💡 还没有灵感记录 / 有想法时随时记下来 / [+ 新建]
- **临时事项**：☑️ 没有临时事项 / 需要临时记录的事情放这里 / [+ 新建]

## 4. 核心交互

| 用户任务 | 操作路径 |
|---------|---------|
| 快速记录一条信息 | [+ 新建] → 录入弹窗 → 输入内容 → ⌘Enter 保存 |
| 粘贴 URL 自动解析 | 录入弹窗粘贴纯 URL（且未手动改过类型）→ 自动切为待阅读 → OG 解析填充标题摘要 |
| 将待阅读转为任务 | 单击展开详情 → 「移入 Focus」→ 选择项目和排期 → 确认 |
| 将灵感沉淀为素材 | 单击展开详情 → 「移入 AreaProjects」→ 选择项目 → 确认 |
| 批量处理 | 「批量选择」→ 勾选 → 底部操作栏选择动作 |
| 搜索特定内容 | 搜索框输入 → 当前分类匹配；无结果时可切换全分类搜索 |

## 5. 状态与规则

### 5.1 录入弹窗

```
┌─ 快速录入 ──────────────────────────────────────────┐
│                                                      │
│  [📖 待阅读 ▾]                                      │
│                                                      │
│  ┌────────────────────────────────────────────────┐ │
│  │ 输入标题、粘贴链接或记录想法...                  │ │
│  │                                                │ │
│  │ （多行，Enter 换行）                            │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│                         [标签 +]    [⌘⏎ 保存]       │
└──────────────────────────────────────────────────────┘
```

**触发**：侧边栏「+ 新建」按钮 或 工作区顶部「+ 新建」按钮。弹窗遵循全局失焦关闭规则。若当前处于批量模式，打开录入弹窗时自动退出批量模式。

**字段**：
- 类型下拉：默认跟随当前侧边栏选中分类（待阅读 / 灵感 / 临时事项）
- 输入框：多行自适应，Enter 换行
- 标签：可选，点击「标签 +」展开标签输入
- 标题为必填（URL 解析成功时自动填充算已填）。保存时 title 为空的降级规则：用输入框首行文本作为 title；纯 URL 场景用 URL 本身或来源域名作为临时 title。OG 解析成功后仅在 title 仍等于临时值且用户未编辑时替换

**保存与关闭**：
- ⌘+Enter → 保存并关闭弹窗
- ⌘+Shift+Enter → 保存并继续录入（弹窗不关，输入框清空）。弹窗按钮只显示主操作 [⌘⏎ 保存]，tooltip 提示此快捷键
- Enter → 正常换行
- Esc（空输入）→ 直接关闭
- Esc（非空）→ 关闭弹窗，不创建 InboxItem，toast「已丢弃」+ [撤销]（5 秒内点击撤销恢复录入弹窗原内容，超时彻底丢弃）
- 失焦（空输入）→ 直接关闭
- 失焦（非空）→ **自动保存为 InboxItem**，不丢弃

### 5.2 URL 自动识别与 OG 解析

**自动识别触发条件**：仅当输入内容是纯 URL（整段只有一个链接）且用户未手动改过类型时，自动切为「待阅读」。否则只提取 URL 元数据补填，不改变当前类型。

**异步解析流程**：
1. 粘贴 URL → 显示 ⏳ 正在解析...
2. 解析成功 → 自动填充标题和摘要，用户可修改
3. 解析失败 → 显示 ⚠️ 无法解析，保留原始 URL
4. V1 仅做 Open Graph 基础解析（og:title / og:description / og:image / og:site_name），不做平台特化

**解析与保存时序**：
- 解析中允许保存，先创建 InboxItem（`metadata_status=pending`）
- 解析成功 → 只补填空字段，不覆盖用户已编辑内容，`metadata_status=ready`
- 解析失败 → 保留原始 URL，`metadata_status=failed`

### 5.3 流出规则：Inbox → Focus

**触发**：详情面板「移入 Focus」按钮 / 批量操作栏（右键菜单为 P1 增强，V1 不验收）

**单条确认面板**：

```
┌─ 移入 Focus ────────────────────────────────────────┐
│                                                      │
│  标题: SwiftUI 状态管理最佳实践         [可编辑]      │
│                                                      │
│  归属项目: [选择项目... ▾]              ← 必选        │
│                                                      │
│  排期: [待排期 ▾]                       ← 默认        │
│        选项: 待排期 / 今日 / 本周 / 指定日期          │
│                                                      │
│                              [取消]  [确认移入]       │
└──────────────────────────────────────────────────────┘
```

**批量确认面板**：

```
┌─ 批量移入 Focus ────────────────────────────────────┐
│                                                      │
│  已选 3 项                                           │
│  · SwiftUI 状态管理最佳实践                           │
│  · WWDC 2026 新 API 汇总                             │
│  · RAG 工程化实践                                     │
│                                                      │
│  归属项目: [选择项目... ▾]              ← 必选，统一   │
│  排期: [待排期 ▾]                       ← 统一应用     │
│                                                      │
│                              [取消]  [确认移入]       │
└──────────────────────────────────────────────────────┘
```

**转化规则**：
- 生成 WorkItem：`source=inbox`，`source_item_id=InboxItem.id`，`schedule` 由用户选择（默认 backlog）
- 原 InboxItem：`status=processed`，记录 `target_type=focus`、`target_ref=WorkItem.id`、`processed_at`
- 原 InboxItem 不物理删除，从列表消失但保留溯源

### 5.4 流出规则：Inbox → AreaProjects

**触发**：详情面板「移入 AreaProjects」按钮 / 批量操作栏（右键菜单为 P1 增强，V1 不验收）

**单条确认面板**：

```
┌─ 移入 AreaProjects ─────────────────────────────────┐
│                                                      │
│  标题: RAG 工程化实践                   [可编辑]      │
│                                                      │
│  目标项目: [选择项目... ▾]              ← 必选        │
│                                                      │
│  存入: _materials/                      ← 固定        │
│                                                      │
│                              [取消]  [确认移入]       │
└──────────────────────────────────────────────────────┘
```

**批量确认面板**：与 Focus 批量面板结构一致，标题区域改为「已选 N 项」+ 条目列表，目标项目统一应用。

**转化规则**：
- V1 所有类型一律存入 `_materials/`，不开放 `_kb/` 选项（遵循知识管道 materials → reports → kb → Anki）
- 文件命名：`{YYYY-MM-DD}-{slug(title)}` 格式；若冲突自动追加短 ID 后缀，绝不覆盖已有文件
- 原 InboxItem：`status=processed`，记录 `target_type=area_projects`、`target_ref=文件路径`、`processed_at`

### 5.5 撤销与回滚

- 移入 Focus/AreaProjects 后 toast 提供 [撤销]（5 秒内）
- 撤销判定：创建目标对象时记录 `created_at`；撤销时判定 `updated_at == created_at` 即视为未编辑，允许删除
- 撤销时：删除未被编辑的目标对象（WorkItem 或文件），InboxItem 从 processed 恢复为 new，清空 target_type/target_ref/processed_at
- 若目标对象已被编辑（`updated_at != created_at`）→ 撤销失败，toast 提示「已在目标位置修改，无法撤销」

### 5.6 自动采集（V2）

V1 侧边栏保留「自动采集」入口但置灰。V2 规划：
- 在 Settings 中配置 Agent 和信息源平台
- Agent 自动采集的内容标记为"待审批"
- 用户审批通过后进入对应分类

## 6. 数据对象

| 对象 | 说明 | 详细定义 |
|------|------|---------|
| InboxItem | 收集箱条目 | 见下方字段定义 |

### InboxItem 字段

```
InboxItem
├── id                : UUID
├── type              : reading | idea | memo       ← 待阅读 | 灵感 | 临时事项
├── title             : String                      ← 必填（URL 解析可自动填充）
├── body              : String?                     ← 正文/备注（灵感的详细内容）
├── url               : URL?                        ← 原始链接
├── tags              : [String]                    ← 用户标签
│
├── status            : new | processed | deleted   ← 生命周期
├── metadata_status   : pending | ready | failed | null  ← OG 解析状态（null=无 URL）
│
│  ── OG 解析补填字段 ──
├── og_title          : String?                     ← Open Graph 标题
├── og_description    : String?                     ← Open Graph 描述
├── og_image          : URL?                        ← Open Graph 封面图
├── og_site_name      : String?                     ← 来源站点名
│
│  ── 流出溯源（仅 status=processed 时必填，其余状态为 null）──
├── target_type       : focus | area_projects | null
├── target_ref        : String?                     ← 目标对象 ID（WorkItem.id 或文件路径）
├── processed_at      : Date?
│
├── created_at        : Date
└── updated_at        : Date
```

**字段约束**：
- `target_type`、`target_ref`、`processed_at` 仅 `status=processed` 时三者必填；`new`/`deleted` 时为 null
- 撤销 processed 时清空这三个字段

## 7. 术语

| 术语 | 含义 |
|------|------|
| Inbox | 页面名称（活动栏 📥），信息统一收集入口 |
| 待阅读 | 收藏的文章/视频/链接，待后续阅读（type=reading） |
| 灵感 | 临时想法、创意片段、摘录、便签类内容的统称（type=idea）。原「灵感」+「便签」合并，不再单独分类 |
| 临时事项 | 需要临时记录的事情（type=memo）。**不是 Focus Task**，只有移入 Focus 后才生成 WorkItem/Task |
| 自动采集 | Agent 根据配置自动抓取的信息源内容（V2 实现） |
| InboxItem | 收集箱条目数据对象 |
| 处理状态 | status 字段：new（未处理，列表可见）→ processed（已移出）/ deleted（已删除） |
| 解析状态 | metadata_status 字段：pending（解析中）/ ready（解析完成）/ failed（解析失败）/ null（无 URL） |
| 流出 | InboxItem 移入 Focus 或 AreaProjects 的操作 |
| 主路径 | 单击展开详情 → 操作按钮（普通用户的完整处理流程） |
| 快捷路径 | 右键菜单直接操作（熟练用户跳过展开的高效路径），V1 作为 P1 增强 |
