## 1. 定位

Studio 是 FocusPilot 的**项目级 AI 工作区**。打开项目目录，选择 Runtime，和 AI 对话工作。

**V1 核心体验 = 对话**。侧边栏提供会话管理、文件浏览、Git 历史三个辅助面板（照搬 Z Code 侧边栏模式）。Studio 和 Focus 相互独立，不做 Task 关联和双向跳转。

### 与其他页面的职责边界

| 页面 | 职责 |
|------|------|
| **Focus** | 计划和任务怎么推进（规划/看板/列表/Task 执行闭环） |
| **Studio** | 打开项目，和 AI 对话工作（纯对话，不涉及任务管理） |
| **AreaProjects** | 项目资产管理和长期沉淀（文件编辑/组织/预览） |

## 3. A 区：侧边栏（260px，⌘B 收起/展开）

### 3.1 A1 — 顶部 Tab 条（36px，固定）

```
┌──────────────────────────────────────┐
│  ( 🤖 )    ( 📁 )    ( 🌿 )        │
│   ━━━                                │
│  会话       文件       Git           │
└──────────────────────────────────────┘
```

三个 Tab 图标，选中项下方蓝色指示线。📁 和 🌿 跟随**当前活跃项目**（= 主区域正在对话的会话所属项目）。

### 3.2 A2 — 内容区（flex，可滚动）

#### 🤖 会话 Tab

```
[+ 添加项目]

── 📌 置顶 ──
📌 ✳ 调试看板拖拽                  ● 3m

── 项目 ──

▾ 📂 FocusPilot              [···] [+]    ← hover 时右侧显示操作按钮
  ✳ 研究 CAAnimation               ● 15m  ← Claude Code Session
  ⚡ 测试 Agent Pull               ✅ 1h   ← Codex Session
  ⚡ 快速原型验证                   ✅ 30m  ← Codex Session

▾ 📂 PilotOne                 [···] [+]
  ◆ 方案评审                       ● 5m   ← Gemini Session

▸ 📂 multica                  [···] [+]   ← 无 Session，折叠
```

**Session 显示格式**：`Runtime图标 + 标题 + 状态(●进行中/✅完成) + 时长`

Runtime 图标是 Session 最重要的标识（创建后不可更改）：✳ Claude Code / ⚡ Codex CLI / ◆ Gemini CLI

**项目行操作**（hover 时显示按钮，参考 Codex）：

| 按钮 | 操作 |
|------|------|
| **[+]** | 创建 Session：弹出 Runtime 选择（✳/⚡/◆），选择后立即创建 |
| **[···]** | 更多操作：编辑名称 / 在访达中打开 / 移除项目 |

[+] 按钮点击弹出：
```
┌──────────────────┐
│ ✳ Claude Code    │
│ ⚡ Codex CLI     │
│ ◆ Gemini CLI    │
└──────────────────┘
```

[···] 按钮点击弹出：
```
┌─────────────────────────┐
│ ✏️ 编辑名称              │
│ 📂 在访达中打开          │
│ ─────────────────────── │
│ ✕ 移除项目              │
└─────────────────────────┘
```

**右键菜单**保留作为快捷方式，内容 = [+] + [···] 合并：
```
┌─────────────────────────┐
│ ✳ 创建会话 (Claude Code) │
│ ⚡ 创建会话 (Codex CLI)  │
│ ◆ 创建会话 (Gemini CLI) │
│ ─────────────────────── │
│ ✏️ 编辑名称              │
│ 📂 在访达中打开          │
│ ─────────────────────── │
│ ✕ 移除项目              │
└─────────────────────────┘
```

创建会话直接选 Runtime，点击后立即创建并在主区域打开对话。Runtime 创建后不可切换，如需使用其他 Runtime，再创建新会话。会话标题自动从首条消息生成。

## 5. C 区：对话区（flex，可滚动）

纯对话界面，和 Claude Code / Codex CLI 一样的聊天体验。消息向上堆叠，新消息在底部。

```
👤 帮我把 animateRow 的动画改成 0.2s ease-out

🤖 好的，我来修改 KanbanDataSource 中的动画参数...

   已修改:
   ~ KanbanDataSource.swift +3 -2

👤 编译一下

🤖 $ make build
   ✓ Build succeeded

👤 安装测试

🤖 $ make install
   ✓ Build succeeded
   ✓ Signing with FocusPilot Dev
   ✓ Installed to /Applications
```

**新建会话空状态**：

```
                  ✳ Claude Code
                claude-opus-4

           开始对话，描述你的需求
```

## 7. 数据模型

### 7.1 CodeSession

```yaml
CodeSession:
  id: "session_abc"
  title: "调试看板拖拽问题"

  # 项目
  project_id: "proj_focuspilot"
  workdir: "/Users/bruce/.../FocusPilot"

  # Runtime（创建时选定，不可更改）
  runtime: claude_code               # claude_code | codex_cli | gemini_cli
  model: "claude-opus-4"             # 由 runtime 决定可选模型

  # 会话状态
  status: active                     # active | idle | done | ended
  pinned: false                      # 是否置顶

  transcript_ref: "sessions/session_abc/transcript.jsonl"

  created_at: "2026-04-23T10:00:00"
  last_active_at: "2026-04-23T10:30:00"
```

### 7.2 Runtime

| Runtime | 图标 | 说明 |
|---------|------|------|
| `claude_code` | ✳ | Claude Code，Anthropic 模型 |
| `codex_cli` | ⚡ | Codex CLI，OpenAI 模型 |
| `gemini_cli` | ◆ | Gemini CLI，Google 模型 |

V2 可扩展更多 Runtime。
