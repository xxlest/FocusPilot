## 1. 定位

Studio 是 FocusPilot 的**项目级 AI 工作区**。打开项目目录，选择 Runtime，和 AI 对话工作。

**V1 核心体验 = 对话**。侧边栏提供会话管理、文件浏览、Git 历史三个辅助面板（照搬 Z Code 侧边栏模式）。Studio 和 Focus 相互独立，不做 Task 关联和双向跳转。

### 与其他页面的职责边界

| 页面 | 职责 |
|------|------|
| **Focus** | 计划和任务怎么推进（规划/看板/列表/Task 执行闭环） |
| **Studio** | 打开项目，和 AI 对话工作（纯对话，不涉及任务管理） |
| **AreaProjects** | 项目资产管理和长期沉淀（文件编辑/组织/预览） |

## 3. A 区：侧边栏（260px，⌘B 收起/展开）

侧边栏分三层：Tab 条（置顶）→ 内容区（可滚动）→ 底部（固定）。

### 3.1 Tab 条（内容区顶部置顶，滚动时不动）

```
┌──────────────────────────────────────┐
│  ( 🤖 )    ( 📁 )    ( 🌿 )        │
│   ━━━                                │
│  会话       文件       Git           │
└──────────────────────────────────────┘
```

三个 Tab 图标，选中项下方蓝色指示线。Tab 条固定在内容区顶部，下方内容滚动时 Tab 不动。📁 和 🌿 跟随**当前活跃项目**（= 主区域正在对话的会话所属项目）。

### 3.2 内容区（Tab 条下方，flex，可滚动）

#### 🤖 会话 Tab

```
🔍 搜索项目和会话...

── 📌 置顶 ──                                  [▸ 折叠]
📌 ✳ 调试看板拖拽                        ● 3m
📌 ◆ 方案评审                            ● 5m

── 📂 项目 ──                    [+ 添加项目] [▸ 折叠]
▾ 📂 FocusPilot                     [···] [+]  ← hover 显示
  ✳ 研究 CAAnimation                     ● 15m
  ⚡ 测试 Agent Pull                     ✅ 1h
  ⚡ 快速原型验证                         ✅ 30m
▾ 📂 PilotOne                       [···] [+]
  ◆ 方案评审                             ● 5m
▸ 📂 multica                        [···] [+]

── 💬 聊天 ──                      [整理 ▾] [▸ 折叠]
✳ 通用技术问答                           ✅ 2m
⚡ API 调研                              ✅ 20m
✳ Markdown 语法查询                      ✅ 5m
```

**三个分区**：

| 分区 | 内容 | 操作 |
|------|------|------|
| **📌 置顶** | 被置顶的 Session（来自任何项目），浮在最上方快速访问 | 可折叠 |
| **📂 项目** | 按项目分组的 Session 列表 | [+ 添加项目]、项目行 hover [···][+]、可折叠 |
| **💬 聊天** | 不按项目分组的平铺 Session 列表（时间顺序） | [整理]菜单、可折叠 |

三个分区均可独立折叠/展开。置顶的 Session 仍属于某个项目，只是在置顶区额外显示一份快捷入口。

**Session 显示格式**：`Runtime图标 + 标题 + 状态(●进行中/✅完成) + 时长`

Runtime 图标是 Session 最重要的标识（创建后不可更改）：✳ Claude Code / ⚡ Codex CLI / ◆ Gemini CLI

```

```

| 分组 | 选项 | 说明 |
|------|------|------|
| 整理 | 按项目 | 💬 聊天区按项目分组显示 |
| | 时间顺序列表 | 💬 聊天区按时间平铺 |
| | 聊天优先 | 活跃聊天排最前 |
| 排序 | 已创建 | 按创建时间排序 |
| | 已更新 | 按最后活跃时间排序 |
| 显示 | 所有聊天 | 显示全部 Session |
| | 相关 | 只显示与当前活跃项目相关的 Session |

   ✓ Build succeeded

👤 安装测试

   ✓ Build succeeded

## 7. 数据模型

### 7.1 CodeSession

```yaml
CodeSession:
  id: "session_abc"
  title: "调试看板拖拽问题"

  # 项目
  project_id: "proj_focuspilot"
  workdir: "/Users/bruce/.../FocusPilot"

  # Runtime（创建时选定，不可更改）
  runtime: claude_code               # claude_code | codex_cli | gemini_cli
  model: "claude-opus-4"             # 由 runtime 决定可选模型

  # 会话状态
  status: active                     # active | idle | done | ended
  pinned: false                      # 是否置顶

  transcript_ref: "sessions/session_abc/transcript.jsonl"

  created_at: "2026-04-23T10:00:00"
  last_active_at: "2026-04-23T10:30:00"
```

### 7.2 Runtime

| Runtime | 图标 | 说明 |
|---------|------|------|
| `claude_code` | ✳ | Claude Code，Anthropic 模型 |
| `codex_cli` | ⚡ | Codex CLI，OpenAI 模型 |
| `gemini_cli` | ◆ | Gemini CLI，Google 模型 |

V2 可扩展更多 Runtime。
