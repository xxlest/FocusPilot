## 1. 定位

Studio 是 FocusPilot 的**项目级 AI 工作区**。打开项目目录，选择 Runtime，和 AI 对话工作。

**V1 核心体验 = 对话**。侧边栏提供会话管理、文件浏览、Git 提交三个辅助面板。Studio 和 Focus 相互独立，不做 Task 关联和双向跳转。自由聊天（无项目绑定的对话）归 Home 页，Studio 不承担。

### 设计来源

| 模块 | 参考来源 | 吸收内容 |
|------|---------|---------|
| **侧边栏布局** | Codex Desktop App | 项目折叠式 Threads 列表、自动化收件箱、hover 操作按钮 |
| **顶栏** | Codex Desktop App | 「用相关工具打开」下拉 |
| **文件管理** | Z Code | 完整资源管理器（目录树 + 隐藏文件切换 + 右键操作） |
| **Git 面板** | Z Code | 提交面板（输入框 + 变更列表 + 提交按钮 + 历史） |
| **右侧栏** | Z Code | 多 Tab 文件浏览窗口 |

### 与其他页面的职责边界

| 页面 | 职责 |
|------|------|
| **Home** | AI 对话入口（含自由聊天）+ 最近操作 |
| **Focus** | 计划和任务怎么推进（规划/看板/列表/Task 执行闭环） |
| **Studio** | 打开项目，和 AI 对话工作（纯项目级对话，不涉及任务管理） |
| **AreaProjects** | 项目资产管理和长期沉淀（文件编辑/组织/预览） |

### 核心交互原则

1. **Workspace = 当前活跃 Session 的项目**。点击任意 Session，整个上下文直接切换（内容区 + 📁 文件 + 🌿 Git + 顶栏 + 终端工作目录），零提示、零延迟
2. **不打断**。高频导航不弹确认框，📁 和 🌿 内部已显示项目名和路径，足够感知当前上下文
3. **不重复**。顶栏显示 Runtime（✳/⚡/◆），不显示具体模型名；模型名只在对话区展示（新建空态 + 消息流）；自由聊天归 Home 页，Studio 不重复
4. **切换即清场**。跨项目切换 Session 时，右侧栏清空并自动收起，终端关闭所有实例并自动收起，确保上下文不污染

## 3. A 区：侧边栏（260px，⌘B 收起/展开）

侧边栏分四层：自动化（固定顶部）→ 搜索 → 项目区（Tab 控制，可滚动）→ 底部（固定）。

### 3.1 自动化区（固定顶部，搜索上方）

```
┌──────────────────────────────────┐
│ 🤖 自动化                   2条  │
│ ──────────────────────────────  │
│ 🔄 每日代码审查       ✅ 2h前   │
│ 🔄 PR 自动检查        🔄 运行中  │
└──────────────────────────────────┘
```

位于侧边栏最顶部（参考 Codex Triage 收件箱位置）。显示自动化任务及其运行状态。

**与普通会话的视觉差异化**：

| | 自动化行 | 普通会话行 |
|---|---|---|
| 图标 | 🔄 固定 | ✳/⚡/◆ Runtime 图标 |
| 副信息 | 运行状态（🔄 运行中 / ✅ 完成 / ❌ 失败）+ 时间 | ● 进行中 / ✅ 完成 + 时长 |

**点击行为**（统一在右侧栏打开，不打断当前会话）：

| 状态 | 点击后 |
|------|--------|
| ✅ 完成 | 右侧栏打开该任务最近一次运行结果的详情视图（若右侧栏关闭则自动展开） |
| ❌ 失败 | 右侧栏打开失败日志视图 |
| 🔄 运行中 | 右侧栏打开实时运行日志视图 |

### 3.2 搜索

```
┌──────────────────────────────────┐
│ 🔍 搜索项目和会话...              │
└──────────────────────────────────┘
```

### 3.3 项目区（Tab 控制）

项目区包含 Tab 条和内容区。Tab 条固定在项目区顶部，内容区可滚动。

Tab 条属于项目区，因为 📁 文件和 🌿 Git 直接影响的是项目，与自动化和搜索无关。

#### Tab 条

```
┌──────────────────────────────────┐
│  ( 🤖 )    ( 📁 )    ( 🌿 )      │
│   ━━━                            │
│  会话       文件       Git       │
└──────────────────────────────────┘
```

三个 Tab 图标，选中项下方蓝色指示线。切换 Tab 时只改变项目区内容。

#### 🤖 会话 Tab

```
📌 置顶                                           [▸ 折叠]
📌 ✳ 调试看板拖拽                              ● 3m
📌 ◆ 方案评审                                  ● 5m

▾ 📂 FocusPilot                     [···] [+]  ← [+] 展开时常显
  ✳ 研究 CAAnimation                       ● 15m
  ⚡ 测试 Agent Pull                       ✅ 1h
  ✳ 快速原型验证                           ✅ 30m
▸ 📂 PilotOne                        [···] [+]  ← 折叠时 hover 显示

── + 添加项目 ──
```

**两个分区**：

| 分区 | 内容 | 操作 |
|------|------|------|
| **📌 置顶** | 被置顶的 Session，从原项目中移出到置顶区（非复制），按最近活跃时间排序 | 可折叠 |
| **📂 项目** | 按项目分组的 Session 列表（Codex 折叠式），已置顶的 Session 不在此显示 | 项目行 hover [···][+]、可折叠 |

**Session 显示格式**：`Runtime图标 + 标题 + 状态(●进行中/✅完成) + 时长`

Runtime 图标是 Session 最重要的标识（创建后不可更改）：✳ Claude Code / ⚡ Codex CLI / ◆ Gemini CLI

**项目行操作**（hover 时显示按钮，参考 Codex）：

| 按钮 | 操作 |
|------|------|
| **[+]** | 创建 Session：弹出 Runtime 选择（✳/⚡/◆），选择后立即创建。项目展开时常显，折叠时 hover 显示 |
| **[···]** | 更多操作：编辑名称 / 在访达中打开 / 移除项目。hover 时显示 |


## 7. E 区：右侧栏（文件浏览窗口，⌘⇧B 切换）

右侧栏提供多 Tab 文件浏览能力，支持同时打开多个文件。左侧保留对话窗口，右侧浏览文件，实现对话 + 文件参照的分屏工作模式。

```
┌─ 右侧栏 ────────────────────────┐
│ ┌─ Tab 条 ────────────────────┐ │
│ │ AppDelegate.swift │ Models… │✕│ │
│ └─────────────────────────────┘ │
│                                  │
│  1  import AppKit                │
│  2  import SwiftUI               │
│  3                               │
│  4  class AppDelegate:           │
│  5    NSObject,                  │
│  6    NSApplicationDelegate {    │
│  7    ...                        │
│                                  │
└──────────────────────────────────┘
```

| 功能 | 说明 |
|------|------|
| **Tab 多页签** | 同时打开多个文件，Tab 切换浏览 |
| **打开方式** | 📁 文件 Tab 单击预览（若关闭则自动展开）/ 📁 文件 Tab 双击（持久 Tab）/ 对话区 Diff 链接点击 / 🌿 Git 变更文件点击 / 右键「在侧栏打开」/ 自动化任务点击（详情/日志） |
| **只读浏览** | 查看文件内容，不提供编辑（编辑交给 AI 或外部工具） |
| **关闭 Tab** | 每个 Tab 可单独关闭（✕），全部关闭后右侧栏自动收起 |
| **快捷键** | ⌘⇧B 切换显隐 |

## 8. F 区：终端面板（输入区下方，最底部）

位于输入区下方、页面最底部的可展开/收起内嵌终端。通过顶栏 [🖥] 按钮或 ⌘J 切换显隐。支持多终端实例，Tab 式管理。

```
┌─ 终端面板 ─────────────────────────────────────┐
│ ┌─ Tab 条 ──────────────────────────────────┐  │
│ │ Terminal 1 ✕ │ Terminal 2 ✕ │     [+]      │  │
│ │ ━━━━━━━━━━━━                               │  │
│ └────────────────────────────────────────────┘  │
│ $ make build                                    │
│ Build succeeded.                                │
│ $                                               │
└─────────────────────────────────────────────────┘
```

| 功能 | 说明 |
|------|------|
| **位置** | 输入区（D 区）下方，页面最底部 |
| **切换显隐** | 顶栏 [🖥] 按钮或 ⌘J 快捷键 |
| **[+] 新建终端** | Tab 条右侧，点击新建一个终端实例（横向排列） |
| **✕ 关闭终端** | 每个 Tab 可单独关闭，最后一个关闭后终端面板自动收起 |
| **工作目录** | 新建终端自动设为当前 Workspace 路径 |
| **切换 Workspace** | 跨项目切换时关闭所有终端实例，终端面板自动收起（切换即清场，与右侧栏一致） |

## 9. 数据模型

### 9.1 CodeSession

```yaml
CodeSession:
  id: "session_abc"
  title: "调试看板拖拽问题"

  # 项目
  project_id: "proj_focuspilot"
  workdir: "/Users/bruce/.../FocusPilot"

  # Runtime（创建时选定，不可更改）
  runtime: claude_code               # claude_code | codex_cli | gemini_cli
  model: "claude-opus-4"             # 由 runtime 决定可选模型

  # 会话状态
  status: active                     # active | idle | done | ended
  pinned: false                      # 是否置顶

  transcript_ref: "sessions/session_abc/transcript.jsonl"

  created_at: "2026-04-23T10:00:00"
  last_active_at: "2026-04-23T10:30:00"
```

### 9.2 AutomationTask

```yaml
AutomationTask:
  id: "auto_001"
  name: "每日代码审查"
  project_id: "proj_focuspilot"

  # 调度
  schedule: "0 9 * * *"              # cron 表达式
  runtime: claude_code

  # 状态
  status: completed                  # running | completed | failed
  last_run_at: "2026-04-24T09:00:00"
  last_result_summary: "发现 2 个潜在问题"
```

### 9.3 Runtime

| Runtime | 图标 | 说明 |
|---------|------|------|
| `claude_code` | ✳ | Claude Code，Anthropic 模型 |
| `codex_cli` | ⚡ | Codex CLI，OpenAI 模型 |
| `gemini_cli` | ◆ | Gemini CLI，Google 模型 |

V2 可扩展更多 Runtime。

## 10. 交互规则汇总

### 10.1 Workspace 上下文切换

| 操作 | 行为 |
|------|------|
| 点击任意 Session | 内容区 + 📁 文件 + 🌿 Git + 顶栏，全部切换到该 Session 所属项目 |
| 点击其他项目的 Session | 同上，直接切换，不弹确认框 |
| 跨项目切换时右侧栏 | 清空所有已打开的文件 Tab，自动收起右侧栏 |
| 跨项目切换时终端 | 关闭所有终端实例，自动收起终端面板 |
| 📁 / 🌿 Tab 切换 | 只改变项目区内容，不影响自动化和搜索 |

### 10.2 置顶规则

| 规则 | 说明 |
|------|------|
| 置顶行为 | Session 从原项目列表中移出到置顶区（移动，非复制） |
| 取消置顶 | Session 回到原所属项目列表中 |
| 排序 | 按最近活跃时间排序 |
| 点击置顶 Session | 和点击普通 Session 一样，切换到该 Session 所属项目的完整上下文 |

### 10.3 文件打开路径

所有文件预览均在右侧栏进行，不占用对话区。

| 触发 | 目标 |
|------|------|
| 📁 文件 Tab 单击文件 | 右侧栏预览（若关闭则自动展开） |
| 📁 文件 Tab 双击文件 | 右侧栏打开为持久 Tab |
| 对话区 Diff 链接点击 | 右侧栏打开新 Tab |
| 🌿 Git Tab 点击变更文件 | 右侧栏打开 Diff 预览 |
| 右键文件 →「在侧栏打开」 | 右侧栏打开新 Tab |

### 10.4 快捷键

| 快捷键 | 功能 |
|--------|------|
| ⌘B | 切换左侧边栏显隐 |
| ⌘⇧B | 切换右侧栏显隐 |
| ⌘J | 切换终端面板显隐 |
| ⌘N | 在当前项目创建新 Session |
| ⌘↩ | Git 提交（🌿 Tab 内） |
