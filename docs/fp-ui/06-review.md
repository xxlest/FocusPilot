# Review 页面设计

> **状态**：可开发
> **更新**：2026-05-28
> **原型**：[00-layout-prototype.html](00-layout-prototype.html)、[06-review-today-prototype.html](06-review-today-prototype.html)、[06-review-challenge-prototype.html](06-review-challenge-prototype.html)

---

## 1. 定位

Review 是 FocusPilot 的**复习与内化中心**，承接知识卡片生成之后的长期记忆与能力内化。

它不是一个附属功能，而是个人终身自迭代系统的核心闭环之一：

`Materials / Reports → 卡片生成 → Review 记忆 → Review 内化 → Studio 实践 → 结果反哺`

**核心目标**：

- 让知识卡片不只停留在“被生成”
- 让用户通过定期复习，把内容从“看过”变成“记住”
- 让用户通过费曼复述和场景应用，把内容从“记住”变成“会用”

---

## 2. 与其他页面的职责边界

| 页面 | 职责 |
|------|------|
| **Home** | 展示今日待复习数、弱项卡、关键卡待确认等摘要，不承担深度复习 |
| **Studio** | 承接真实任务和实践，把已内化知识用于执行，并反哺卡片 |
| **Projects** | 管理卡片归属、来源、知识沉淀结构和追溯关系 |
| **Review** | 完成记忆、内化、卡片管理和复习统计 |

**硬规则**：

- Home 只看摘要和跳转，不做深度复习
- Studio 只做实践，不做逐卡记忆
- Review 才是“今日复习”和“内化挑战”的主场

---

## 3. 一级导航

Review 作为活动栏一级模块，侧边栏固定四个入口：

1. `今日复习`
2. `内化挑战`
3. `卡片库`
4. `统计`

默认进入 `今日复习`。

各入口职责：

- **今日复习**：今日队列主入口，完成记忆阶段
- **内化挑战**：承接今天需要进一步理解与迁移应用的卡
- **卡片库**：按 `Area` 或 `来源` 浏览、检索和管理卡片
- **统计**：看习惯、记忆强度、内化程度和 Area 薄弱点

---

## 4. 主链路

Review 的主链路固定为：

`今日复习队列 → 记忆阶段 → 强提示进入内化 → 内化挑战`

说明：

- 记忆和内化不是一回事
- 记忆阶段先走四档反馈
- 今日复习完成后，系统强提示进入 `内化挑战`
- 但不自动跳转，用户可以稍后再做

---

## 5. 今日复习队列

### 5.1 队列组成

今日复习队列由三部分组成：

- **到期卡**：按遗忘曲线排到今天的卡
- **新卡**：新进入系统的卡
- **弱项强化**：近期答错、模糊、或长期内化不足的卡

### 5.2 记忆阶段反馈

每张卡复习后使用四档反馈：

- `忘了`
- `模糊`
- `记得`
- `熟练`

这四档用于更新：

- 记忆强度
- 下次复习时间
- 是否进入弱项强化池

### 5.3 今日完成后的动作

记忆阶段完成后，页面展示：

- 今日完成张数
- 记忆反馈分布
- 新增弱项卡数量
- 可进入内化挑战的卡数

主 CTA：

- `进入内化挑战`

次 CTA：

- `稍后再做`

---

## 6. 内化挑战

### 6.1 挑战对象

不是所有卡都进入内化挑战，只抽取：

- 新卡
- 弱项卡
- 关键卡

### 6.2 关键卡机制

关键卡采用混合判定：

- AI 先推荐
- 用户可确认或覆盖

### 6.3 挑战方式

内化挑战采用混合模式：

- **费曼复述**
  - 适用于概念类、框架类卡片
  - 目标是确认“你能不能用自己的话解释清楚”
- **场景应用**
  - 适用于方法类、决策类、经验类卡片
  - 目标是确认“你能不能在情境中用出来”

AI 根据卡片类型自动推荐题型。

---

## 7. 卡片库

卡片库支持双视图：

- **按 Area / 能力主题**
- **按来源**

默认按 `Area` 浏览，也可切到 `来源` 看追溯链。

卡片库承担的职责：

- 检索卡片
- 查看来源
- 查看复习历史
- 查看内化记录
- 标记关键卡 / 暂停 / 归档

---

## 8. 统计

统计页不做操作，只做反馈和诊断。

建议分为四组指标：

- **复习习惯**
  - 连续复习天数
  - 完成率
  - 每周复习时长
- **记忆状态**
  - 忘了 / 模糊 / 记得 / 熟练 分布
  - 到期积压量
  - 新卡进入量
- **内化状态**
  - 未内化 / 能解释 / 能应用 / 已融会 分布
  - 最近提升的卡
  - 长期卡住的卡
- **Area 视角**
  - 哪些主题掌握度高
  - 哪些主题弱项多
  - 哪些主题复习多但实践少

---

## 9. 卡片生命周期

卡片状态分为两层：

### 9.1 生产状态

- `draft`
- `pending_confirm`
- `active`
- `suspended`
- `archived`

说明：

- 普通卡可从 `draft` 直接进入 `active`
- 关键卡从 `draft` 进入 `pending_confirm`
- 用户确认后进入 `active`

### 9.2 学习状态

学习状态不再用单一总等级，而是拆成两条维度：

- **记忆强度**
  - `忘了`
  - `模糊`
  - `记得`
  - `熟练`
- **内化程度**
  - `未内化`
  - `能解释`
  - `能应用`
  - `已融会`

这两个维度独立变化。

例如：

- 一张卡可以 `记得`，但仍然 `未内化`
- 一张卡要进入 `已融会`，最好有真实实践证据支持

## 11. V1 范围

V1 要做：

- 自建复习系统
- 今日复习队列
- 四档记忆反馈
- 简单遗忘曲线排期
- 内化挑战
- 双维度掌握度
- 关键卡确认机制
- Home 摘要接入

V1 暂不做：

- 复杂 SRS 算法调参
- 多人共享卡组
- 公开题库市场
- 重型知识图谱自动推理
- 强依赖 Anki 的主流程
