## 1. 定位

Projects 是 FocusPilot 的**记忆与输入管理层**，承载信息收集和知识沉淀两个阶段。

侧边栏通过 Tab 切换两个模块：
- **Inbox**（未归属信息入口）：待阅读内容、灵感和临时事项的统一收集处，信息在此短暂停留，整理后流出
- **Projects**（已归属知识容器）：项目的资产沉淀区，包含执行类项目和知识类项目

核心理念：Inbox 是未归属池，Project 是已归属容器。两者同处"个人系统记忆层"，但状态和操作严格区分。

### 与其他页面的职责边界

| 维度 | Projects（本页） | Studio | Review |
|------|-----------------|--------|--------|
| 关注点 | 信息在哪里、归到哪里、沉淀在哪里 | 任务怎么推进、和 Agent 怎么工作 | 知识如何被记住并内化 |
| 核心操作 | 录入、分类、流出、浏览、编辑、组织文件 | 规划、看板、对话、代码、Git | 复习、挑战、统计 |
| 数据形态 | InboxItem + 文件/文件夹 | WorkItem + Session + ExecutionRun | ReviewCard |

**原则**：Projects 管信息的收集和存放，Studio 管任务和会话。三者通过项目名关联但不互相替代。

### 两类项目的差异

| | 执行类项目 | 知识类项目 |
|---|---|---|
| 典型内容 | 代码仓库、交付物、设计稿 | 学习笔记、参考资料、行业分析 |
| 与 Studio 关联 | 有对应的 WorkItem/Task | 无 |
| 知识管道 | 可选（项目附带沉淀） | 核心（主要使用场景） |
| 产出结构 | task + 交付物 + 可选 _materials/_reports/_kb/ | _materials/ + _reports/ + _kb/ |

## 2. 侧边栏

顶部 Tab 切换 `[📥 Inbox | 📂 Projects]`，与 Home 页的 `[💬 对话 | 📊 概览]` 模式一致。同一时刻只显示一个 Tab 的内容，切换时整个侧边栏内容替换。

### 2.1 默认 Tab 规则

| 进入方式 | 默认 Tab |
|---------|---------|
| 点击活动栏 Projects 图标 | **上次用户选择的 Tab**（首次进入默认 Projects Tab） |
| 点击活动栏 Projects 角标数字 | Inbox Tab |
| Home Widget "Inbox 待处理" 点击跳转 | Inbox Tab |
| 全局快捷键 ⌘⇧N 录入后 | 不跳转（录入弹窗独立于页面） |
| 其他页面"归入项目"流出完成 | Projects Tab + 高亮目标文件 |
| Home Widget "活跃项目" 点击跳转 | Projects Tab |

**角标与指标来源**：
- 活动栏 Projects 角标 = Inbox 未处理数（`InboxItem.status=new` 总数）
- Home 顶部"待处理"指标 = 同上（Inbox 未处理数）
- Home 侧边栏"待沉淀素材" = 已进入 Projects 但尚未加工的素材数（`_materials/` 中 `status=new` 的 Material 数）

核心原则：**Projects 是稳定入口，Inbox 靠角标和定向跳转引导，不主动劫持**。

- 搜索仅覆盖 `status=new` 的可见项，不搜已处理/已删除的历史

### 2.4 Projects Tab 工具栏

4 个图标按钮（仅图标，hover 显示文字提示）：

| 按钮 | 功能 | 说明 |
|------|------|------|
| 📁 | 新建文件夹 | 在选中位置下创建，同名自动后缀编号 |
| 📄 | 新建文件 | 默认 Markdown 格式，在选中位置下创建 |
| ⭐ | 收藏筛选 | 切换：仅显示已收藏的项目 |
| ⤡ | 展开/折叠全部 | 点击切换：全部展开 ↔ 全部折叠 |

工具栏操作的"选中位置"规则：
- 有选中文件夹 → 在该文件夹内创建
- 有选中文件 → 在该文件所在文件夹创建
- 无选中项 → 提示先选择项目

### 2.5 Projects Tab 文件树交互

**展开/折叠**：
- 单击文件夹 → 展开/折叠切换
- 单击文件 → 编辑器打开该文件
- 展开/折叠状态按项目持久化（项目粒度，重启恢复）

**拖拽**：
- 拖拽文件/文件夹到目标文件夹 → 移动
- 拖拽到项目根目录 → 移到项目顶层
- 拖拽过程中目标文件夹高亮指示
- 拖到知识管道文件夹（_materials/_reports/_kb/）时正常移入
- 不允许拖拽项目本身（项目顺序通过右键菜单置顶调整）
- 不允许跨项目拖拽（使用右键菜单「移动到...」完成跨项目移动）

**内联重命名**：
- 双击文件/文件夹名 → 进入内联编辑模式
- Enter 确认，Esc 取消
- 知识管道文件夹（_materials/_reports/_kb/）不允许重命名

**多选**：
- Cmd+单击 → 追加选中/取消选中
- 多选后右键 → 批量操作菜单（移动到... / 删除）
- 多选不可跨项目

### 2.7 知识管道文件夹标记

知识管道的三个特殊文件夹使用区别于普通文件夹的图标和角标：

| 文件夹 | 图标 | 角标 | 含义 |
|--------|------|------|------|
| `_materials/` | 📥 | 数字（`status=new` 的素材数） | 待整合的原始素材 |
| `_reports/` | 📊 | 无 | 增量整合报告 |
| `_kb/` | 🧠 | 数字（总卡片数） | 知识卡片 |

- 角标为 0 时不显示
- 项目内无 `_materials/` `_reports/` `_kb/` 文件夹时，不自动创建、不占位显示
- 首次使用知识管道功能（如从 Inbox 流入素材）时自动创建对应文件夹

工作区内容随侧边栏 Tab 和选中项切换。

### 3.1 Inbox Tab 工作区

统一列表视图，行内内容按分类类型适配。

#### 3.1.4 批量操作

顶部工具栏点击「批量选择」进入批量模式：

```

  ─────────────────────────────────────────────────
  ☐ 📖 SwiftUI 状态管理最佳实践
  ☑ 📖 WWDC 2026 新 API 汇总
  ☑ 📖 RAG 工程化实践

  ┌─ 底部浮现操作栏 ────────────────────────────────┐

  └─────────────────────────────────────────────────┘
```

- 行前出现勾选框，底部浮现操作栏
- 仅限当前分类，不可跨分类选择
- 底部操作栏随选中数量浮现/消失
- Esc 或「取消」退出批量模式

#### 3.1.5 删除行为

- 单条删除：无确认弹窗，底部 toast 提示「已删除」+ [撤销]（5 秒内可撤销）
- 批量删除：确认弹窗「确定删除 N 项？」→ 确认后 toast + [撤销]（5 秒）
- 撤销 → InboxItem 从 deleted 恢复为 new

#### 3.1.6 Inbox 空状态

当前分类无条目时显示专属空状态：

- **待阅读**：📖 没有待阅读的内容 / 粘贴一个 URL 或点击「新建」添加 / [+ 新建]
- **灵感**：💡 还没有灵感记录 / 有想法时随时记下来 / [+ 新建]
- **临时事项**：☑️ 没有临时事项 / 需要临时记录的事情放这里 / [+ 新建]

**模式切换**：面包屑右侧 [预览|编辑] 分段控件。

| 模式 | 行为 |
|------|------|
| **预览**（默认） | 渲染 Markdown，只读。点击编辑区域不进入编辑 |
| **编辑** | 左侧源码编辑，右侧实时预览（如工作区宽度不足则隐藏预览） |

**V1 支持的 Markdown 特性**：

| 特性 | 支持 |
|------|------|
| 标题 / 粗体 / 斜体 / 列表 / 链接 / 图片 | ✅ |
| 代码块（语法高亮） | ✅ |
| 表格 | ✅ |
| 复选框 `- [ ]` | ✅ |
| 脚注 | ✅ |
| Mermaid 图表 | V2 |
| 数学公式（KaTeX） | V2 |
| Wikilink `[[]]` | V2（兼容 VaultOne） |

**自动保存**：编辑模式下每 3 秒自动保存（无脏数据时不触发），手动 ⌘S 立即保存。

**Frontmatter 处理**：
- 预览模式：frontmatter 折叠为一行摘要（`type: project · status: planning · 3 tags`），点击展开完整 YAML

## 7. Inbox 模块规格

本章节是 Inbox 的完整领域契约（数据模型、生命周期、流出规则、撤销规则、URL 解析），是所有 Inbox 相关实现的权威来源。

### 7.1 录入弹窗

```
┌─ 快速录入 ──────────────────────────────────────────┐
│                                                      │
│  [📖 待阅读 ▾]                                      │
│                                                      │
│  ┌────────────────────────────────────────────────┐ │
│  │ 输入标题、粘贴链接或记录想法...                  │ │
│  │                                                │ │
│  │ （多行，Enter 换行）                            │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│                         [标签 +]    [⌘⏎ 保存]       │
└──────────────────────────────────────────────────────┘
```

**触发**：侧边栏「+ 新建」按钮、工作区顶部「+ 新建」按钮、全局快捷键 ⌘⇧N。弹窗遵循全局失焦关闭规则。若当前处于批量模式，打开录入弹窗时自动退出批量模式。

**字段**：
- 类型下拉：默认跟随当前侧边栏选中分类（待阅读 / 灵感 / 临时事项）
- 输入框：多行自适应，Enter 换行
- 标签：可选，点击「标签 +」展开标签输入
- 标题为必填（URL 解析成功时自动填充算已填）。保存时 title 为空的降级规则：用输入框首行文本作为 title；纯 URL 场景用 URL 本身或来源域名作为临时 title。OG 解析成功后仅在 title 仍等于临时值且用户未编辑时替换

**保存与关闭**：
- ⌘+Enter → 保存并关闭弹窗
- ⌘+Shift+Enter → 保存并继续录入（弹窗不关，输入框清空）。弹窗按钮只显示主操作 [⌘⏎ 保存]，tooltip 提示此快捷键
- Enter → 正常换行
- Esc（空输入）→ 直接关闭
- Esc（非空）→ 关闭弹窗，不创建 InboxItem，toast「已丢弃」+ [撤销]（5 秒内点击撤销恢复录入弹窗原内容，超时彻底丢弃）
- 失焦（空输入）→ 直接关闭
- 失焦（非空）→ **自动保存为 InboxItem**，不丢弃

### 7.2 URL 自动识别与 OG 解析

**自动识别触发条件**：仅当输入内容是纯 URL（整段只有一个链接）且用户未手动改过类型时，自动切为「待阅读」。否则只提取 URL 元数据补填，不改变当前类型。

**异步解析流程**：
1. 粘贴 URL → 显示 ⏳ 正在解析...
2. 解析成功 → 自动填充标题和摘要，用户可修改
3. 解析失败 → 显示 ⚠️ 无法解析，保留原始 URL
4. V1 仅做 Open Graph 基础解析（og:title / og:description / og:image / og:site_name），不做平台特化

**解析与保存时序**：
- 解析中允许保存，先创建 InboxItem（`metadata_status=pending`）
- 解析成功 → 只补填空字段，不覆盖用户已编辑内容，`metadata_status=ready`
- 解析失败 → 保留原始 URL，`metadata_status=failed`

### 7.3 流出规则：Inbox → Studio（转为工作项）

**触发**：详情面板「转为工作项」按钮 / 批量操作栏（右键菜单为 P1 增强，V1 不验收）

**单条确认面板**：

```
│  标题: SwiftUI 状态管理最佳实践         [可编辑]      │

│                                                      │
│  排期: [待排期 ▾]                       ← 默认        │
│        选项: 待排期 / 今日 / 本周 / 指定日期          │
│                                                      │
│                              [取消]  [确认]           │
└──────────────────────────────────────────────────────┘
```

**批量确认面板**：

**触发**：详情面板「归入项目」按钮 / 批量操作栏（右键菜单为 P1 增强，V1 不验收）

**单条确认面板**：

```
┌─ 归入项目 ──────────────────────────────────────────┐
│                                                      │
│  标题: RAG 工程化实践                   [可编辑]      │
│                                                      │
│  目标项目: [选择项目... ▾]              ← 必选        │
│                                                      │
│  存入: _materials/                      ← 固定        │
│                                                      │
│                              [取消]  [确认归入]       │
└──────────────────────────────────────────────────────┘
```

**批量确认面板**：与转为工作项的批量面板结构一致，标题区域改为「已选 N 项」+ 条目列表，目标项目统一应用。

**转化规则**：
- V1 所有类型一律存入 `_materials/`，不开放 `_kb/` 选项（遵循知识管道 materials → reports → kb → Anki）
- 文件命名：`{YYYY-MM-DD}-{slug(title)}` 格式；若冲突自动追加短 ID 后缀，绝不覆盖已有文件
- 原 InboxItem：`status=processed`，记录 `target_type=area_projects`、`target_ref=文件路径`、`processed_at`

**归入后体验**：确认移入后，侧边栏自动切换到 Projects Tab，文件树展开到新创建的文件位置并高亮闪烁 1 秒，让用户看到"信息落地了"。

### 7.5 撤销与回滚

- 转为工作项/归入项目后 toast 提供 [撤销]（5 秒内）
- 撤销判定：创建目标对象时记录 `created_at`；撤销时判定 `updated_at == created_at` 即视为未编辑，允许删除
- 撤销时：删除未被编辑的目标对象（WorkItem 或文件），InboxItem 从 processed 恢复为 new，清空 target_type/target_ref/processed_at
- 若目标对象已被编辑（`updated_at != created_at`）→ 撤销失败，toast 提示「已在目标位置修改，无法撤销」

### 7.6 自动采集（V2）

V1 侧边栏保留「自动采集」入口但置灰。V2 规划：
- 在 Settings 中配置 Agent 和信息源平台
- Agent 自动采集的内容标记为"待审批"
- 用户审批通过后进入对应分类

### 7.7 Inbox 流入 Projects 规则

Inbox 归入项目后，Projects 侧的接收行为：

| 步骤 | 行为 |
|------|------|
| 用户在 Inbox Tab 选择「归入项目」| 弹出确认面板，选择目标项目 |
| 确认归入 | 在目标项目 `_materials/` 下创建文件（`{YYYY-MM-DD}-{slug}.md`） |
| 侧边栏切换 | 自动切到 Projects Tab，文件树展开到新文件位置并高亮 |
| 无 _materials/ 时 | 自动创建 _materials/ 文件夹 |

## 8. 数据对象

### 8.1 InboxItem

```
InboxItem
├── id                : UUID
├── type              : reading | idea | memo       ← 待阅读 | 灵感 | 临时事项
├── title             : String                      ← 必填（URL 解析可自动填充）
├── body              : String?                     ← 正文/备注（灵感的详细内容）
├── url               : URL?                        ← 原始链接
├── tags              : [String]                    ← 用户标签
│
├── status            : new | processed | deleted   ← 生命周期
├── metadata_status   : pending | ready | failed | null  ← OG 解析状态（null=无 URL）
│
│  ── OG 解析补填字段 ──
├── og_title          : String?                     ← Open Graph 标题
├── og_description    : String?                     ← Open Graph 描述
├── og_image          : URL?                        ← Open Graph 封面图
├── og_site_name      : String?                     ← 来源站点名
│
│  ── 流出溯源（仅 status=processed 时必填，其余状态为 null）──
├── target_type       : focus | area_projects | null  ← 字段值沿用历史命名，用户侧显示 Projects
├── target_ref        : String?                     ← 目标对象 ID（WorkItem.id 或文件路径）
├── processed_at      : Date?
│
├── created_at        : Date
└── updated_at        : Date
```

**字段约束**：
- `target_type`、`target_ref`、`processed_at` 仅 `status=processed` 时三者必填；`new`/`deleted` 时为 null
- 撤销 processed 时清空这三个字段

### 8.2 Project

项目（执行类/知识类），文件夹层级的顶层单元。详见 [PRD §4.1](../PRD.md)。

## 9. 术语

| 术语 | 含义 |
|------|------|
| Projects | 页面名称（活动栏 📂），个人系统的记忆与输入管理层 |
| Inbox | Projects 页面的第一个侧边栏 Tab，未归属信息入口 |
| 待阅读 | 收藏的文章/视频/链接，待后续阅读（type=reading） |
| 灵感 | 临时想法、创意片段、摘录、便签类内容的统称（type=idea）。原「灵感」+「便签」合并，不再单独分类 |
| 临时事项 | 需要临时记录的事情（type=memo）。**不是 Studio WorkItem**，只有转为工作项后才生成 WorkItem/Task |
| 自动采集 | Agent 根据配置自动抓取的信息源内容（V2 实现） |
| InboxItem | 收集箱条目数据对象 |
| 处理状态 | status 字段：new（未处理，列表可见）→ processed（已移出）/ deleted（已删除） |
| 解析状态 | metadata_status 字段：pending（解析中）/ ready（解析完成）/ failed（解析失败）/ null（无 URL） |
| 流出 | InboxItem 转为工作项或归入项目的操作 |
| 转为工作项 | InboxItem → Studio WorkItem 的流出动作 |
| 归入项目 | InboxItem → Projects _materials/ 的流出动作 |
| 主路径 | 单击展开详情 → 操作按钮（普通用户的完整处理流程） |
| 快捷路径 | 右键菜单直接操作（熟练用户跳过展开的高效路径），V1 作为 P1 增强 |
| 执行类项目 | 需要拆解执行的项目，与 Studio WorkItem 关联 |
| 知识类项目 | 知识沉淀和积累，不关联任务 |
| 辅助栏 | 工作区右侧可收起的终端/Agent 对话面板（⌘⇧B 切换） |
| 知识管道 | _materials/ → _reports/ → _kb/ → Anki 的完整加工链路 |
| 同步整合 | 触发知识管道的操作：新素材融入报告 + 提炼卡片 |
| KB 卡片 | 知识卡片，原子化要点，_kb/ 文件夹下的特殊 Markdown 文件 |
| Mastery | 掌握层次：know（知道）→ understand（理解）→ master（精通） |
| 卡片视图 | 选中 _kb/ 文件夹时，工作区从编辑器切换为卡片列表展示 |

## 10. 验收用例

| # | 用例 | 预期结果 |
|---|------|---------|
| 1 | 点击活动栏 Projects | 进入上次选择的 Tab（首次默认 Projects Tab） |
| 2 | 点击活动栏 Projects 角标数字 | 进入 Inbox Tab |
| 3 | Inbox 录入一条 → 点"归入项目" | 确认后切 Projects Tab，文件树展开并高亮新文件 |
| 4 | 归入项目后 5 秒内点撤销 | InboxItem 恢复为 new，目标文件删除 |
| 5 | ⌘⇧N 快速录入 | 弹窗弹出，保存后生成 InboxItem |
| 6 | Home 概览 Widget "Inbox 待处理" 点击 | 跳转 Projects 页 Inbox Tab |
| 7 | Inbox 批量选择 3 项 → "转为工作项" | 批量确认面板弹出，确认后 3 项变为 processed |
| 8 | Inbox 全部处理完 → 切到 Projects Tab → 离开再回来 | 仍显示 Projects Tab（不被空 Inbox 干扰） |
