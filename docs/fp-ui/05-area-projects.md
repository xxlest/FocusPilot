# AreaProjects 页面设计

> **状态**：草稿
> **更新**：2026-05-15
> **参考**：Obsidian 文件树、VS Code Explorer

---

## 1. 定位

AreaProjects 是项目的**资产沉淀区**，承载两类项目：
1. **执行类项目**：需要拆解和执行的项目（与 Focus 的 WorkItem 关联）
2. **知识类项目**：知识沉淀和积累（文档、笔记、参考资料）

### 与其他页面的职责边界

| 维度 | AreaProjects | Focus | Studio |
|------|-------------|-------|--------|
| 关注点 | 资料和产物沉淀在哪里 | 计划和任务怎么推进 | 和 Agent 怎么工作 |
| 核心操作 | 浏览、编辑、组织文件 | 拖拽、排期、状态流转 | 对话、代码、Git |
| 数据形态 | 文件/文件夹 | WorkItem | CodeSession |

**原则**：AreaProjects 管文件，Focus 管任务，Studio 管会话。三者通过项目名关联但不互相替代。

## 2. 侧边栏

类似 Obsidian / VS Code 的文件树导航。

```
┌─ 侧边栏 ──────────────────┐
│  🔍 搜索...                │
│  [📁] [📄] [⭐] [⤡]       │  <- 工具栏：新建文件夹/文件、收藏筛选、展开折叠
│                            │
│  ▾ 📂 FocusPilot           │
│    📄 架构设计.md           │
│    📄 技术选型.md           │
│    ▸ 📂 参考资料            │
│  ▾ 📂 大客户项目           │
│    📄 客户画像.md           │
│    📄 竞品分析.md           │
│  ▸ 📂 个人知识库           │
│                            │
└────────────────────────────┘
```

- 文件树支持拖拽排序和移动
- 右键菜单：重命名、删除、移动、复制路径
- 搜索匹配文件夹和文件，结果分区显示
- 收藏筛选：仅显示已收藏的项目

### 工具栏

4 个图标按钮（仅图标，hover 显示文字）：
- 📁 创建文件夹：在选中位置创建，同名自动后缀编号
- 📄 创建文件：默认 Markdown 格式
- ⭐ 收藏筛选：按收藏的项目过滤展示
- ⤡ 展开/折叠全部

## 3. 工作区

左右分栏布局，分割符可拖动调整比例。

```
┌─ 工作区 ──────────────────────┬─ 辅助栏（可收起）──────┐
│                                │                        │
│  📄 架构设计.md    [预览|编辑] │  [终端 | Agent 对话]   │
│  ───────────────────────────  │                        │
│                                │  （内容随 Tab 切换）   │
│  （Markdown 内容）             │                        │
│                                │                        │
│                                │                        │
└────────────────────────────────┴────────────────────────┘
```

### 左半区（编辑器）

- 点击侧边栏文件 -> 显示文件内容
- 支持 Markdown 预览和编辑模式切换
- 顶部显示文件路径面包屑

### 右半区（辅助栏）

- 顶部标题栏最右边按钮控制收起/展开
- 顶部 `[+]` 弹出下拉选择：新建终端 / 新建 Agent 对话
- Agent 对话可选择 Runtime

---

## 4. 核心交互

| 用户任务 | 操作路径 |
|---------|---------|
| 浏览项目文件 | 侧边栏展开文件树 -> 点击文件 -> 编辑器显示内容 |
| 新建文档 | 工具栏 📄 或右键菜单 -> 创建文件 -> 开始编辑 |
| 整理文件结构 | 拖拽文件/文件夹到目标位置 |
| 对文件提问 | 右半区新建 Agent 对话 -> 上下文绑定当前文件 |
| 从 Inbox 接收素材 | Inbox 页面「移入 AreaProjects」-> 选择目标文件夹 |

---

## 5. 状态与规则

### 文件操作

- 创建文件夹：已有同名则后缀 1、2、3
- 删除：二次确认弹窗，删除后可撤销（保留 30 秒）
- 移动：拖拽或右键菜单「移动到...」

### 与 Focus 的关联

- 执行类项目在 Focus 中有对应的 WorkItem
- AreaProjects 不直接管理任务状态，只管理文件资产

## 6. 数据对象

| 对象 | 说明 | 详细定义 |
|------|------|---------|
| Project | 项目（执行类/知识类），文件夹层级的顶层单元 | [PRD §3.5](../PRD.md) |
| File | 项目内的文件/文档，默认 Markdown | -- |

---

## 7. 术语

| 术语 | 含义 |
|------|------|
| AreaProjects | 页面名称（活动栏 📁） |
| 执行类项目 | 需要拆解执行的项目，与 Focus WorkItem 关联 |
| 知识类项目 | 知识沉淀和积累，不关联任务 |
| 辅助栏 | 工作区右侧可收起的终端/Agent 对话面板 |
