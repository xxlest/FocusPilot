## 1. 定位

AreaProjects 是项目的**资产沉淀区**，承载两类项目：
1. **执行类项目**：需要拆解和执行的项目（与 Focus 的 WorkItem 关联）
2. **知识类项目**：知识沉淀和积累（文档、笔记、参考资料）

核心能力：浏览和组织项目文件 + Markdown 编辑/预览 + 知识管道可视化 + KB 卡片管理 + Anki 同步。

### 与其他页面的职责边界

| 维度 | AreaProjects | Focus | Studio | Inbox |
|------|-------------|-------|--------|-------|
| 关注点 | 资料和产物沉淀在哪里 | 计划和任务怎么推进 | 和 Agent 怎么工作 | 收集和暂存 |
| 核心操作 | 浏览、编辑、组织文件 | 拖拽、排期、状态流转 | 对话、代码、Git | 录入、分类、流出 |
| 数据形态 | 文件/文件夹 | WorkItem | CodeSession | InboxItem |
| 与本页关系 | — | Focus Task 可关联项目文件 | Studio Session 绑定项目目录 | Inbox 流出素材到 _materials/ |

**原则**：AreaProjects 管文件，Focus 管任务，Studio 管会话。三者通过项目名关联但不互相替代。

### 两类项目的差异

| | 执行类项目 | 知识类项目 |
|---|---|---|
| 典型内容 | 代码仓库、交付物、设计稿 | 学习笔记、参考资料、行业分析 |
| 与 Focus 关联 | 有对应的 WorkItem/Task | 无 |
| 知识管道 | 可选（项目附带沉淀） | 核心（主要使用场景） |
| 产出结构 | task + 交付物 + 可选 _materials/_reports/_kb/ | _materials/ + _reports/ + _kb/ |

## 2. 侧边栏

文件树导航，类似 Obsidian / VS Code Explorer。

### 2.1 整体结构

```
┌─ 侧边栏 ──────────────────┐
│  🔍 搜索文件和项目...      │
│  [📁] [📄] [⭐] [⤡]       │  ← 工具栏
│                            │
│  ── 项目 ────────────────  │
│  ▾ 📂 FocusPilot           │  ← 执行类项目
│    📄 架构设计.md           │
│    📄 技术选型.md           │
│    ▸ 📥 _materials  (3)    │  ← 知识管道文件夹，特殊图标
│    ▸ 📊 _reports           │
│    ▸ 🧠 _kb         (5)   │
│    ▸ 📂 参考资料            │
│  ▾ 📂 分布式系统笔记       │  ← 知识类项目
│    📄 CAP 定理.md           │
│    📄 Raft 协议.md          │
│    ▸ 📥 _materials  (1)    │
│    ▸ 📊 _reports           │
│    ▸ 🧠 _kb         (8)   │
│  ▸ 📂 大客户跟踪           │
│                            │
│  ── + 添加项目 ──────────  │  ← 虚线按钮
└────────────────────────────┘
```

### 2.2 工具栏

4 个图标按钮（仅图标，hover 显示文字提示）：

| 按钮 | 功能 | 说明 |
|------|------|------|
| 📁 | 新建文件夹 | 在选中位置下创建，同名自动后缀编号 |
| 📄 | 新建文件 | 默认 Markdown 格式，在选中位置下创建 |
| ⭐ | 收藏筛选 | 切换：仅显示已收藏的项目 |
| ⤡ | 展开/折叠全部 | 点击切换：全部展开 ↔ 全部折叠 |

工具栏操作的"选中位置"规则：
- 有选中文件夹 → 在该文件夹内创建
- 有选中文件 → 在该文件所在文件夹创建
- 无选中项 → 提示先选择项目

### 2.3 文件树交互

**展开/折叠**：
- 单击文件夹 → 展开/折叠切换
- 单击文件 → 编辑器打开该文件
- 展开/折叠状态按项目持久化（项目粒度，重启恢复）

**拖拽**：
- 拖拽文件/文件夹到目标文件夹 → 移动
- 拖拽到项目根目录 → 移到项目顶层
- 拖拽过程中目标文件夹高亮指示
- 拖到知识管道文件夹（_materials/_reports/_kb/）时正常移入
- 不允许拖拽项目本身（项目顺序通过右键菜单置顶调整）
- 不允许跨项目拖拽（使用右键菜单「移动到...」完成跨项目移动）

**内联重命名**：
- 双击文件/文件夹名 → 进入内联编辑模式
- Enter 确认，Esc 取消
- 知识管道文件夹（_materials/_reports/_kb/）不允许重命名

**多选**：
- Cmd+单击 → 追加选中/取消选中
- 多选后右键 → 批量操作菜单（移动到... / 删除）
- 多选不可跨项目

### 2.5 知识管道文件夹标记

知识管道的三个特殊文件夹使用区别于普通文件夹的图标和角标：

| 文件夹 | 图标 | 角标 | 含义 |
|--------|------|------|------|
| `_materials/` | 📥 | 数字（`status=new` 的素材数） | 待整合的原始素材 |
| `_reports/` | 📊 | 无 | 增量整合报告 |
| `_kb/` | 🧠 | 数字（总卡片数） | 知识卡片 |

- 角标为 0 时不显示
- 项目内无 `_materials/` `_reports/` `_kb/` 文件夹时，不自动创建、不占位显示
- 首次使用知识管道功能（如从 Inbox 流入素材）时自动创建对应文件夹

## 3. 工作区

左右分栏布局，右侧辅助栏可收起。

### 3.1 整体结构

```
┌─ 工作区 ────────────────────────────┬─ 辅助栏（⌘⇧B 收起/展开）──┐
│                                      │                            │
│ ┌─ Tab 条 ──────────────────────┐  │  [终端 | Agent]    [+] [×] │
│ │ 架构设计.md ✕ │ CAP 定理.md ✕ │  │  ─────────────────────── │
│ └────────────────────────────────┘  │                            │
│ 📂 FocusPilot / docs / 架构设计.md   │  （内容随 Tab 切换）       │
│ ──────────────── [预览|编辑]        │                            │
│                                      │                            │
│ （Markdown 内容）                    │                            │
│                                      │                            │
└──────────────────────────────────────┴────────────────────────────┘
```

### 3.2 Tab 条

- 点击侧边栏文件 → 在 Tab 条中打开（已打开则激活，未打开则新增 Tab）
- 每个 Tab 右侧 ✕ 关闭按钮
- Tab 可拖拽排序
- 未保存的文件 Tab 名称前显示圆点标记 `● 架构设计.md`
- 最后一个 Tab 关闭后显示空状态

### 3.3 面包屑导航

Tab 条下方显示当前文件的完整路径，`/` 分隔，各级可点击跳转。

```
📂 FocusPilot / docs / 架构设计.md
```

- 点击路径中的文件夹 → 侧边栏展开并高亮该文件夹
- 点击项目名 → 侧边栏收起其他项目，展开该项目根目录

### 3.4 编辑器

**模式切换**：面包屑右侧 [预览|编辑] 分段控件。

| 模式 | 行为 |
|------|------|
| **预览**（默认） | 渲染 Markdown，只读。点击编辑区域不进入编辑 |
| **编辑** | 左侧源码编辑，右侧实时预览（如工作区宽度不足则隐藏预览） |

**V1 支持的 Markdown 特性**：

| 特性 | 支持 |
|------|------|
| 标题 / 粗体 / 斜体 / 列表 / 链接 / 图片 | ✅ |
| 代码块（语法高亮） | ✅ |
| 表格 | ✅ |
| 复选框 `- [ ]` | ✅ |
| 脚注 | ✅ |
| Mermaid 图表 | V2 |
| 数学公式（KaTeX） | V2 |
| Wikilink `[[]]` | V2（兼容 VaultOne） |

**自动保存**：编辑模式下每 3 秒自动保存（无脏数据时不触发），手动 ⌘S 立即保存。

**Frontmatter 处理**：
- 预览模式：frontmatter 折叠为一行摘要（`type: project · status: planning · 3 tags`），点击展开完整 YAML
- 编辑模式：直接显示 YAML 源码

### 3.5 KB 卡片视图

当侧边栏选中 `_kb/` 文件夹时，工作区切换为**卡片列表视图**（替代普通的单文件编辑器）。

```
┌─ 知识卡片 ─────────────────────────────────────────────┐
│                                                         │
│ 🧠 FocusPilot / _kb/              [筛选 ▾] [同步 Anki] │
│ ─────────────────────────────────────────────────────  │
│                                                         │
│ ┌─ 卡片 ──────────────────────────────────────────┐   │
│ │ 🧠 CAP 定理                                      │   │
│ │ 口诀：一致可用分区，三选二                         │   │
│ │ [分布式] [CAP]          know 🟡   ✅ Anki 已同步  │   │
│ └──────────────────────────────────────────────────┘   │
│                                                         │
│ ┌─ 卡片 ──────────────────────────────────────────┐   │
│ │ 🧠 Raft 选举机制                                  │   │
│ │ 类比：班长选举，任期+多数票                        │   │
│ │ [分布式] [共识]     understand 🟢   ⏳ 待同步      │   │
│ └──────────────────────────────────────────────────┘   │
│                                                         │
│ ┌─ 卡片 ──────────────────────────────────────────┐   │
│ │ 🧠 向量数据库选型对比                              │   │
│ │ Pinecone vs Milvus vs Weaviate 三维度对比          │   │
│ │ [AI] [向量数据库]      know 🟡    ⏳ 待同步        │   │
│ └──────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**卡片行信息**：
- 第一行：卡片标题
- 第二行：口诀/类比/摘要（取 frontmatter 下第一行正文，最多 1 行）
- 第三行：标签 + mastery 等级 + Anki 同步状态

**Mastery 等级标记**：

| 等级 | 标记 | 颜色 | 含义 |
|------|------|------|------|
| know | know 🟡 | 黄色 | 知道概念 |
| understand | understand 🟢 | 绿色 | 理解原理 |
| master | master 🔵 | 蓝色 | 能应用和讲解 |

**Anki 同步状态**：

| 状态 | 显示 | 触发 |
|------|------|------|
| `anki_synced: true` | ✅ Anki 已同步 | 上次同步后无修改 |
| `anki_synced: false` | ⏳ 待同步 | 新建或修改后未同步 |

**卡片操作**：
- 单击卡片 → 编辑器（工作区下半部分）展开该卡片完整内容
- 点击 [同步 Anki] → 批量同步所有待同步卡片到 Anki（AnkiConnect API）
- [筛选] 下拉：按 mastery 等级 / 按 Anki 状态 / 按标签

### 3.6 知识管道操作

知识管道的操作入口分散在文件树和工作区中：

| 操作 | 入口 | 行为 |
|------|------|------|
| **同步整合** | _materials/ 右键菜单「🔄 同步整合」| 触发 Pipeline：新素材 → 融入 _reports/ → 提炼 _kb/ 卡片 |
| **Anki 同步** | KB 卡片视图顶部 [同步 Anki] | 将所有 `anki_synced: false` 的卡片推送到 Anki |
| **手动提炼** | 辅助栏 Agent 对话 | 对话中要求 Agent 从当前文件提炼 KB 卡片 |

**同步整合进度**：触发后工作区顶部显示进度条：

```
🔄 正在整合 3 条新素材...  ████████░░░░  2/3
```

完成后 toast 提示：「整合完成：更新 1 篇报告，新增 2 张卡片」

### 3.7 空状态

**无项目时**：

```
        📁 还没有项目

  添加已有项目目录，或新建一个空项目

           [+ 添加项目]
```

**项目内无文件时**：

```
        📄 项目为空

  创建第一个文件，或从 Inbox 导入素材

     [📄 新建文件]  [📥 从 Inbox 导入]
```

**KB 文件夹无卡片时**：

```
        🧠 还没有知识卡片

  从素材和报告中提炼知识卡片，或手动创建

  [🔄 整合提炼]  [📄 新建卡片]
```

## 4. 辅助栏（⌘⇧B 收起/展开）

工作区右侧的可收起面板，提供终端和上下文 Agent 对话两个 Tab。

### 4.1 与 Studio 对话的差异

| | AreaProjects 辅助栏 | Studio 对话区 |
|---|---|---|
| 定位 | 轻量文件问答和知识加工 | 项目级 AI 编程执行 |
| 上下文 | 当前打开的文件内容 | 整个项目目录 + Git 状态 |
| Agent | 使用通用问答 Agent | 使用 AICrew 配置的执行器 |
| 能力范围 | 解释内容、提炼卡片、总结报告 | 代码修改、Diff 审查、Git 交付 |
| 终端 | 简单终端（查看/搜索用） | 完整终端组（多实例，跟随 Session） |

### 4.2 终端 Tab

```
┌─ 辅助栏 ──────────────────────┐
│  [终端·] [Agent]     [+] [×] │
│  ──────────────────────────  │
│                                │
│  $ rg "Knowledge Pipeline" .   │
│  docs/PRD.md:317               │
│  docs/PRD.md:319               │
│                                │
│  $                             │
└────────────────────────────────┘
```

- 工作目录：自动设为当前侧边栏选中的项目根目录
- 切换项目 → 终端不销毁，但新终端跟随新项目路径
- [+] 新建终端实例（Tab 形式）
- [×] 关闭当前终端

### 4.3 Agent 对话 Tab

```
┌─ 辅助栏 ──────────────────────┐
│  [终端] [Agent·]     [+] [×] │
│  ──────────────────────────  │
│                                │
│  📎 上下文: 架构设计.md        │  ← 自动绑定当前文件
│                                │
│  👤 这篇文档的核心观点是什么？  │
│                                │
│  🤖 本文档描述了项目产出的      │
│     通用结构，核心要点：        │
│     1. 每个层级都包含...       │
│                                │
│  ┌─────────────────────────┐  │
│  │ 输入问题...    [↑ 发送] │  │

└────────────────────────────────┘
```

- 自动绑定当前编辑器打开的文件为上下文
- 切换编辑器文件 → 上下文自动切换（对话历史保留，但新消息关联新文件）
- 典型用途：解释文档内容、从素材提炼 KB 卡片、生成报告章节

## 5. 核心交互

| 用户任务 | 操作路径 |
|---------|---------|
| 浏览项目文件 | 侧边栏展开文件树 → 点击文件 → 编辑器显示内容 |
| 新建文档 | 工具栏 📄 或右键菜单 → 创建文件 → 自动进入编辑模式 |
| 编辑文档 | 编辑器点击 [编辑] → 左侧源码右侧预览 → ⌘S 保存 |
| 整理文件结构 | 拖拽文件/文件夹到目标位置 / 右键「移动到...」 |
| 跨项目移动文件 | 右键「移动到...」→ 弹出项目+文件夹选择器 → 确认 |
| 浏览 KB 卡片 | 侧边栏点击 _kb/ → 工作区切换为卡片列表 → 点击卡片查看 |
| 同步到 Anki | KB 卡片视图 → [同步 Anki] → 批量推送 |
| 触发知识管道 | _materials/ 右键「🔄 同步整合」→ 自动加工 → 更新 reports + kb |
| 对文件提问 | 辅助栏 Agent Tab → 上下文自动绑定当前文件 → 输入问题 |
| 从 Inbox 接收素材 | Inbox 页面「移入 AreaProjects」→ 选择目标项目 → 存入 _materials/ |
| 搜索文件 | 侧边栏搜索框 → 输入关键词 → 结果分项目/文件两区 → 点击定位 |

## 6. 状态与规则

### 6.1 文件操作

| 操作 | 规则 |
|------|------|
| 创建文件/文件夹 | 同名自动后缀编号（`文件.md` → `文件 1.md`） |
| 删除文件 | 单条：无确认弹窗，toast「已删除」+ [撤销]（5 秒）|
| 删除文件夹 | 非空文件夹：确认弹窗「文件夹内有 N 个文件，确定删除？」；空文件夹同单条 |
| 批量删除 | 确认弹窗「确定删除 N 项？」→ toast + [撤销]（5 秒）|
| 移动文件 | 拖拽或右键「移动到...」，目标同名时提示覆盖/重命名 |
| 重命名 | 双击或 F2 内联编辑；重名时拒绝并提示 |

### 6.2 项目管理

| 操作 | 规则 |
|------|------|
| 添加项目 | 指向已有目录或新建空项目，不允许嵌套（项目目录不能包含另一个项目） |
| 移除项目 | 仅从列表中移除，不删除磁盘文件。确认弹窗提示「移除后文件仍保留在磁盘」 |
| 项目排序 | 默认按添加时间排序；右键「📌 置顶」置顶；置顶项按置顶时间排序 |
| 收藏 | 项目级收藏，⭐ 工具栏可按收藏筛选 |

### 6.3 知识管道规则

| 规则 | 说明 |
|------|------|
| 新素材标记 | 从 Inbox 流入或手动放入 _materials/ 的文件，frontmatter `status: new` |
| 整合触发 | 手动触发「🔄 同步整合」或 `auto_sync: true` 时自动触发 |
| 整合范围 | 仅处理 `status: new` 的素材，已整合的不重复处理 |
| 报告更新 | 增量融入 _reports/ 现有文档，不覆盖用户手动编辑内容 |
| 卡片提炼 | 从新内容中提炼原子化 KB 卡片，frontmatter 默认 `mastery: know`、`anki_synced: false` |
| Anki 同步 | 通过 AnkiConnect API（localhost:8765）推送；失败时 toast 提示并保持 `anki_synced: false` |

### 6.4 Inbox 流入规则

Inbox 侧定义了完整的流出规则（见 [02-inbox.md §5.4](02-inbox.md)），AreaProjects 侧的接收行为：

| 步骤 | 行为 |
|------|------|
| 用户在 Inbox 选择「移入 AreaProjects」| 弹出确认面板，选择目标项目 |
| 确认移入 | 在目标项目 `_materials/` 下创建文件（`{YYYY-MM-DD}-{slug}.md`） |
| 文件树刷新 | _materials/ 角标数 +1，文件树自动展开到新文件位置 |
| 无 _materials/ 时 | 自动创建 _materials/ 文件夹 |

## 7. 数据对象

| 对象 | 说明 | 详细定义 |
|------|------|---------|
| Project | 项目（执行类/知识类），文件夹层级的顶层单元 | [PRD §4.1](../PRD.md) |
| File | 项目内的文件/文档，默认 Markdown | — |
| KBCard | 知识卡片，_kb/ 下的特殊 Markdown 文件 | 见下方 |
| Material | 原始素材，_materials/ 下的文件 | 见下方 |

### 7.1 KBCard Frontmatter

```yaml
---
type: kb-card
source: _reports/chapter-1.md       # 提炼来源
tags: [分布式, CAP]
anki_deck: "分布式系统"              # Anki 牌组名
anki_synced: false                   # Anki 同步状态
mastery: know                        # know → understand → master
---
## CAP 定理
**口诀**：一致可用分区，三选二
**类比**：像三个人传话，要么慢(C)，要么可能传错(A)，但电话线断了(P)总会发生
**费曼检验**：能否用一句话给外行解释 CAP？
```

### 7.2 Material Frontmatter

```yaml
---
type: material
status: new                          # new → integrated
source_type: inbox                   # inbox | manual | crew_collect
source_ref: "inbox_item_abc"         # 来源引用（Inbox 流入时记录）
integrated_at: null                  # 整合时间
---
```

## 8. 术语

| 术语 | 含义 |
|------|------|
| AreaProjects | 页面名称（活动栏 📁），项目资产沉淀区 |
| 执行类项目 | 需要拆解执行的项目，与 Focus WorkItem 关联 |
| 知识类项目 | 知识沉淀和积累，不关联任务 |
| 辅助栏 | 工作区右侧可收起的终端/Agent 对话面板（⌘⇧B 切换） |
| 知识管道 | _materials/ → _reports/ → _kb/ → Anki 的完整加工链路 |
| 同步整合 | 触发知识管道的操作：新素材融入报告 + 提炼卡片 |
| KB 卡片 | 知识卡片，原子化要点，_kb/ 文件夹下的特殊 Markdown 文件 |
| Mastery | 掌握层次：know（知道）→ understand（理解）→ master（精通） |
| 卡片视图 | 选中 _kb/ 文件夹时，工作区从编辑器切换为卡片列表展示 |
