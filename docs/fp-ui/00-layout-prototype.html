项目产出结构

所有项目都交付产物，同时积累知识沉淀。FocusPilot 采用分层存储模型，每个 Project 下统一包含 task、_materials、_kb 和 _reports 四个标准目录，确保产出可追溯、知识可复用。

FocusPilot/ ├── task-001-重构架构.md ├── task-002-性能优化.md ├── _materials/ │ ├── RAG工程化实践.md │ ├── 向量数据库对比.md │ └── 分布式共识论文.pdf ├── _kb/ │ ├── CAP 定理.md │ ├── Raft 选举机制.md │ └── CRDT 数据结构.md └── _reports/ ├── 周报-W22.md └── 月报-05.md

知识管道（Knowledge Pipeline）

知识管道将原始素材逐步加工为结构化知识卡片，分五个阶段推进：

阶段 目录 说明 产出
