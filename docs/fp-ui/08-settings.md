## 1. 定位

Settings 是全局配置中心，管理用户偏好和各功能模块的默认行为。

一句话职责：Settings 管**全局默认值和个人偏好**，具体 Agent 配置去 AICrew。

### 与 AICrew 的职责边界

| 维度 | Settings | AICrew |
|------|----------|--------|
| 管理对象 | 全局偏好和默认值 | Agent 成员的详细配置 |
| Agent 相关 | 引用默认 Agent、全局并发上限 | 管理每个 Agent 的角色/能力/MCP |
| 典型问题 | "默认派给哪个 Agent？" | "代码工程师能不能访问 GitHub？" |

**硬规则**：Settings 管全局默认值和偏好，具体 Agent 配置去 AICrew，不重复。

## 4. 核心交互

| 用户任务 | 操作路径 | 规则 |
|----------|----------|------|
| 修改个人资料 | 个人资料 → 编辑 → 更新资料 | 点击按钮提交保存 |
| 切换主题 | 偏好设置 → 点击主题卡片 | 即时预览，无需手动保存 |
| 配置通知 | 通知 → toggle 开关 | 即时生效 |
| 录入快捷键 | 快捷键 → 点击当前值 → 按下新组合键 | 冲突时红色提示，需先解决冲突 |
| 查看 Daemon 状态 | Daemon → 诊断信息 | 只读展示 |
| 检查更新 | 更新 → 立即检查 | 触发检查，有更新时提示重启 |
| 修改默认 Agent | Focus / Studio → 默认 Agent select | 即时生效，引用 AICrew 成员 |
| 跳转 AICrew | AICrew → 前往 AICrew → | 切换到 AICrew 页面 |

## 5. 状态与规则

### 保存行为

- 偏好设置、通知、快捷键、Focus/Studio/AICrew 配置：**即时生效**，无需手动保存
- 个人资料：**按钮提交**保存（因为 textarea 内容较长，避免频繁写入）
- 主题切换：**即时预览**

### 快捷键冲突检测

- 录入新组合键时，检查是否与已有快捷键冲突
- 冲突时该行显示红色 pill："与「显示/隐藏面板」冲突"
- 冲突状态下不保存，用户需修改为不冲突的组合键

### V1 不做

| 项目 | 原因 |
|------|------|
| API Token | 个人版无云端 API |
| GitHub 集成 | V1 纯本地 |
| 集成/实验室 | 无第三方集成和实验功能 |
| 成员管理 | 个人版无多用户 |
| 代码仓库 | 项目管理在 AreaProjects |
| 项目模式（Agile/Flow/Lite/Free） | 无页面承载此功能 |
