## 1. 定位

Home 是 FocusPilot 的**收件箱驱动处理中心** + **全局概览**，是用户打开主面板后的默认落地页。

### 核心场景

用户在 Studio / 项目里向 AI 发出请求（任务执行、规划、修复），请求需要处理时间，用户会切走干别的。**回到 Home 第一眼要知道：哪些任务回来了？哪些等我拍板？** Home 把所有"任务状态变更通知"汇聚成一个收件箱，点开即处理。

### 与 Studio 的职责边界（硬规则）

| 维度 | Home | Studio |
|------|------|-------|
| 定位 | 收件箱（通知汇聚 + 详情查看 + 回复）+ 概览 | 规划 + 执行 + 评估 + 验收 |
| 数据展示 | 通知流（按状态/项目筛选）+ 聚合计数 | 完整任务列表 + 筛选 + 排序 + 分组 |
| 允许操作 | 查看详情、回复 Agent（触发下一轮）、标已读、跳转 | 新建、拖拽、改状态、批量处理、字段编辑 |
| 不允许操作 | 新建 Task、拖拽改状态、批量处理、内联编辑字段 | -- |

**原则**：Home 是「收件箱 + 导航层」，不是第二个 Studio。看到通知、点进去看 / 回复，重操作回 Studio。

### 不再承载自由对话

Home **不提供独立的对话入口**（移除旧版「💬 对话」Tab）。自由对话统一由**右下角快捷助手**承载（常驻、与 Studio 共享同一 `StudioSession`）。Home 专注收件箱单一职责（符合「克制」准则）。

## 2. 整体布局：三栏 master-detail

```
┌──A──┬──────── B 侧边栏 = 列表面板 ────────┬──────── C 工作区 = 详情面板 ────────┐
│ 🏠  │ 收件箱  17 · 5 未读            ⋯   │                                      │
│ 📁  │ ─────────────────────────────────│        默认：概览 dashboard          │
│ 🛠  │ [全部17] [只看未读] [⊟ 时间▾]      │      （屏幕中央、放大的 4 张卡）       │
│ ✅  │ ········ 淡灰分割线 ········       │                                      │
│ 🤖  │ ◌待规划1 ○待办2 ●进行中3 ●审核中6  │        点消息标题 → 替换为详情        │
│     │ ●已完成1 ●已阻塞2                  │      （ExecutionRun 时间线 + 元数据）  │
│ ⚙  │ ═════════════════════════════════ │                                      │
│     │ ✳ 上海北京天气  ●审核中  3天       │                                      │
│     │ ✳ 北京离上海多远 ●审核中 3天       │                                      │
│     │ …（消息列表，主滚动区）            │                                      │
└─────┴────────────────────────────────────┴──────────────────────────────────────┘
```

| 栏 | 角色 | 类比 |
|----|------|------|
| A 活动栏 | 一级导航（🏠 Home 选中） | -- |
| B 侧边栏 | **列表面板**：标题 + 筛选/分组 + 消息列表 | 邮件的文件夹 + 邮件列表 |
| C 工作区 | **详情面板**：默认概览，点消息显详情 | 邮件的阅读窗格 |

## 3. 侧边栏（B 栏）= 列表面板

所有列表相关元素都收纳在侧边栏。从上到下：

### 3.1 标题栏

```
收件箱   17 · 5 未读                ⋯
```

- 标题 `收件箱` + 计数 `总数 · 未读数`（未读数 accent/蓝高亮）
- `⋯`：更多菜单（当前含「全部标为已读」）

### 3.2 第一行 控制条

```
[全部 17]   [● 只看未读]   [⊟ 时间 ▾]
```

- **全部**：状态单选的"全部"项（默认选中），点击清除状态过滤
- **只看未读**：开关，叠加在任意状态过滤之上
- **分组 ▾**：下拉，顶部灰色分组标题「分组方式」，选项 `无（按时间倒序）` / `按项目`

### 3.3 淡灰分割线

控制行与状态行之间用 `--line-soft` 淡灰细线分隔（比列表前的主分隔线更轻）。

### 3.4 第二行 状态 tab（6 态单选）

```
◌待规划1  ○待办2  ●进行中3  ●审核中6  ●已完成1  ●已阻塞2
```

- 6 个状态各一个小 tab，带状态彩点 + 计数
- **单选**：点某个 tab 只显示该状态的消息；与「全部」互斥（选状态时「全部」取消高亮）
- 轻量选中态（淡底色高亮），不是重的描边/灰显多选
- 状态色对齐浮球固定调色板（见 §6）

### 3.5 消息列表（主滚动区）

- 每条 = **FocusPilot 禅圆 logo 头像** + 标题 + 副文本 + 状态图标 + 时间
- 未读：标题前蓝点 + 加粗；已读：标题灰、无蓝点
- 选中：左侧 accent 色条 + 行高亮
- 副文本 = "状态设为 X" 或 Agent 回流内容摘要
- **置顶**：行内悬停显示 📌 图钉，点击置顶/取消（已置顶则常亮 accent 色）。置顶消息独立成 **「📌 置顶」一栏浮在列表最上方**，不随时间/分组下沉；仍受当前状态/未读筛选约束，组内按时间倒序。置顶栏与下方非置顶之间用**一条灰色细分隔线**分隔
- `分组 = 按项目` 时，列表插入项目分组标题；组内仍按时间倒序（置顶栏始终在所有分组之上）

## 4. 工作区（C 栏）= 详情面板

工作区有两个态：**概览（默认）** 和 **详情**，在工作区内整块切换（非抽屉、非弹窗）。

### 4.1 概览 dashboard（默认态）

未选中任何消息时，工作区**顶对齐、可滚动**显示概览：

```
              ☀ 早上好，Bruce · 6月13日 周六

  [📁 项目5] [📚待阅读5] [💡灵感2] [📌临时3]   ← pill 紧贴日期下方

      ┌────┐   ┌────┐   ┌────┐
      │ 4  │   │ 9  │   │ 28 │      ← 3 张大卡 = scope 选择器（单选，默认今日聚焦）
      │今日│   │本周│   │本月│
      │聚焦│   │计划│   │计划│
      └────┘   └────┘   └────┘

  ┌─ 🔥 今日聚焦                          4 项 ─┐  ← 跟随选中卡片切换内容
  │ ● 番茄钟时长可配置        FocusPilot · 进行中 │
  │ ● 上海和北京的天气        天气查询 · 审核中   │
  │ ● 缓存层重构              FocusPilot · 已阻塞 │
  │ ● 用 slides 介绍 Multica  竞品调研 · 进行中   │
  └──────────────────────────────────────────────┘

        左侧收件箱有 5 条未读 · 点击任意消息查看详情
```

- **一排 pill（紧贴日期下方）**：活跃项目数 / 待阅读 / 灵感 / 临时事项（点击跳转 Projects 对应区）
- **3 张大卡 = scope 选择器**：今日聚焦 / 本周计划 / 本月计划，与 Studio 三个时间 scope 对应。**单选**（默认「今日聚焦」高亮），点某张卡 → 下方任务列表切到该 scope 的内容（今日聚焦 ⊂ 本周计划 ⊂ 本月计划）
- **任务列表**：跟随选中卡片显示该 scope 的任务（状态色点 + 任务名 + 归属项目·状态）。超出展示数时尾部提示「…还有 N 项 · 去 Studio 查看全部」
- **点击任务行 → 打开该任务的详情（对话框/ExecutionRun），同时在左侧收件箱高亮选中并滚动到对应消息**（与点收件箱消息同一行为）
- 底部提示当前未读数

### 4.2 详情态（点消息标题触发）

点击侧边栏消息标题 → 概览整块替换为该消息的详情：

```
┌─ 工作区 ───────────────────────────────────────────────┐
│ FocusPilot › FP-001 数据模型设计      ↑ ↓     ● 已完成 │ ← 面包屑 + 翻条 + 状态pill
│ ─────────────────────────────────────────────────────│
│ [任务描述]                                             │
│ 🤖 代码工程师 · 自动执行(第1轮) · 00:08:12             │
│    [代码摘要 + Build/tests] [查看Diff][日志]           │
│ 🔍 质量审查员 · 评估 第1轮/共3轮 [2条意见]             │
│    Critical … / Important …                            │
│ 🤖 代码工程师 · 自动修复(第2轮) …                      │
│ 👤 你 · 回复=触发下一轮 …                              │  ← 右侧元数据栏：
│ ✅ 你已验收通过 · 拖入「已完成」                        │     状态/优先级/项目/
│ ─────────────────────────────────────────────────────│     归属目标/调度/Agent/截止
│ [让 Agent 继续调整…（回复即触发下一轮执行）] [↑发送]   │
└─────────────────────────────────────────────────────────┘
```

- **详情 = 复用 Studio 的 ExecutionRun 面板**（执行时间线 + 右侧元数据 + 底部回复框），不另造组件
- **顶栏面包屑**：`项目 › 编号 标题`（项目名 dim 色可点 → 真实产品跳到该项目；原型中返回概览）。**不使用 ✕**
- **翻条**：`↑ / ↓` 在当前筛选结果内翻到上/下一条；键盘 `↑/↓` 同效
- **状态 pill**：随消息状态变色
- **回复**：底部输入框，回复即触发 Agent 下一轮执行
- **返回概览**：`Esc`，或点面包屑项目名，或再次点击侧边栏已选中的消息

## 5. 消息（通知）模型

收件箱里每条 = 一次**任务状态变更通知**（如"状态设为 审核中"，或 Agent 回流内容摘要）。

### 5.1 消息态：只有「已读 / 未读」

- 消息层**只有已读/未读两态**，没有"待回复"等其它消息状态
- "是否需要我处理"由消息引用的**任务状态**（看板 6 态）表达，不在消息层重复建模
- 点未读消息 → **立即标已读**（蓝点消失、计数更新）+ 打开详情
- 点已读消息 → 直接打开详情（无状态变化）
- `⋯ → 全部标为已读`：一键清零未读

### 5.2 筛选 / 排序 / 分组（三维独立）

| 维度 | 选项 | 默认 | 说明 |
|------|------|------|------|
| 状态过滤 | 全部 / 6 态之一（单选） | 全部 | 侧边栏状态 tab |
| 未读过滤 | 只看未读 开关 | 关 | 叠加在状态过滤上 |
| 分组 | 无（按时间倒序）/ 按项目 | 无 | 时间倒序始终生效；分组后组内仍按时间 |

## 6. 数据对象与口径

### 6.1 收件箱消息（Notification）

| 字段 | 说明 |
|------|------|
| taskId | 引用的任务（点击打开其 ExecutionRun 详情） |
| title | 任务标题 |
| summary | 副文本（"状态设为 X" 或 Agent 回流摘要） |
| taskStatus | 引用任务的看板状态（决定状态图标与状态 tab 归类） |
| project | 归属项目（用于「按项目」分组） |
| timestamp | 时间（列表按此倒序） |
| read | 已读 / 未读（**单一字段**，不再 read+unread 双存） |
| pinned | 是否置顶（置顶消息浮到「📌 置顶」栏顶部） |

### 6.2 状态色（权威源：浮球 STATUS_MODEL，固定 hex、主题无关）

对齐 commit a1dff70「进行中→蓝（对齐 Studio）」+ `floating-ball-options-prototype.html:2117`；母版 `00-layout taskStatusDefs` 已同步为同值：

| 状态 | key | 固定色 | 图标 |
|------|-----|--------|------|
| 待规划 | backlog | `#999ca2` 灰 | ◌ |
| 待办 | todo | `#74777d` 灰 | ○ |
| 进行中 | in_progress | `#147af3` 蓝 | ◔ |
| 审核中 | in_review | `#8b5cf6` 紫 | ◑ |
| 已完成 | done | `#31c75a` 绿 | ✓ |
| 已阻塞 | blocked | `#f97316` 橙 | ⊘ |

> 状态语义色**不跟随主题 accent**（避免默认白主题下"进行中"变红）。

### 6.3 概览指标口径

| 指标 | 口径 | 跳转目标 |
|------|------|---------|
| 今日聚焦 | `scheduled_date = today` 且 `status ∉ {done, cancelled}` | Studio 全局视图，筛选今日 |
| 本周待办 | `scheduled_date ∈ 本周` 且 `status ∉ {done, cancelled}` | Studio 全局视图，筛选本周 |
| 本周计划 | 本周内的目标/里程碑数 | Studio 全局视图，本周计划 |
| 本月计划 | `schedule = month` 的项目级目标 | Studio 全局视图，筛选本月 |
| 项目 | `lifecycle = active` 的活跃项目数 | Projects（Projects Tab） |
| 待阅读 / 灵感 / 临时事项 | 采集箱三类计数 | Projects（Inbox Tab，见 05-area-projects §7） |

## 7. 核心交互

| 用户任务 | 操作路径 |
|---------|---------|
| 看哪些任务回来了 / 等我拍板 | 打开 Home → 侧边栏收件箱（默认全部·时间倒序） |
| 只看某一状态 | 侧边栏点状态 tab（单选） |
| 只看未读 | 点「只看未读」 |
| 按项目归类 | 分组 ▾ → 按项目 |
| 查看 / 处理一条通知 | 点消息 → 工作区显详情 → 看时间线 / 回复 Agent |
| 连续处理多条 | 详情顶栏 `↑ / ↓` 翻条（或键盘 ↑/↓） |
| 标记已读 | 点未读消息即自动已读；或 `⋯ → 全部标为已读` |
| 回到概览 | `Esc` / 点面包屑项目名 / 再点已选中消息 |
| 看今日工作全貌 | Home 默认态即概览 dashboard |
| 发起自由对话 | 右下角快捷助手（Home 不再提供对话入口） |

## 8. 状态与规则

### 操作边界（硬规则）

Home **允许**：查看详情、回复 Agent（触发下一轮）、标已读、状态/未读/分组筛选、点指标/pill 跳转。

Home **不允许**：新建 Task / WorkItem、拖拽改状态或排序、批量操作、内联编辑任务字段（这些回 Studio 做）。

> "回复 Agent" 与"标已读"属轻量收尾操作，不违反"不改看板状态"——回复本质是对话，标已读只改消息层。

### 视图切换规则

- 选中消息 → 详情态；未选 / 返回 → 概览态
- **概览是默认态**：详情只是处理某条消息时的临时态，不跨页面持久
- **每次进入 Home 页（点 🏠）一律回到概览**，并关闭上次打开的详情；想继续看某条消息再从收件箱点开

## 9. 术语

| 术语 | 含义 |
|------|------|
| Home | 页面名称（活动栏 🏠） |
| 收件箱 | Home 的核心——任务状态变更通知的汇聚列表 |
| 消息 / 通知 | 收件箱里一条，引用一个任务，含已读/未读 |
| 概览 dashboard | 工作区默认态（4 卡 + pills） |
| 详情态 | 工作区显示某条消息的 ExecutionRun 详情 |
| 今日聚焦 | `scheduled_date = today` 的未完成任务 |
| 采集箱 | 待阅读/灵感/临时事项三类（在 Projects Inbox Tab） |
