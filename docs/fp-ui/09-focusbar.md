## 1. 定位

悬浮层是 FocusPilot 常驻在**任意桌面、任意 App 之上**的系统级入口与监控层，独立于六个一级页面，不占用活动栏。它由三个部件组成：

| 部件 | 角色 | 触发 |
|------|------|------|
| **悬浮球** `.ball` | 常驻入口、未读徽标、专注/休息状态环 | 始终可见（可在设置关闭） |
| **快捷面板** `.float-panel` | 规划速览 + Focus Bar 开关 + 对话入口 | hover 悬浮球 / 面板，或点击悬浮球锁定 |
| **Focus Bar** `.focus-bar` | 顶部实时状态条 + 番茄计时器 | "实时状态"开关 / 设置默认置顶 / 进入专注模式 |

**职责边界**：悬浮层只做"速览 + 快捷操作 + 状态监控 + 计时"，**不承载深度编辑**。任何需要展开操作的任务一律跳转到主看板（Studio）或对话视图。与"全局快捷对话助手"（聊天窗口 `#chatWindow`）的关系：快捷面板底部"问问 FocusPilot"是它的入口，聊天窗口本体规格见 [00-layout.md §全局快捷对话助手](00-layout.md)，本文不重复。

> **三层交互**：悬浮球（常驻入口）→ 快捷面板（hover/单击弹出）→ 主看板（双击/Dock 图标）。本文聚焦前两层 + Focus Bar；主看板见 [04-studio.md](04-studio.md)。

## 2. 形态结构（三层部件）

悬浮层没有侧边栏；其结构是三个相对独立、可分别开关的部件，统一挂在 `.fb-root` 下。

```
┌──────────────────────────── 屏幕顶部 ───────────────────────────┐
│ ▰ Focus Bar  [未读6][待规划3│进行中2│审核中2│已完成2][今日聚焦5] │ ← 状态条（可置顶）
│                              🍅启动专注▾   ▱▱▱进度▱▱▱        ×  │
└─────────────────────────────────────────────────────────────────┘



                                          ┌──── 快捷面板 ────┐
                                          │ 顶 实时状态  ⦿开关 │
                                          │ 今 今日聚焦   3 › ──┼──┐
                                          │ 周 本周计划   4 › │  │ ┌── 二级菜单 ──┐
                                          │ 月 本月计划   3 › │  └→│今日聚焦 [状][项]│
                                          │ 全 全局规划  10 › │    │• 任务A     进行中│
                                          │ ─────────────── │    │• 任务B     待办  │
                                          │ 💬 问问 FocusPilot │    │  ＋ 新建任务     │
                                          └──────────────────┘    └────────────────┘
                                                            ●  ← 悬浮球（含未读徽标/状态环）
```

- **悬浮球**：`width/height 38px`、正圆（`border-radius:50%`）、红色玻璃径向渐变 + 内禅圆；右上 `ball-badge` 未读数；`ball-ring` 在专注（红）/休息（绿）时显示。锁定态变琥珀色（见 §4）。
- **快捷面板**：宽 312px，浮在悬浮球左侧；动作行 5 项 + 底部对话入口（chat-strip）。
- **Focus Bar**：贴屏幕顶部的横向状态条，6 个分组下拉 + 计时器 + 进度条 + 关闭按钮。

精确色值/尺寸/动画参数以原型与 [DesignGuide §6.3 / §7.3](../DesignGuide.md) 为准。

## 3. 区域结构

### 3.1 快捷面板动作行

| 动作行 | 图标 | 作用 | 二级菜单 |
|--------|:---:|------|---------|
| **实时状态** | 顶 | 开关 Focus Bar（等价 `bar-pinned`）；副标题随开关切换文案 | 无 |
| **今日聚焦** | 今 | 今日规划清单速览 / 新建 / 进详情 | 有（规划清单） |
| **本周计划** | 周 | 本周规划清单 | 有 |
| **本月计划** | 月 | 本月规划清单 | 有 |
| **全局规划** | 全 | 全部规划清单（= 今日∪本周∪本月并集） | 有 |
| **问问 FocusPilot** | 💬 | 打开全局快捷对话助手（`#chatWindow`） | 无 |

每个规划行右侧 `qp-count` 显示该 scope 任务数（由数据派生，恒等于清单条数）。

### 3.2 规划清单二级菜单

- 标题 + 分组切换钮 `[按状态][按项目]`；下方为 `plan-list`（任务行）；底部 `＋ 新建任务`。
- 任务行 `plan-item`：状态圆点（按状态色）+ 标题 + 状态文字；点击 → `openRunDetail(id)` 进入对话详情做后续操作（验收/对话/下发指令/删除）。

### 3.3 Focus Bar 分组下拉

6 个 `bar-group`，每个下拉含标题 + `[按状态][按项目]` 分组钮 + `bar-list`（`mini-task` 列表）：

| 下拉 | tone | 内容（由池按状态/未读派生） |
|------|------|------|
| 未读 | unread | 收件箱未读项（红色徽标） |
| 待规划 | plan | 状态 ∈ {待规划, 待办} |
| 进行中 | running | 状态 = 进行中 |
| 审核中 | review | 状态 = 审核中 |
| 已完成 | done | 状态 = 已完成 |
| 今日聚焦 | focus | = 今日规划成员 |

`bar-num`（标题数字）恒等于下拉列表条数（由派生计数保证一致）。

### 3.4 Focus Bar 计时器

- 闲时：`🍅 启动专注 ▾` + 下拉（▶ 工作 / 休息）。
- 专注态（`data-mode="focus"`）：显示"工作中 32:38 / 35:00"，悬浮球红环。
- 休息态（`data-mode="rest"`，`.resting`）：显示"闭眼远眺 · 1/5"，悬浮球绿环。
- 右侧 `bar-progress` 进度条；最右 `×` 关闭 Focus Bar 回常规视图。

### 3.5 状态数字徽章（醒目度 · 方案E 智能分级）

6 个 `bar-num` 不再是「彩色裸数字」，而是按**行动优先级分级的徽章**——让状态条一眼能区分「该先看哪」，而非六个一起喊：

| 档位 | 适用下拉 | 样式 | 设计意图 |
|------|---------|------|---------|
| **实心徽章** | 未读、今日聚焦 | 实心状态色底 + 白字 + 轻投影 | 把「最响」的视觉额度只给真正要你**行动**的两类：未读=待处理、今日聚焦=今天要推进 |
| **柔和底色徽章** | 待规划 / 进行中 / 审核中 / 已完成 | 状态色 @15% 底 + 状态色字 | 生命周期态只做轻量强调，成块但不吵，守住「克制」基调 |
| **灰字（0 态）** | 任意下拉计数 = 0（`.is-zero`） | 灰色裸数字、无底色/描边/投影 | 不出现「空徽章」；0 是「没有」而非「需要关注」，刻意弱化 |

- **徽章色取各态状态色**（`--tone`，与看板/浮球同源）：未读红 `--red`、今日聚焦/进行中蓝 `--blue`、待规划灰 `#6b7280`、审核中紫 `#8B5CF6`、已完成绿 `#1f8f43`。色值/几何参数见 [DesignGuide §4.9](../DesignGuide.md)。
- **已去掉未读前导红点**（旧 `.bar-dot`）：实心红徽章本身已是更强的「有未读」信号，红点冗余，故移除。
- **0 态由 `fbFillBar` 派生时挂 `.is-zero`**：计数仍写入 `textContent`（自检 I2「计数一致」不受影响），仅视觉退化；护栏见 §6.2 I9。
- **为何不全量实心**：6 颗实心色块并排像「红绿灯墙」，与克制基调冲突；分级让醒目度**有层次**。其它被否档位（全量实心 / 描边圆环 / 纯放大字号 / 主次反转）经并排对比后排除。
- **审核中色已统一为紫**：徽章 / 状态条与看板、[DesignGuide §4.8](../DesignGuide.md) 一并使用紫 `#8B5CF6`。早期 Focus Bar 曾用琥珀 `#b4690e`，因与执行轴「排队」琥珀 `#E6A23C`、生命周期「已阻塞」橙 `#F97316` 撞暖色、且偏离权威而收敛为紫——**踩坑留痕：暖色会让两轴/邻态难以区分**。（浮层详情 `rd-statuspill[data-tone=review]` 早已是紫 `#6d3bd4` 文字，可印证。）

## 4. 核心交互

### 4.1 快捷面板的弹出与收起

| 任务 | 操作路径 |
|------|---------|
| 打开面板 | hover **悬浮球本体**（`.ball`，见下「感应区」行）→ `floating-zone.open`，淡入上移。**原型 hover 立即弹；真机实现应保留 ~150ms hover 延迟（V4 `hoverDelay`）防贴边/路过误触** |
| 感应区 | 触发弹出的热区**仅为悬浮球可见本体**，**不含"未弹出的面板占位区"**。实现：`.float-panel` 默认 `pointer-events:none`，仅在 `.floating-zone:hover` / `.open`（hover 球本体或锁定）时恢复 `auto`——弹出后能移到面板上操作，但**鼠标在球外空白（含面板尚未弹出的占位区）移动不会误触发弹出**。⚠️ 切勿把面板占位区设为常驻 `pointer-events:auto`（面板 `opacity:0` 仍会拦截鼠标，曾导致"鼠标还没碰到球就弹面板"） |
| 一级 → 二级菜单 | hover 动作行 → 二级菜单从左侧弹出；一/二级间有**透明桥接热区**（`.sub-menu::after`，14px）覆盖 10px 间隙，移动途中不脱离 hover |
| 收起 | 鼠标离开 floatStack / floatPanel → `scheduleFloatingClose`：**320ms hover-intent 延迟**后移除 `.open`，再 ~150ms 淡出；锁定 / `fbAutoCollapse=false` 时不收起（详见下「收起验收口径」） |
| **锁定常驻** | **点击悬浮球** → 切换 `fb-locked`：悬浮球变琥珀色 + 面板保持打开 + 不自动收起；**再点解除** |

> **收起验收口径**：离开后先 ~320ms hover-intent 延迟（`scheduleFloatingClose` 移除 `.open`），再 ~150ms 淡出（`.float-menu` opacity→0）。**判定"已收起"看延迟 + 动画结束后的最终态**：`.float-menu` `opacity:0` 且 `.float-panel` 不可命中；过渡期间采样到接近 1 的 opacity 属正常动画帧，不算"未收起"。

### 4.2 分组切换（快捷面板二级菜单 & Focus Bar 下拉通用）

- 点 `[按状态]` → 按生命周期状态分组；点 `[按项目]` → 按项目分组；**再点已选项 = 取消 = 回扁平**。
- 分组标题 `ur-head` 用**专用青绿强调色 `#0d9488`** + 加粗 + 顶部分隔线，**刻意区别于任务行的灰色文字**，避免分组名与清单内容撞色。
- 各 scope / 各下拉的分组状态**相互独立**。

### 4.3 Focus Bar 开关与钉住

| 任务 | 操作路径 |
|------|---------|
| 开/关 Focus Bar | 快捷面板"实时状态"行，或设置"默认置顶显示"（`bar-pinned`） |
| 钉住某下拉 | 点下拉标题 `bar-btn` → 该组 `pinned` 常开 + 高亮；再点取消；点条外取消 |
| 进入专注/休息 | 计时器 → 工作 / 休息，整条进入对应模式 |
| 关闭 | Focus Bar 最右 `×` → 回常规视图（清 `bar-pinned`/`timer-running`） |

### 4.4 悬浮球与 Focus Bar 解耦

- "显示悬浮球"开关（`fbToggle` → `fb-hidden`）**只隐藏悬浮球本体及其快捷面板**，不连带 Focus Bar。
- Focus Bar 由 `bar-pinned` 独立控制；**悬浮球关闭后仍可单独使用 Focus Bar**，两者分别开关、互不牵连。

## 5. 状态与规则

### 5.1 关键 class / 状态

| class / 标志 | 含义 |
|------|------|
| `floating-zone.open` | 快捷面板展开 |
| `fb-locked` | 悬浮球锁定（琥珀色 + 面板常驻 + 不自动收起） |
| `fb-hidden` | 悬浮球本体隐藏（不影响 Focus Bar） |
| `bar-pinned` | Focus Bar 置顶显示 |
| `timer-running` + `data-mode=focus/rest` | 计时进行中（专注 / 休息） |
| `.resting` | 休息态（绿环 / 闭眼远眺） |
| `window.fbAutoCollapse` | `false` = 关闭"hover 离开自动收起"，面板全局常驻 |

### 5.2 硬规则

- **面板常驻条件**：`fbAutoCollapse === false`（设置项）**或** `fb-locked`（点击悬浮球）任一成立，离开都不收起。
- **非侵入**：悬浮层 `nonactivating`、不抢焦点；面板失焦自动关闭（除非锁定/常驻）。
- **悬浮层调色板独立**：`.fb-root` 使用固定浅色玻璃调色板，**不跟随母版 `[data-theme]`**——这是有意为之（系统级浮层稳定质感），非"未适配主题"的 bug，详见 [DesignGuide §4.7](../DesignGuide.md)。状态语义色（进行中蓝、分组青绿等）仍与全局一致。
- **详情可开**：所有展示任务 id 均为 `RUN_DETAILS` 已有项，点击统一走 `openRunDetail`，不出现死链。

### 5.3 设置项（Settings 页 / 原型设置区）

| 分区 | 设置 | 作用 |
|------|------|------|
| 悬浮球 | 显示悬浮球 | 隐藏/显示悬浮球本体（不连带 Focus Bar） |
| 悬浮球 | hover 离开后自动收起面板 | 关闭 → 面板常驻（等价点击悬浮球锁定） |
| 悬浮球 | 弹出动画时长 | 面板淡入动画毫秒数 |
| Focus Bar | 默认置顶显示 | 打开主面板时自动显示状态条 |
| Focus Bar | 透明度 | 状态条不透明度 |

## 6. 数据对象

### 6.1 单一同源任务池

浮球**规划清单**与 Focus Bar **六个下拉**的任务**全部从同一个 `FB_POOL` 派生**，杜绝多处各写一份、独立漂移：

- **`FB_POOL`**：`id → { 标题, 状态, 项目, unread, 副标题 }`，浮球域任务属性的唯一来源；所有 id 均 ∈ `RUN_DETAILS`。
- **`planScopes`**：规划清单各 scope 仅存成员 id；**`all`（全局规划）自动 = 今日 ∪ 本周 ∪ 本月并集**，保证全局是各子集的超集。
- Focus Bar 下拉内容与 `bar-num` 由池按状态 / 未读派生，**计数恒等于列表条数**。
- **看板 `workItems` 是独立的真实任务数据，不并入本池**（轻量统一：浮球/Focus Bar 详情走 `openRunDetail`，看板走自身详情，互不改动）。

模型详情见 [Architecture §2 数据模型](../Architecture.md) 与 [PRD](../PRD.md)；两轴状态模型（`status` + `run_state`）权威源见 [04-studio.md §2.4 / §3.2](04-studio.md)。

### 6.2 常驻自检（回归护栏）

母版内置不变量自检，访问 `00-layout-prototype.html?selfcheck`（或 `#selfcheck`）时运行，页面顶部显示 PASS/FAIL 横幅，结果写入 `window.__SELFCHECK__`。校验项：

| 项 | 断言 |
|----|------|
| I1 | 全局规划 = 今日∪本周∪本月并集（防包含倒置） |
| I2 | 三处计数一致（DOM 条数 = `bar-num` = 池派生数） |
| I3 | 今日聚焦下拉 = 今日规划成员 |
| I4 | 所有展示 id 详情可开（∈ `RUN_DETAILS`） |
| I5 | `.float-action` 行恰好 3 列（防箭头换行回归） |
| I6 | 下拉 `data-st`/`data-pj` 与池一致（多源一致） |
| I7 | 面板未弹出时 `.float-panel` 为 `pointer-events:none`（感应区只含悬浮球本体，防"球外空白误触发弹出"回归） |
| I8 | 球外空白点（面板占位区内、球外）`elementFromPoint` 穿透浮层、不落在 `.floating-zone`（真实命中校验，补 I7 只查 CSS 的盲区） |
| I9 | 每个 `bar-num`「计数为 0」⟺ 带 `.is-zero`（状态数字徽章 0 态退化护栏，防 0 露出空徽章） |

**改动浮球/Focus Bar 数据或结构后，带 `?selfcheck` 复跑确认全绿**，避免"断言只贴当轮功能"导致回归漏网。

## 7. 术语

| 术语 | 含义 |
|------|------|
| 悬浮层 `.fb-root` | 悬浮球 + 快捷面板 + Focus Bar 的统称与样式作用域 |
| 快捷面板 / float-panel | hover/单击悬浮球弹出的速览面板 |
| 二级菜单 / sub-menu | 规划行右展开的清单浮层 |
| Focus Bar | 屏幕顶部的实时状态条 |
| bar-group / bar-list / mini-task | Focus Bar 的分组下拉 / 列表容器 / 任务行 |
| 锁定 / fb-locked | 点击悬浮球进入的常驻态（琥珀色） |
| 桥接热区 | 一/二级菜单间填充间隙的透明 hover 区，防"移动途中秒收" |
| 同源池 / FB_POOL | 浮球域任务的唯一数据来源 |
