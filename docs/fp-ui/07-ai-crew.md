> **状态**：草稿

> **关联**：[PRD §3.4 Crew 数字团队](../PRD.md)

---

## 1. 定位

AICrew 是 AI Agent 团队的**管理中心**。用户面对的不是技术概念（Agent/MCP/Skill），而是管理一个"数字团队"——每个成员有角色、擅长领域和工作方式。

### 与 Settings 的职责边界

| 维度 | AICrew | Settings |
|------|--------|---------|
| 管理对象 | Agent 成员（角色、能力、配置） | 全局偏好（快捷键、主题、默认值） |
| 操作粒度 | 单个 Agent 的详细配置 | 引用 AICrew 配置的全局默认值 |

**原则**：AICrew 管"谁能做什么"，Settings 管"默认用谁"。

## 2. 侧边栏

团队成员列表 + 新建入口。

```
┌─ 侧边栏 ──────────────────┐
│  🔍 搜索成员...             │
│                            │
│  团队成员                  │
│  ┌──────────────────────┐ │
│  │ 🧑‍💻 代码工程师       │ │  <- 选中态
│  │    claude-code · 本地 │ │
│  ├──────────────────────┤ │
│  │ 📊 数据分析师         │ │
│  │    claude · 云端      │ │
│  ├──────────────────────┤ │
│  │ 📝 文档撰写           │ │
│  │    claude · 云端      │ │
│  └──────────────────────┘ │
│                            │
│  [+ 新建成员]              │
└────────────────────────────┘
```

- 每行：头像/图标 + 角色名 + Runtime + 部署位置
- 点击成员 -> 工作区显示详情配置
- `[+ 新建成员]` -> 工作区切换为空白配置表单

## 4. 核心交互

| 用户任务 | 操作路径 |
|---------|---------|
| 添加新团队成员 | [+ 新建成员] -> 填写配置 -> 保存 |
| 修改成员能力 | 侧边栏选中成员 -> 工作区编辑配置 -> 自动保存 |
| 删除成员 | 工作区 [删除] -> 确认弹窗 -> 移除 |
| 配置 MCP Server | 工作区展开 MCP 区域 -> 勾选/取消 |
| 设为默认 Agent | 在 Settings 页面引用此成员 |

## 5. 状态与规则

### 成员配置项

| 配置项 | 必填 | 说明 |
|--------|:----:|------|
| 角色名 | 是 | 用户可见的成员名称 |
| 头像 | 否 | 预置图标或自定义 |
| 擅长领域 | 否 | 文本描述，用于任务分配参考 |
| Runtime | 是 | 绑定的 AI Runtime（claude-code / claude / 自定义） |
| 部署位置 | 是 | 本地或云端 |
| 并发数 | 是 | 最大同时执行任务数，默认 1 |
| MCP Server | 否 | 可用的工具集 |
| 常驻职责 | 否 | cron / event 触发的自动任务 |

### V1 预置成员

```
🧑‍💻 代码工程师 — claude-code, 本地, 并发 2
```

V1 仅预置一个成员，用户可新增。

### 引用关系

- Focus 中 Task 的 `assigned_agent` 从 AICrew 选择
- Settings 中的默认 Agent 引用 AICrew 配置
- Agent 并发数在此配置，Focus 执行时受此限制

## 7. 术语

| 术语 | 含义 |
|------|------|
| AICrew | 页面名称（活动栏 🤖） |
| Crew 成员 | 一个配置完整的 AI Agent 角色 |
| Runtime | Agent 执行时绑定的 AI 引擎 |
| 常驻职责 | 自动触发的后台任务（cron 或 event 驱动） |
| 并发数 | 单个 Agent 最大同时执行任务数 |
