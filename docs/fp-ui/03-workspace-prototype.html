看板状态模型实现
● In Progress
📄 规划
🤖 自动
🖐 手动
📝 任务描述
实现 Workspace 看板视图的状态模型，采用 Multica 7 态流转：backlog → todo → in_progress → in_review → done，旁路状态 blocked / cancelled。需要实现拖拽跨列变更状态，状态变更持久化，以及与规划视图的数据同步。
📋 规划文档 ● READY
✓ plan.md · brainstorming 生成

实现步骤

  1. 定义 TaskStatus 枚举，包含 7 种状态及合法转换规则
  2. 创建 KanbanDataSource，负责按 status 分组并驱动 UI
  3. 实现 NSTableView drag & drop 协议，支持跨列拖拽
  4. 状态变更时通过 ConfigStore 持久化
  5. 添加拖拽动画（0.25s ease-out），放置时卡片弹性缩放
  6. 同步更新规划视图中对应节点的 status 标记

验收标准

拖拽跨列后 status 正确更新且持久化
动画流畅无闪烁，响应时间 < 100ms
重启后看板状态保持不丢失
blocked/cancelled 卡片在独立折叠区显示
规划视图中任务状态标记同步变更

技术约束

  • 纯 AppKit 实现，不引入 SwiftUI 视图
  • 复用现有 QuickPanelView 的 NSTableView 架构
  • 状态转换规则在 Model 层校验，UI 层不做业务逻辑
