Feynman Prompt
请不用术语，假设你在向完全不懂状态机的人解释：为什么 `blocked` 恢复后要回到阻塞前状态？
AI 会看你是否抓住了“blocked 是中断而不是新阶段”这个核心规律，并会继续追问遗漏点。
我会这样解释：

`blocked` 就像你做一件事做到一半，被外部条件卡住了。问题解决之后，你应该回到原来做到哪一步，而不是重新被当成“还没开始”。

如果把它直接丢回 `todo`，就会失去真实上下文，团队也看不出来这件事其实已经做到评估或执行阶段了。

原卡片摘要

blocked 不是流程新阶段，而是对当前工作流的临时打断。恢复后要回到阻塞前状态，避免上下文丢失。

Scenario Prompt
