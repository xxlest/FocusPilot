# AreaProjects 页面设计

> **状态**：草稿
> **更新**：2026-04-18

---

## 定位

AreaProjects 是项目的沉淀区，分为两类：
1. **执行类项目**：需要拆解和执行的项目
2. **知识类项目**：知识沉淀和积累

## 侧边栏

### 顶部双行 Tab 栏

**第一行**：搜索栏
- 快速搜索匹配文件夹和文件
- 结果分区显示：上方文件夹，分隔符，下方文件

**第二行**：4 个图标 Tab（仅图标，hover 显示文字）
- 创建文件夹：创建未命名文件夹，已有同名则后缀 1、2、3
- 创建文件：在选中文件夹或根目录下创建，默认 Markdown 格式
- 收藏：按收藏的领域项目过滤展示
- 展开/折叠

### 文件树

类似 Obsidian / VS Code 的文件树导航。

## 工作区

### 左半区

点开文件后显示文件内容，支持 Markdown 预览和原文切换。

### 右半区

点击顶部标题栏最右边，可收起/展开右半工作区。左右分割符可拖动调整面积。

右半区顶部标题栏有"+"号，点击弹出下拉框选择：
- 新建终端
- 新建 Agent 对话框

Agent 对话框可选择：
1. Runtime
2. Claude Code
3. Codebase

## 与 Workspace 的关系

AreaProjects 中的任务可通过「移入 Workspace」操作导入 Workspace，成为 Task（source=area_project）。
