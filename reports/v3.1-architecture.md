# V3.1 架构设计：全部窗口 Tab + 收藏管理

## 概述

将快捷面板从"已选/收藏"双 Tab 重构为"全部/收藏"双 Tab，移除手动添加 App 概念，全部 Tab 自动显示所有运行中 App。

---

## 1.1 模块划分

| 模块 | 文件 | 变更类型 | 说明 |
|------|------|---------|------|
| QuickPanelView | `QuickPanel/QuickPanelView.swift` | **修改** | Tab 枚举 `.selected` → `.all`；"全部"Tab 数据源改为 AppMonitor 全部运行 App；App 行添加右键收藏菜单 |
| AppMonitor | `Services/AppMonitor.swift` | **修改** | `refreshRunningApps()` 改为追踪所有 regular App（不再依赖 ConfigStore.appConfigs） |
| ConfigStore | `Services/ConfigStore.swift` | **修改** | 语义变更：`appConfigs` 仅存储收藏 App；移除 `toggleFavorite()`（用 addApp/removeApp 替代）；添加数据迁移逻辑 |
| Models | `Models/Models.swift` | **修改** | AppConfig 移除 `isFavorite` 字段（在 appConfigs 中即为收藏） |
| AppConfigView | `MainKanban/AppConfigView.swift` | **重写** | 移除"已选/可选"双区，改为"收藏管理"列表（系统 App + 收藏星标切换） |
| MainKanbanView | `MainKanban/MainKanbanView.swift` | **修改** | KanbanTab.appConfig 标题改为"收藏管理" |
| Constants | `Helpers/Constants.swift` | **微调** | `maxApps` 限制仅约束收藏数量，"全部"Tab 不受限 |

**防过度设计自查**：
- 不引入新数据模型，复用 AppConfig + RunningApp
- 不增加新 Service 层，复用 AppMonitor + ConfigStore
- 不增加缓存/本地数据库，继续使用 UserDefaults
- 不引入新的通知类型，复用现有 appStatusChanged / windowsChanged

## 1.2 接口契约

### AppMonitor 数据流变更

**Before（V3.0）**：
```
ConfigStore.appConfigs → AppMonitor.refreshRunningApps() → runningApps（仅已配置 App）
```

**After（V3.1）**：
```
NSWorkspace.runningApplications(.regular) → AppMonitor.refreshRunningApps() → runningApps（所有运行中 App）
```

核心变更：`refreshRunningApps()` 不再遍历 `ConfigStore.appConfigs`，而是直接遍历 `NSWorkspace.shared.runningApplications`，过滤 `activationPolicy == .regular`。

```swift
// AppMonitor.refreshRunningApps() 新逻辑伪代码
func refreshRunningApps() {
    let workspace = NSWorkspace.shared.runningApplications
        .filter { $0.activationPolicy == .regular && $0.bundleIdentifier != nil }
        .sorted { ($0.localizedName ?? "") .localizedCompare($1.localizedName ?? "") == .orderedAscending }

    var updated: [RunningApp] = []
    for nsApp in workspace {
        let bundleID = nsApp.bundleIdentifier!
        let windows = WindowService.shared.listWindows(for: nsApp.processIdentifier)
        let app = RunningApp(
            bundleID: bundleID,
            localizedName: nsApp.localizedName ?? bundleID,
            icon: nsApp.icon ?? NSImage(named: NSImage.applicationIconName)!,
            nsApp: nsApp,
            windows: windows,
            isRunning: true
        )
        updated.append(app)
    }
    runningApps = updated
}
```

### ConfigStore 收藏数据结构

**Before（V3.0）**：
```swift
appConfigs: [AppConfig]  // 手动添加的 App 列表，isFavorite 标记收藏
favoriteAppConfigs: [AppConfig]  // computed: appConfigs.filter { $0.isFavorite }
```

**After（V3.1）**：
```swift
appConfigs: [AppConfig]  // 仅存储收藏的 App（在列表中即为收藏，移除 isFavorite 字段）
```

AppConfig 结构简化：
```swift
struct AppConfig: Codable, Identifiable, Equatable {
    var id: String { bundleID }
    let bundleID: String
    var displayName: String
    var order: Int
    // isFavorite 移除 — 在 appConfigs 中即为收藏
}
```

ConfigStore 接口变更：
```swift
// 移除
func toggleFavorite(_ bundleID: String)
var favoriteAppConfigs: [AppConfig]

// 保留（语义变更）
func addApp(_ bundleID: String, displayName: String)    // 添加收藏
func removeApp(_ bundleID: String)                       // 移除收藏

// 新增便捷方法
func isFavorite(_ bundleID: String) -> Bool {
    appConfigs.contains { $0.bundleID == bundleID }
}
```

### QuickPanelView 数据流

```
"全部" Tab:
  AppMonitor.runningApps → 直接显示所有运行中 App + 窗口
  收藏状态：ConfigStore.isFavorite(bundleID) 用于右键菜单文案

"收藏" Tab:
  ConfigStore.appConfigs → 收藏列表
  运行状态：AppMonitor.runningApps.first(where: bundleID) 判断是否运行
  窗口数据：从 AppMonitor.runningApps 获取
```

## 1.3 行为约束

### Tab 切换数据源

| Tab | 数据源 | 空状态文案 |
|-----|--------|-----------|
| 全部 | `AppMonitor.shared.runningApps`（全部运行中 regular App） | "没有正在运行的应用" |
| 收藏 | `ConfigStore.shared.appConfigs`（收藏列表）+ AppMonitor 运行状态 | "尚未收藏任何应用" |

### "全部" Tab 排序规则

按 App 名称字母序排列（`localizedCompare`），与系统活动监视器一致。排序在 `AppMonitor.refreshRunningApps()` 中完成。

### 收藏持久化方式

继续使用 `AppConfig` 结构体数组（`[AppConfig]`），存储于 UserDefaults key `FocusCopilot.appConfigs`。每个 AppConfig 包含：
- `bundleID`: 唯一标识
- `displayName`: 显示名称
- `order`: 排序序号（收藏列表内排序）

移除 `isFavorite` 字段，在 `appConfigs` 中存在即表示已收藏。

### 收藏 App 未运行时的显示行为

- 灰度显示（`alphaValue = 0.5`，name 用 `.tertiaryLabelColor`）
- 状态指示器：灰色圆点（⚪）
- 图标：通过 `NSWorkspace.shared.icon(forFile:)` 获取（不依赖运行中进程）
- 点击行为：启动该 App
- 不显示窗口列表

### 右键菜单

**"全部" Tab App 行右键菜单**：
- "添加到收藏" — 当 `!ConfigStore.shared.isFavorite(bundleID)` 时显示
- "从收藏中移除" — 当 `ConfigStore.shared.isFavorite(bundleID)` 时显示

**"收藏" Tab App 行右键菜单**：
- "从收藏中移除"

**窗口行右键菜单**：保持不变（重命名窗口）

## 1.4 影响分析（增量场景）

### 需修改的模块

1. **AppMonitor.refreshRunningApps()**
   - 移除对 ConfigStore.appConfigs 的依赖
   - 遍历所有 `NSWorkspace.shared.runningApplications`（`.regular`）
   - 按 localizedName 排序

2. **AppMonitor.refreshAllWindows()**
   - 当前逻辑已遍历 runningApps，无需大改
   - 但 `listWindows(for: bundleID)` 调用需保留

3. **ConfigStore**
   - 移除 `toggleFavorite()`、`favoriteAppConfigs`
   - 添加 `isFavorite(_ bundleID: String) -> Bool`
   - `addApp()` 内的 `isFavorite: false` 参数移除
   - 添加 V3.1 数据迁移方法

4. **AppConfig（Models.swift）**
   - 移除 `isFavorite` 属性
   - 自定义 decoder 移除 `isFavorite` 解码（保留向后兼容，忽略旧字段）

5. **QuickPanelView**
   - `QuickPanelTab.selected` → `.all`
   - 按钮文案 "已选" → "全部"
   - `buildContent()` 的 `.all` 分支：从 AppMonitor.runningApps 取数据
   - `buildContent()` 的 `.favorites` 分支：从 ConfigStore.appConfigs + AppMonitor 运行状态
   - App 行添加 `contextMenuProvider`（右键收藏菜单）
   - 移除"全部"Tab 的 maxApps 限制
   - `resetToNormalMode()` 中 `.selected` → `.all`

6. **AppConfigView（MainKanban）**
   - 重写为"收藏管理"页面
   - 移除"已选应用"拖拽排序区
   - 移除"可选应用"勾选添加区
   - 新增：系统 App 列表（运行中 + 已安装），星标切换收藏

7. **MainKanbanView**
   - `KanbanTab.appConfig` rawValue 改为 "收藏管理"

### 数据迁移

在 `ConfigStore.load()` 中添加一次性迁移：

```swift
// V3.1 迁移：移除非收藏的 appConfigs
private func migrateToV31() {
    let migrationKey = "FocusCopilot.v31Migrated"
    guard !defaults.bool(forKey: migrationKey) else { return }

    // 过滤：仅保留 isFavorite == true 的配置
    // 旧数据中 isFavorite 可能不存在（默认 false），这些会被移除
    if let data = defaults.data(forKey: Constants.Keys.appConfigs),
       let oldConfigs = try? decoder.decode([OldAppConfig].self, from: data) {
        let favorites = oldConfigs.filter { $0.isFavorite }
        // 转为新格式（无 isFavorite 字段）并保存
        let newConfigs = favorites.enumerated().map { index, old in
            AppConfig(bundleID: old.bundleID, displayName: old.displayName, order: index)
        }
        if let newData = try? encoder.encode(newConfigs) {
            defaults.set(newData, forKey: Constants.Keys.appConfigs)
        }
    }

    defaults.set(true, forKey: migrationKey)
}
```

注：使用临时的 `OldAppConfig` 结构（包含 isFavorite）来解码旧数据，或在 AppConfig 的 decoder 中兼容处理（忽略 isFavorite 字段）。由于 AppConfig 的自定义 decoder 已经用 `decodeIfPresent` 处理 isFavorite，迁移可以更简单：在 load() 之后过滤掉非收藏项。

## 1.5 验收用例

### UC-1: 全部 Tab 显示所有运行中 App
- **前置**：系统运行 Safari、Xcode、Terminal 三个 App
- **操作**：打开快捷面板，默认"全部"Tab
- **预期**：列表显示三个 App，按名称字母序排列，每个 App 显示其窗口

### UC-2: 全部 Tab 实时响应 App 启退
- **前置**："全部"Tab 已打开
- **操作**：启动 Finder / 退出某个 App
- **预期**：列表自动更新，新 App 出现 / 退出 App 消失

### UC-3: 右键收藏 App
- **前置**："全部"Tab 显示 Safari
- **操作**：右键 Safari 行 → "添加到收藏"
- **预期**：Safari 被收藏；再次右键显示"从收藏中移除"

### UC-4: 收藏 Tab 显示
- **前置**：已收藏 Safari、Terminal；Safari 运行中，Terminal 已退出
- **操作**：切换到"收藏"Tab
- **预期**：Safari 正常显示（绿点 + 窗口列表）；Terminal 灰度显示（灰点 + 无窗口 + 点击可启动）

### UC-5: 收藏持久化
- **前置**：收藏了 Safari
- **操作**：退出并重启 Focus Copilot
- **预期**：收藏 Tab 仍显示 Safari

### UC-6: 收藏 Tab 右键取消收藏
- **前置**："收藏"Tab 显示 Safari
- **操作**：右键 Safari 行 → "从收藏中移除"
- **预期**：Safari 从收藏列表消失

### UC-7: MainKanban 收藏管理
- **前置**：打开主看板"收藏管理"页
- **操作**：浏览系统 App 列表，点击星标收藏 Xcode
- **预期**：Xcode 星标高亮；快捷面板"收藏"Tab 出现 Xcode

### UC-8: 数据迁移
- **前置**：V3.0 用户已有 5 个 appConfigs，其中 2 个 isFavorite=true
- **操作**：升级到 V3.1 首次启动
- **预期**：appConfigs 仅保留 2 个收藏 App；"全部"Tab 正常显示所有运行中 App

## 1.6 非目标声明

以下功能不在 V3.1 范围内：

1. **收藏排序/拖拽**：收藏列表按添加顺序排列，不支持手动排序（后续可加）
2. **"全部"Tab 搜索/过滤**：不添加搜索框，App 数量通常不多
3. **收藏分组/文件夹**：不引入分组概念
4. **App 快捷操作**：不在右键菜单中添加"退出 App"等操作
5. **新 Onboarding 流程**：不修改引导页
6. **收藏上限变更**：保持 8 个上限（`Constants.maxApps`）
7. **全部 Tab 记住折叠状态**：App 启退后折叠状态重置（当前行为保持）
